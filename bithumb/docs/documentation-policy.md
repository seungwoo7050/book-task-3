# 문서 정책

## 목적

이 저장소의 문서는 학습 과정을 숨기지 않으면서도, 다른 사람이 빠르게 이해하고 재현할 수 있도록 돕는 것을
목표로 합니다. 문서의 역할은 코드 설명을 대신하는 것이 아니라, 문제 정의와 검증 기준을 분명하게 만드는 것입니다.

## 기본 원칙

- 한국어를 기본 언어로 사용하고, 기술명·패키지명·CLI 명령·설정 키는 원문을 유지합니다.
- 모든 명령은 레포 루트 기준으로 적습니다.
- 추상적인 감상보다 재현 가능한 명령, 파일 경로, 입력과 출력을 우선합니다.
- 학습용 레포라는 점을 숨기지 않되, 그대로 복제한 포트폴리오가 아니라 스스로 설명을 다시 세우도록 돕는 방향으로 씁니다.

## README 작성 원칙

프로젝트 `README.md`는 아래 순서를 기본으로 유지합니다.
- 프로젝트 한줄 소개
- 왜 배우는가
- 현재 구현 범위
- 빠른 시작
- 검증 명령
- 먼저 읽을 파일
- 포트폴리오 확장 힌트
- 알려진 한계

`docs/README.md`는 링크 목록이 아니라 “이 문서 묶음으로 무엇을 이해할 수 있는지”를 먼저 설명합니다.
`python/README.md`는 구현 범위, 실행 예시, 테스트, 상태, 구현 메모를 간단히 정리합니다.

## notion 정책

- `notion/`은 공개용 백업 문서로 레포에 포함합니다.
- 새 `notion/`은 `README.md`, `00-problem-framing.md`, `01-approach-log.md`, `02-debug-log.md`, `03-retrospective.md`, `04-knowledge-index.md`, `05-reproduction-guide.md`를 기본 세트로 사용합니다.
- `05-reproduction-guide.md`는 학습자가 가장 빠르게 재현할 수 있도록, 레포 루트 기준 명령, 기대 결과, 실패 시 먼저 볼 파일을 구체적으로 적습니다.
- 기존 `notion/`을 다시 쓰고 싶으면 먼저 기존 폴더를 `notion-archive/`로 보존합니다.
- `notion-archive/`는 과거 기록을 보존하는 용도이므로, 오래된 명령이나 표현이 남아 있을 수 있습니다.

## 포트폴리오 확장 원칙

이 저장소를 참고하는 사람에게는 다음 네 가지를 스스로 다시 쓰라고 안내합니다.
- 내가 풀고 있는 문제는 무엇인가
- 어떤 제약을 실제로 선택했는가
- 무엇을 검증했고 아직 무엇이 비어 있는가
- 왜 이 구현이 다음 프로젝트로 이어지는가

이 레포의 문서는 그 질문에 답하는 예시를 제공해야 하며, 결과 코드만 나열하는 식으로 쓰지 않습니다.
