# 문제 정리

## 원래 문제

보안 거버넌스에서 중요한 예외와 증적 흐름을 작은 메모리 모델로 먼저 구현해야 합니다.
중요한 점은 예외를 단순 mute가 아니라 승인, 만료, 증적, audit가 분리된 관리 대상 레코드로 다루는 것입니다.

## 제공된 자료

- CLI demo가 생성하는 finding scope
- 예외 요청 정보
- 증적 메타데이터와 audit event 흐름

## 제약

- 영속 저장소는 사용하지 않습니다.
- 외부 티켓 시스템이나 승인 워크플로와 연동하지 않습니다.

## 통과 기준

- 예외 승인 상태가 생성되고 증적이 연결되어야 합니다.
- audit event가 append-only 개수로 검증되어야 합니다.
- 만료와 scope 개념이 테스트로 고정되어야 합니다.

## 이번 프로젝트에서 일부러 제외한 것

- DB 영속화
- 외부 티켓 시스템 연동
- 조직 단위 예외 승인 체계
