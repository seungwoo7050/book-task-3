# 접근 기록

## 핵심 선택

- 서로 다른 로그 포맷을 바로 비교할 수 있도록 `EventRecord` 같은 공통 필드 집합을 먼저 정했습니다.
- 저장소는 별도 서버가 필요 없는 DuckDB를 택하고, 결과를 Parquet로도 남겼습니다.
- ETL 전체를 작은 함수 몇 개로 유지해 정규화 -> 적재 -> 요약 흐름이 한 파일 안에서 보이도록 했습니다.

## 버린 대안

- CloudTrail 원본 스키마를 전부 보존하는 방향은 학습 범위에 비해 지나치게 무겁다고 판단했습니다.
- 외부 데이터베이스나 분산 저장소는 설치/운영 설명이 본질을 가려 배제했습니다.
- 시계열 이상 탐지까지 확장하지 않고, 먼저 queryable한 상태를 만드는 데 집중했습니다.

## 다음 프로젝트와의 연결

`07-security-lake-mini`는 바로 이 정규화 감각 위에서 detection query를 얹습니다. 그래서 지금 단계에서 중요한 것은
로그를 많이 모으는 것이 아니라, 어떤 공통 필드를 뽑아야 이후 query가 쉬워지는지를 이해하는 것입니다.

## 다시 써도 유지할 기준

- 정규화 기준이 분명해야 합니다.
- 입력과 출력 경로가 명시적이어야 합니다.
- 요약 쿼리가 실제로 학습 효과를 만들어야 합니다.
