### Session 1

C++ companion도 같은 payload structure test를 통과하는지 확인했다. 별 문제 없었는데, 하나 걸렸던 건 `memcmp`와 `matches_u64_le`의 byte order 가정이었다. little-endian을 명시적으로 가정하고 있어서, 만약 big-endian 시스템에서 돌리면 깨진다. 하지만 이 프로젝트의 target이 x86-64이니까 그건 무시하기로 했다.

```bash

$ cd ../cpp
$ make clean && make test
```

official target 경로와 companion verifier 경로 모두 유지.

## 2026-03-11
### Session 1
publication policy를 문서에 확정했다. 원칙은 단순하다: gadget 주소와 payload shape는 설명하되, 외부 target에 바로 복붙할 수 있는 raw exploit는 만들지 않는다.

이 정책을 정하면서 느낀 건, companion validator라는 형식이 이 딜레마를 깔끔하게 풀어 준다는 거다. validator 코드 자체가 "이 payload는 이런 구조여야 한다"를 선언하니까, 직접 exploit을 배포하지 않으면서도 학습 내용을 충분히 전달할 수 있다.

CLI:

```bash
$ cd c && make test
$ cd ../cpp && make test
```

공개 표면에는 구조 설명과 검증 경로만 남긴다.
