# Bomb Lab 개발 타임라인 (2026-03-08 ~ 2026-03-09)

## 2026-03-08
### Session 1
bomb류 프로젝트의 최대 딜레마부터 맞닥뜨렸다. 공식 바이너리를 열어서 phase 구조를 분석하는 건 학습의 핵심인데, 그걸 그대로 공개하면 답안집이 된다. 정답 문자열을 올리는 건 확실히 안 되고, 그렇다고 아무것도 안 올리면 "여기서 뭘 배웠는지"가 남지 않는다.

결국 companion mini-bomb이라는 개념을 세웠다. 공식 bomb의 각 phase 패턴을 — 문자열 비교, 수열 검증, switch 분기, linked list 재배열, tree search — C 함수로 다시 구현한다. 정답 자체가 아니라 "이 phase가 어떤 제어 흐름 패턴을 강제하는가"를 보여 주는 거다.

CLI:

```bash
$ cd problem
$ make restore-official
$ make verify-official
$ cd ../c
$ make clean && make test
```

official verify 경로가 6개 phase를 돈다. companion test도 통과.

### Session 2
초기 phase는 비교적 순탄했다. phase 1은 단순 문자열 비교, phase 2는 `1 2 4 8 16 32` 같은 기하 수열 검증, phase 3은 switch-case lookup. 여기까지는 패턴이 뻔했다.

문제는 phase 4부터였다. 재귀 함수의 path code를 맞춰야 하는데, 처음엔 함수 자체를 이해하기보다 brute force로 답을 찾으려고 했다. 그러다가 "이 함수가 뭘 하는 건지"를 먼저 이해하니까 답이 자연스럽게 나왔다. `func4(value, 0, 14) == 0`이 되려면 binary search에서 항상 왼쪽으로만 가야 한다는 걸 깨달은 시점.

## 2026-03-09
### Session 1
phase 6이 이 프로젝트에서 가장 오래 걸린 부분이었다. 입력 6개 숫자의 중복 검사 → `7 - value` 재매핑 → linked list 재배열 → 내림차순 검증. 각 단계가 왜 필요한지를 한 번에 이해하기가 어려웠다.

처음엔 `7 - value` 재매핑이 왜 있는지 몰랐다. 나중에 보니 node 값의 내림차순이 목표인데, 입력은 재매핑 전 기준이라서 "어떤 순서로 입력해야 재매핑 후 node가 내림차순이 되는가"를 거꾸로 풀어야 했다.

```c
int bomb_phase_6(const char *input)
{
    int values[6];
    int seen[7] = {0};
    Node nodes[6];
    Node *ordered[6];
    int index;

    if (!parse_six_ints(input, values)) return 0;
    for (index = 0; index < 6; ++index) {
        if (values[index] < 1 || values[index] > 6 || seen[values[index]])
            return 0;
        seen[values[index]] = 1;
        values[index] = 7 - values[index];
    }
    reset_nodes(nodes);
    for (index = 0; index < 6; ++index)
        ordered[index] = node_at(nodes, values[index]);
    for (index = 0; index < 5; ++index)
        ordered[index]->next = ordered[index + 1];
    ordered[5]->next = NULL;
    for (index = 0; index < 5; ++index)
        if (ordered[index]->value < ordered[index + 1]->value) return 0;
    return 1;
}
```

이 코드를 companion으로 만들고 나서야 phase 6을 "숫자 맞추기"가 아니라 "자료구조 재배열 문제"로 볼 수 있게 됐다. 정답 문자열을 외우는 것과 패턴을 이해하는 건 완전히 다른 일이었다.

### Session 2
secret phase를 잡았다. phase 6까지 다 풀면 숨겨진 입구가 열리는데, 여기서 binary tree를 탐색하며 path code `6`을 만들어야 한다.

```c
int bomb_secret_phase(const char *input)
{
    TreeNode n6 = {6, NULL, NULL};
    TreeNode n20 = {20, NULL, NULL};
    TreeNode n35 = {35, NULL, NULL};
    TreeNode n45 = {45, NULL, NULL};
    TreeNode n99 = {99, NULL, NULL};
    TreeNode n1001 = {1001, NULL, NULL};
    TreeNode n22 = {22, &n20, &n35};
    TreeNode n107 = {107, &n99, &n1001};
    TreeNode n8 = {8, &n6, &n22};
    TreeNode n50 = {50, &n45, &n107};
    TreeNode root = {36, &n8, &n50};
    int value;
    if (!parse_one_int(input, &value)) return 0;
    return fun7(&root, value) == 6;
}
```

처음엔 `fun7`이 뭘 반환하는지 감이 안 와서 여러 값을 넣어 봤는데, 나중에 보니 왼쪽으로 갈 때 `2 * result`, 오른쪽으로 갈 때 `2 * result + 1`, 같으면 `0`을 반환하는 규칙이었다. path code `6 = 110₂`이니까 right-right-left 순서로 가면 나오는 값 `35`가 답이다.
