# CS-Core 블로그

cs-core 프로젝트 전체의 개발 타임라인 모음.
각 프로젝트를 작업하면서 실제로 겪은 시행착오, 가설, 수정 과정을 시간 순서대로 남긴다.

## 트랙 구성

| 트랙 | 프로젝트 수 | 내용 |
|------|------------|------|
| [Foundations-CSAPP](Foundations-CSAPP/) | 5 | Data Lab, Architecture Lab, Bomb Lab, Attack Lab, Performance Lab |
| [Systems-Programming](Systems-Programming/) | 3 | Malloc Lab, Shell Lab, Proxy Lab |
| [Operating-Systems-Internals](Operating-Systems-Internals/) | 4 | Scheduling, Virtual Memory, Filesystem, Synchronization |
| [Programming-Languages-Foundations](Programming-Languages-Foundations/) | 3 | Parser-Interpreter, Static Type Checking, Bytecode IR |

## 파일 번호 규칙

- `00-series-map.md` — 프로젝트 개요와 타임라인 목차
- `10-*.md`, `20-*.md`, ... — 날짜 순서 타임라인 챕터

## 글쓰기 원칙

이 블로그는 "이미 알고 있었던 것"을 정리한 문서가 아니라, 작업 당시의 탐색 과정을 재현한 기록이다.
가설이 틀렸던 순간, 예상과 다른 동작을 만난 순간, 그것을 어떻게 수정했는지가 핵심이다.
