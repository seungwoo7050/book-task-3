# Shell Lab 개발 타임라인 (2026-03-10 ~ 2026-03-11)

## 2026-03-10
### Session 1
SIGINT와 SIGTSTP handler를 정리했다. 처음엔 handler에서 직접 job state를 바꾸는 걸로 짰는데, foreground child가 있는지 확인하고 kill을 보내는 게 전부여야 했다.

```c
static void sigint_handler(int sig)
{
    int saved_errno = errno;
    pid_t pid = fgpid(jobs);
    if (pid > 0) {
        kill(-pid, SIGINT);
    }
    errno = saved_errno;
}
```

`kill(-pid, SIGINT)`에서 `-pid`가 process group whole에 signal을 보내는 뜻이라는 걸 처음에 몰랐다. child process가 fork될 때 `setpgid(0, 0)`으로 자신의 process group에 넣으면, 이후 negative pid로 group 전체에 signal을 보낼 수 있다. shell이 Ctrl-C를 받아도 shell 자신이 아니라 foreground job에 전달해야 하니까 이 구조가 필요했다.

### Session 2
race condition이 하나 있었다. child를 fork한 직후 parent가 addjob을 하기 전에 SIGCHLD가 오면, handler가 deletejob을 호출하는데 job이 아직 없어서 무시된다. 이후 addjob이 이루어지면 job은 있는데 아무도 삭제해 주지 않는다.

해결은 fork 전에 SIGCHLD를 block하고, addjob 직후에 unblock한다. 이렇게 하면 child가 끝나도 SIGCHLD handler는 addjob이 완료될 때까지 pending 상태로 대기한다.

## 2026-03-11
### Session 1
이 프로젝트에서 배운 신호 처리의 핵심은 세 가지였다. (1) signal handler는 async로 실행되므로 errno 보존이 필수, (2) SIGCHLD handler가 job table을 갱신하고 waitfg는 job table을 관찰하는 구조, (3) fork-addjob 사이의 race를 signal block으로 닫는다.

이 세 가지를 모두 글로 적으면 짧은데, 각각을 몸으로 체득하기까지는 꽤 걸렸다.
