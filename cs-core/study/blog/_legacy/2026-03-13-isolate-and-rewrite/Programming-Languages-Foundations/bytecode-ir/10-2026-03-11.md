# Bytecode IR 개발 타임라인 (2026-03-11)

## 2026-03-11
### Session 1
이 프로젝트는 parser-interpreter 위에 컴파일러와 VM을 얹는다. 처음 질문은 "bytecode instruction set을 어떻게 설계할 것인가"였다. 너무 복잡하면 VM 구현이 어렵고, 너무 단순하면 closure 같은 기능을 표현할 수 없다.

결국 선택한 instruction들: `PUSH_CONST`, `PUSH_LOCAL`, `PUSH_CAPTURE`, `STORE_LOCAL`, `BINARY_OP`, `UNARY_OP`, `JUMP_IF_FALSE`, `JUMP`, `CALL`, `RETURN`, `MAKE_CLOSURE`, `LOAD_CLOSURE`.

가장 고민한 건 closure 표현이었다. compiler가 함수를 컴파일하면 `FunctionProto`가 만들어진다. 이 proto에는 instruction list뿐만 아니라 capture해야 할 변수의 목록(`capture_names`, `capture_sources`)도 들어있다.

```python
@dataclass(frozen=True)
class FunctionProto:
    name: str
    arity: int
    instructions: tuple[Instruction, ...]
    local_count: int
    capture_names: tuple[str, ...]
    capture_sources: tuple[SourceRef, ...]
```

runtime에서 `MAKE_CLOSURE` instruction은 `FunctionProto`와 현재 frame의 upvalue들을 묶어 closure object를 만든다.

### Session 2
compiler의 name resolution이 복잡했다. local variable인지, outer scope의 capture인지, 전역인지를 구분해야 한다.

```python
def resolve(self, name: str) -> SourceRef | None:
    if name in self.local_bindings:
        return SourceRef("local", self.local_bindings[name], name)
    if name in self.capture_bindings:
        return SourceRef("capture", self.capture_bindings[name], name)
    if self.parent is None:
        return None
    outer = self.parent.resolve(name)
    if outer is None:
        return None
    ...
```

`CompileContext`가 parent를 참조하는 chain 구조다. "local" 또는 "capture"로 먼저 찾고, 없으면 parent에 재귀한다. outer에서 찾으면 현재 context의 capture list에 추가하고 `SourceRef("capture", ...)` 로 반환한다.

이 resolve 과정을 따라가면서 "closure가 어떻게 outer variable을 capture하는지"를 컴파일러 관점에서 이해할 수 있었다. evaluator에서는 environment를 통째로 들고 다녔는데, bytecode compiler는 필요한 변수만 명시적으로 capture list에 올린다는 차이가 있었다.

### Session 3
VM의 call frame 관리가 까다로웠다. 함수 호출 시 새 frame을 push하고, RETURN에서 이전 frame으로 pop하면 된다는 건 알았는데, local 변수의 offset을 어떻게 관리하는지가 처음엔 불명확했다.

각 frame이 자신의 local 변수 배열을 들고 있다. instruction stream의 `PUSH_LOCAL 0`은 현재 frame의 locals[0]을 읽는 뜻이다. frame이 분리돼 있으니까 재귀 호출이 이전 frame의 locals를 덮어쓰지 않는다.
