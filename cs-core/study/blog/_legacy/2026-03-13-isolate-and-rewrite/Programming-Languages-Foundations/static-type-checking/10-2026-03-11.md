# Static Type Checking 개발 타임라인 (2026-03-11)

## 2026-03-11
### Session 1
parser-interpreter 위에 type checker를 얹는 형태라, AST와 lexer/parser는 이미 있었다. 새로 필요한 건 `TypeEnvironment`와 `check_expression` 함수뿐이다.

처음 설계한 `TypeEnvironment`는 단순한 dict였다. `name: str → type: TypeExpr`. 그런데 nested scope(`let x = 1 in let x = true in ...`)에서 inner scope의 `x`가 outer `x`를 shadow해야 했다. single dict로는 shadow를 표현할 수 없다.

```python
@dataclass
class TypeEnvironment:
    bindings: dict[str, TypeExpr] = field(default_factory=dict)
    parent: "TypeEnvironment | None" = None

    def lookup(self, name: str, line: int, column: int) -> TypeExpr:
        if name in self.bindings:
            return self.bindings[name]
        if self.parent is not None:
            return self.parent.lookup(name, line, column)
        raise TypeDiagnosticError(Diagnostic(f"unbound name '{name}'", line, column))
```

parent pointer로 scope chain을 만들었다. `let` expression이 새 scope를 만들 때마다 `TypeEnvironment` 인스턴스를 새로 만들어 parent를 연결한다.

### Session 2
함수 타입 checking이 예상보다 섬세했다. `fun (x: Int) -> Int = x + 1`에서 타입 checker는 무엇을 해야 하는가?

1. 파라미터 `x`를 `Int` 타입으로 새 scope에 바인딩
2. body `x + 1`을 그 scope에서 check
3. body 타입이 선언된 return type `Int`와 같은지 확인
4. 전체 expression의 타입은 `Int -> Int`

```python
if isinstance(expr, FunExpr):
    fun_env = TypeEnvironment(parent=env)
    for param in expr.params:
        if param.annotation is not None:
            fun_env.define(param.name, param.annotation)
    body_type = check_expression(expr.body, fun_env)
    ...
```

파라미터에 annotation이 없으면 어떻게 할지 결정해야 했다. 가장 간단한 방법은 "annotation이 없는 파라미터는 type check 대상에서 제외"였다. 즉, 완전히 annotated된 함수만 타입 검사를 통과한다. 이렇게 하면 partial annotation도 허용해야 했는데, "annotation 없으면 untyped로 처리"로 타협했다.

### Session 3
에러 메시지 품질이 생각보다 중요했다. 처음엔 "type error at line X"만 출력했는데, "expected Int, got Bool"처럼 기대 타입과 실제 타입을 같이 출력해야 유용하다.

```python
class Diagnostic:
    message: str
    line: int
    column: int
```

`Diagnostic`에 line과 column을 넣으니까 "어디서" 틀렸는지가 출력에 나왔다. 처음엔 `raise TypeError(message)` 방식으로 짰는데, column 정보를 `TypeError`에 넣기가 어색해서 `TypeDiagnosticError`로 wrapping했다.
