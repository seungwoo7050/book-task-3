# Architecture Lab 개발 타임라인 (2026-03-10 ~ 2026-03-11)

## 2026-03-10
### Session 1
공식 benchmark를 돌렸다. `Average CPE 9.16`, `Score 26.8/60.0`. 점수가 절반도 안 되는 게 좀 씁쓸했지만, 이 프로젝트의 목표는 만점이 아니라 "왜 unroll이 CPE를 낮추는지 설명 가능한 형태로 남기는 것"이다.

처음엔 CPE 자체를 더 줄이려고 `iaddq`를 적극적으로 활용하는 변형을 시도했다. 그런데 companion C 모델에서는 실제 Y86 pipeline 효과를 정확히 재현할 수 없으니까, 공식 benchmark 결과를 텍스트로 기록하고 해석은 companion 코드에서 맡기는 구조로 타협했다.

CLI:

```bash
$ cd problem
$ make verify-official
$ cd ../cpp
$ make clean && make test
```

C++ companion에서도 optimized pseudo-CPE가 baseline보다 낮은 걸 확인했다. 두 언어가 같은 의미 모델을 공유한다는 건 이 시점에서 확정.

### Session 2
8-way unroll로 다시 실험해 봤다. Y86 instruction memory 제한(1000 bytes)에 걸려서 실패. 나중에 보니 이 제한 자체가 "무한정 unroll하면 안 된다"는 교육적 메시지였다. unroll factor를 올릴수록 remainder 처리 코드가 길어져서, 어느 지점부터는 코드 크기 대비 CPE 개선이 사라진다. 결국 4-way가 이 제약 안에서의 sweet spot이었다.
