# Notion Documents Overview

This directory contains six narrative files created during the lab:

- `00-problem-framing.md`: high-level description of the task and why it's hard.
- `01-approach-log.md`: a chronological account of your design decisions, brainstorming, and chosen strategy.
- `02-debug-log.md`: debugging diary with bugs encountered, symptoms, root causes, and fixes.
- `03-retrospective.md`: post-mortem analysis: what was hard, what you learned, and possible improvements.
- `04-knowledge-index.md`: technical reference / cheat sheet of core concepts used in this project.
- `05-development-timeline.md`: step-by-step log of CLI commands, environment notes, and build/test flow not captured in code.

## How to read

- If you just want to understand the problem, start with `00-problem-framing`.
- If you're trying to reproduce or improve the implementation, read `01-approach-log` and `05-development-timeline`.
- When tracking down bugs, `02-debug-log` shows what went wrong and how it was fixed.
- For future review or teaching, `03-retrospective` summarizes lessons learned.
- For quick technical reference, jump to `04-knowledge-index`.
