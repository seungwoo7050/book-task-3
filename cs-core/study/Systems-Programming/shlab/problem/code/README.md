# Shell Lab starter code 경계

## 이 디렉터리가 가르치는 것

이 디렉터리는 공개 트리에서 제거된 starter shell 코드의 경계를 설명합니다.

## 누구를 위한 문서인가

- starter shell이 왜 비어 있는지 빠르게 알고 싶은 사람
- 실제 구현 파일 위치를 찾고 싶은 사람

## 먼저 읽을 곳

1. [`../README.md`](../README.md)
2. [`../../c/src/tsh.c`](../../c/src/tsh.c)
3. [`../../cpp/src/tsh.cpp`](../../cpp/src/tsh.cpp)

## 디렉터리 구조

```text
code/
  README.md
```

## 검증 방법

- 검증은 이 디렉터리에서 하지 않습니다.
- 실행 경로는 [`../../c/README.md`](../../c/README.md)와 [`../../cpp/README.md`](../../cpp/README.md)를 따릅니다.

## 스포일러 경계

- 제거된 starter 자산 자체는 공개하지 않습니다.
- README는 대체 구현 위치만 안내합니다.

## 포트폴리오로 확장하는 힌트

- 비운 디렉터리에도 "왜 비어 있는가"를 설명하는 README를 두면 혼란이 줄어듭니다.
