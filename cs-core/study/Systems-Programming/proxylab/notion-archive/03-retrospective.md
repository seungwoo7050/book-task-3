# Proxy Lab — 회고

## 세 마일스톤의 교육 설계

이 랩의 세 마일스톤 구조가 교육적으로 정말 잘 설계되어 있다고 느꼈다.

마일스톤 1(순차 전달)은 네트워크 프로그래밍의 기본을 다룬다. 소켓을 열고, 데이터를 읽고 쓰고, 닫는다. HTTP 프로토콜의 구조를 이해하고, URI 파싱과 헤더 재작성을 구현한다. 여기까지는 단일 스레드로 동작하므로, 동시성 없이 네트워크 I/O에만 집중할 수 있다.

마일스톤 2(동시 처리)는 스레딩을 추가한다. 코드 변경은 사실 적다 — `pthread_create`와 `pthread_detach` 몇 줄이면 된다. 하지만 이 몇 줄이 만드는 차이가 크다: 느린 요청 하나가 다른 모든 요청을 블록하던 것이, 이제는 독립적으로 처리된다. 동시성 테스트(두 개의 2초 요청이 총 3.5초 이내에 완료)가 이것을 확인한다.

마일스톤 3(캐시)는 동시성 + 공유 상태의 조합이다. 캐시 없이는 스레드 간 공유 상태가 없어서 락이 필요 없었다. 캐시를 추가하는 순간 뮤텍스가 필수가 된다. 그리고 "락 안에서 I/O를 하면 동시성이 죽는다"는 교훈을 직접 체험하게 된다.

## 테스트 하네스의 가치

공식 드라이버 없이 검증 시스템을 직접 만든 것이 이 프로젝트의 큰 수확이었다.

Python `ThreadingHTTPServer`를 원본 서버로 사용하면, 테스트 시나리오를 완전히 통제할 수 있다. `/slow/*`는 2초 딜레이로 동시성을 테스트하고, `/cacheable/*`는 히트 카운터로 캐시 동작을 검증하고, `/large/*`는 120KB 응답으로 MAX_OBJECT_SIZE 정책을 확인한다, `/headers`는 프록시의 헤더 재작성을 에코한다.

이 원본 서버와 `curl -x`(프록시 경유 요청)를 결합한 쉘 하네스가 모든 마일스톤을 자동으로 검증한다. `make test` 한 번으로 전달, 헤더, 캐시, 동시성, 실패 복구를 모두 확인할 수 있다.

## 뮤텍스 하나로 충분한가

캐시에 뮤텍스 하나만 사용한 것은 의도적으로 보수적인 선택이다. 프로덕션 프록시라면 읽기-쓰기 락(`pthread_rwlock`)이나 세그먼티드 락을 사용해서 읽기 동시성을 높여야 한다.

하지만 이 프로젝트의 목표는 고성능 프록시가 아니라 "공유 상태에 대한 올바른 동기화"를 이해하는 것이다. 뮤텍스 하나로 시작하면 정확성을 보장하는 것이 쉽고, 디버깅도 단순하다. 최적화는 정확한 코드 위에서만 의미가 있다.

## 에러 처리의 균형

프록시에서 에러 처리의 범위를 어디까지로 할 것인가가 고민이었다.

최종적으로 선택한 범위:
- 잘못된 요청 라인 → 400 Bad Request
- GET이 아닌 메서드 → 501 Not Implemented
- 원본 서버 연결 실패 → 502 Bad Gateway
- SIGPIPE → 무시 (해당 연결만 정리)

이것이 모든 HTTP 에러 상황을 커버하지는 않지만, 랩의 학습 목표를 달성하기에 충분하다. 완전한 HTTP 프록시를 만들려면 chunked encoding, SSL tunneling, 인증 등을 추가해야 하는데, 그것은 다른 프로젝트의 범위다.

## csapp.c 의존성에 대한 생각

이 프로젝트는 CS:APP의 `csapp.c`/`csapp.h`에 의존한다. `open_clientfd()`, `open_listenfd()`, `rio_*()` 등의 래퍼 함수가 소켓 프로그래밍의 보일러플레이트를 숨겨준다.

교육적으로는 좋은 설계다 — 소켓 프로그래밍의 세부사항에 빠지지 않고 프록시의 논리에 집중할 수 있다. 하지만 이 의존성 때문에 `problem/code/`에 `csapp.c`/`csapp.h`를 보관하고 빌드 시 포함해야 한다.

## 다음에 다시 한다면

- 읽기-쓰기 락으로 캐시 동시성을 높이겠다. 캐시 읽기는 여러 스레드가 동시에 할 수 있어야 하고, 쓰기만 독점 락이 필요하다.
- 캐시 히트/미스 통계를 기록하겠다. 히트율을 추적하면 캐시 정책의 효과를 수치로 확인할 수 있다.
- HTTP CONNECT 메서드를 추가하겠다. HTTPS 터널링은 GET 전달보다 훨씬 복잡하고, 실제 프록시에서 필수적인 기능이다.
