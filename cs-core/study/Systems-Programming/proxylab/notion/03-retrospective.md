# 03. 회고

## 이번 프로젝트에서 크게 남은 교훈

프록시는 "요청 하나를 전달했다"보다
"어떤 경로로 전달했고, 동시에 몇 개를 처리했고, cache는 언제 개입했는가"를 설명하는 프로젝트였다.

특히 다음이 남았다.

- 네트워크와 동시성은 분리된 주제가 아니다
- cache 정책은 lock 정책과 함께 설명해야 한다
- local harness가 있으면 공개 저장소도 검증 가능성이 높아진다

## 저장소 차원에서 잘한 결정

- local origin server를 함께 두었다
- 기능, 동시성, cache hit를 각각 보이게 테스트를 나눴다
- 공개 문서를 request path와 cache path 중심으로 정리했다
