# Performance Lab — 지식 인덱스

## 캐시 구조

### 파라미터
- **s** (set index bits): 세트 수 = $2^s$
- **E** (associativity): 세트 당 라인 수. E=1이면 직접 매핑(direct-mapped)
- **b** (block offset bits): 캐시 라인 크기 = $2^b$ 바이트
- 캐시 용량 = $2^s \times E \times 2^b$ 바이트

### 주소 분해
```
| tag (나머지 비트) | set index (s 비트) | block offset (b 비트) |
```
- `set_index = (address >> b) & ((1 << s) - 1)`
- `tag = address >> (s + b)`

### LRU 교체 정책
- 가장 오래 접근하지 않은 라인을 evict
- 구현: 글로벌 클럭 카운터 + 라인별 `lru` 타임스탬프
- 클럭은 1부터 시작 (0은 초기화되지 않은 라인과 구분)

## 캐시 접근 유형

| 유형 | 설명 | 시뮬레이터 동작 |
|---|---|---|
| I | instruction fetch | 무시 |
| L | data load | cache_access 1회 |
| S | data store | cache_access 1회 |
| M | data modify (load+store) | cache_access 2회 |

## 공간/시간 지역성

### 공간 지역성 (Spatial Locality)
- 인접한 주소가 같은 캐시 라인에 들어감
- 행 순서로 배열을 순회하면 공간 지역성을 활용
- 열 순서 순회는 공간 지역성을 파괴

### 시간 지역성 (Temporal Locality)
- 최근에 접근한 데이터가 아직 캐시에 있음
- 블로킹은 작업 세트를 캐시 크기 안에 유지해서 시간 지역성을 높임

## 행렬 전치 최적화 전략

### 블로킹 (Tiling)
- 매트릭스를 B×B 블록으로 분할해서 블록 단위로 처리
- 블록 크기는 캐시 용량과 매트릭스 크기에 따라 결정

### 전략 요약

| 크기 | 전략 | 블록 크기 | 핵심 기법 | 미스 수 | 임계값 |
|---|---|---|---|---|---|
| 32×32 | 블로킹 + 대각선 지연 쓰기 | 8×8 | A[i][i]와 B[i][i]의 세트 충돌 회피 | 284 | <300 |
| 64×64 | 사분면 분할 + B 임시 저장 | 8×8(4×4 사분면) | 4행 주기 conflict 회피 | 1,176 | <1,300 |
| 61×67 | 일반 블로킹 + 경계 가드 | 16×16 | 비정방 행렬 경계 처리 | 1,989 | <2,000 |

### 대각선 지연 쓰기 패턴
- 문제: 대각선 블록에서 A[i][j]와 B[j][i]가 같은 캐시 세트에 매핑 (i==j일 때)
- 해결: 대각선 원소를 변수에 저장, 행의 나머지를 먼저 처리, 마지막에 대각선 쓰기
- 적용: 직접 매핑 캐시에서 같은 세트로 매핑되는 두 배열 원소의 충돌 회피

### 64×64 사분면 전략
```
1단계: A 상단 4행 전체 → B 좌측 4열 + B 임시 영역
2단계: A 하단 좌측 4열 ↔ B 임시 영역 교환
3단계: A 하단 우측 4열 → B 우측 4열
```
- 핵심: B 행렬의 "아직 쓰이지 않은" 영역을 임시 버퍼로 활용
- 이유: 4행 주기로 같은 세트에 매핑되므로, 상단/하단 동시 접근 시 conflict

## 캐시 시뮬레이션 파라미터

### 공식 환경 (s=5, E=1, b=5)
- 32 세트, 직접 매핑, 32바이트 라인
- 캐시 용량: 32 × 1 × 32 = 1,024 바이트
- `int`(4바이트) 기준: 라인당 8개 원소
- 32×32 매트릭스: 8행마다 같은 세트 (32행 × 4바이트 × 32열 = 4,096바이트 = 캐시 4주기)
- 64×64 매트릭스: 4행마다 같은 세트 (64열 × 4바이트 = 256바이트 = 캐시 1/4주기)

### 매트릭스 주소 배치
- A: 주소 0부터
- B: 주소 1MB(`1 << 20`)부터
- 오프셋이 중요: A와 B의 상대 위치가 충돌 패턴을 결정
