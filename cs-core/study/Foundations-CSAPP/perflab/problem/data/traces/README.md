# Performance Lab sample trace

## 이 디렉터리가 가르치는 것

이 디렉터리는 공개 가능한 sample trace가 어떤 역할을 하는지 설명합니다.

## 누구를 위한 문서인가

- `study.trace`가 왜 들어 있는지 빠르게 알고 싶은 사람
- 공식 trace 대신 self-owned 자산을 두는 예시가 필요한 사람

## 먼저 읽을 곳

1. [`../../README.md`](../../README.md)
2. [`study.trace`](study.trace)

## 디렉터리 구조

```text
traces/
  README.md
  study.trace
```

## 검증 방법

- 이 trace는 [`../../../../c/README.md`](../../../../c/README.md)와 [`../../../../cpp/README.md`](../../../../cpp/README.md)의 cache simulator 테스트에서 사용합니다.

## 스포일러 경계

- sample trace는 self-authored 자산입니다.
- 공식 course trace 자산을 대체하거나 복제하지 않습니다.

## 포트폴리오로 확장하는 힌트

- 공개 가능한 테스트 자산을 따로 두면 과제 공개 정책을 설명하기 쉬워집니다.
