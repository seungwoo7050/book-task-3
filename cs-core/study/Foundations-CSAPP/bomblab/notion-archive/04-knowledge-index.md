# Bomb Lab — 핵심 개념 인덱스

## 이 문서의 목적

Bomb Lab을 통해 체화한 리버스 엔지니어링 개념과 어셈블리 패턴을 정리한 인덱스다. 나중에 관련 주제를 다시 볼 때 빠르게 참조하기 위한 용도다.

---

## x86-64 호출 규약 (System V AMD64 ABI)

**등장**: 모든 페이즈의 함수 진입부

인자 전달 순서: `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`, `%r9`. 반환 값: `%rax`. 이 규약을 모르면 "이 함수가 무엇을 인자로 받는지"를 디스어셈블리에서 읽을 수 없다.

---

## gdb 핵심 명령어

**등장**: 전체 리버스 엔지니어링 워크플로우

| 명령어 | 용도 |
|---|---|
| `break explode_bomb` | 안전 브레이크 |
| `break phase_N` | 페이즈 함수 진입 중단 |
| `disas phase_N` | 페이즈 함수 디스어셈블리 |
| `stepi` / `nexti` | 명령어 단위 실행/스킵 |
| `info registers` | 현재 레지스터 상태 |
| `x/s 주소` | 메모리에서 문자열 읽기 |
| `x/6d $rsp` | 스택에서 정수 6개 읽기 |
| `print/x $rax` | 레지스터 값 16진수 출력 |

## 어셈블리 제어 흐름 패턴

**등장**: Phase 2~6

### 루프 인식
- 카운터 레지스터 초기화 → 조건 비교 → 조건 점프 → 카운터 증가
- `cmp`/`jl` 또는 `cmp`/`jne` 쌍이 루프 경계를 형성

### 점프 테이블 (switch)
- `jmpq *주소(,%rax,8)` 형태의 간접 점프
- 메모리에서 점프 테이블 주소를 읽어 각 case의 코드 위치 확인

### 재귀 호출
- 함수가 자기 자신을 `callq`로 호출
- 반환 값이 `add`, `shl` 등으로 조합됨

### 포인터 체이닝
- `mov (%rax), %rax` — 연결 리스트 순회
- 데이터 섹션에서 노드 구조체의 값과 next 포인터 읽기

## 페이즈 패밀리 분류

| 패밀리 | 핵심 구조 | 인식 신호 | 미니 봄 구현 |
|---|---|---|---|
| 문자열 비교 | `strcmp` 호출 | `callq strings_not_equal` | `bomb_phase_1` |
| 수열 검증 | 루프 + 이전값 참조 | 카운터 루프, `imul` 또는 `add` | `bomb_phase_2` |
| 스위치 | 점프 테이블 | `jmpq *주소(,%rax,8)` | `bomb_phase_3` |
| 재귀 경로 | 재귀 호출 + 반환값 인코딩 | `callq func4`, `shl`/`add` | `bomb_phase_4` |
| 니블 룩업 | 비트 마스킹 + 테이블 인덱싱 | `and $0xf, %eax` | `bomb_phase_5` |
| 리스트 재정렬 | 포인터 재연결 + 순서 검증 | `mov (%rax), %rax` | `bomb_phase_6` |
| BST 경로 | 트리 순회 + 경로 인코딩 | `callq fun7`, 재귀 | `bomb_secret_phase` |

## 도구 참조

| 도구 | 역할 | 비고 |
|---|---|---|
| `gdb` | 디버거 | `break`, `stepi`, `x/s`, `info reg` |
| `objdump -d` | 디스어셈블리 | 전체 바이너리 → 텍스트 |
| `strings` | 문자열 추출 | 리터럴 문자열 탐색 |
| `nm` | 심볼 테이블 | 함수 이름과 주소 |
| Docker | 크로스 플랫폼 실행 | 공식 봄은 Linux/amd64 전용 |
