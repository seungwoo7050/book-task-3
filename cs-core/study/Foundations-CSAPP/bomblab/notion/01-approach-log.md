# 01. 접근 기록

## 실제로 택한 접근

이 프로젝트는 처음부터 두 줄기로 나눴다.

1. 공식 self-study bomb는 workflow와 검증 경계만 유지
2. 공개 저장소의 실코드는 companion mini-bomb으로 다시 작성

## phase를 읽을 때 따랐던 순서

1. 입력 형태부터 확인
2. 제어 흐름 패턴 분류
3. 의사코드 수준으로 축약
4. companion 구현으로 같은 개념군을 다시 표현

이 순서를 지키면 raw disassembly에 매몰되지 않고, 공개 문서도 답안집처럼 보이지 않는다.

## 문서화에서 의식한 것

- phase별 정답보다 phase별 패턴을 남긴다
- 도구 사용 순서를 남기되, 외부 자산 dump는 늘리지 않는다
- companion 정답은 companion 검증용이라는 점을 명확히 쓴다
