# 02. 디버그 기록

## 실제로 다시 보게 된 포인트

### 1. `explode_bomb` 브레이크포인트를 먼저 안 건 문제

처음에는 phase를 추측하다가 폭발을 반복했다.
이 프로젝트는 답보다 먼저 디버깅 환경을 고정하는 편이 훨씬 효율적이었다.

### 2. phase 4 재귀 반환값 해석

재귀 함수의 "경로 인코딩"을 단순 반환값 비교로만 보면 금방 틀린다.
중간 분기 구조를 적어 놓고 다시 보는 편이 낫다.

### 3. phase 6의 `7 - x` 변환

연결 리스트 phase는 재연결보다 앞선 인덱스 변환을 놓치면 전체 가설이 무너진다.
pointer chase보다 입력 전처리를 먼저 보는 습관이 필요했다.

### 4. 공개 범위 판단

분석이 잘 될수록 공개 문서에 더 많이 싣고 싶어진다.
하지만 특정 외부 bomb의 raw answer나 긴 dump는 남기지 않는 쪽이 맞았다.
