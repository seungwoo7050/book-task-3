# Bomb Lab Docs

## Purpose

This folder keeps the public explanation for the migrated Bomb Lab project.

The goal is to show:

- how to approach reverse-engineering safely
- what each phase category is trying to teach
- what can and cannot be published in a public study repository

It should not turn the repository into an answer sheet for a specific official bomb instance.

## Document Map

- [`concepts/reverse-engineering-workflow.md`](concepts/reverse-engineering-workflow.md): the default debugging loop for an unknown bomb
- [`concepts/phase-patterns.md`](concepts/phase-patterns.md): the recurring phase families and what to recognize
- [`references/publication-policy.md`](references/publication-policy.md): what public docs may reveal and what stays local
- [`references/verification.md`](references/verification.md): current commands and results for both the public self-study bomb and the companion tracks

## Disclosure Policy

- explain methodology
- explain phase families
- explain the companion implementation contract
- do not publish official raw answers for a specific course bomb
- do not commit the official bomb binary

Longer process notes, failure logs, and annotated references live in `../notion/`.
