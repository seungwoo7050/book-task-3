# 02. 디버그 기록

## 실제로 자주 걸렸던 함정

### 1. `isTmax`에서 `-1`을 같이 잡는 문제

`x + 1`과 `~x`의 관계만 보면 `Tmax`와 `-1`이 비슷하게 보인다.
그래서 `-1`을 분리하는 조건을 따로 넣지 않으면 금방 오답이 된다.

### 2. `isLessOrEqual`에서 overflow를 무시하는 문제

`x - y` 부호만 보면 될 것 같지만, 서로 다른 부호일 때는 overflow 때문에 해석이 바뀐다.
그래서 같은 부호 비교와 다른 부호 비교를 분리해야 했다.

### 3. `floatFloat2Int`에서 범위 초과 처리

부호와 exponent 처리를 뒤섞으면 `0x80000000u`를 반환해야 할 케이스를 놓치기 쉽다.
정수 변환은 "먼저 범위 판정, 나중에 shift" 순서가 더 안전했다.

### 4. 공식 검증에서 timeout처럼 보이는 문제

Apple Silicon 호스트에서 amd64 handout을 Docker로 돌릴 때 `btest` 기본 제한시간 10초가 빡빡했다.
이 저장소는 `-T 20`으로 늘려 공식 검증을 안정화했다.
