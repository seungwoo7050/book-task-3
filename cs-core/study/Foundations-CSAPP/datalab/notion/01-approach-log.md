# 01. 접근 기록

## 실제로 택한 순서

처음부터 함수 13개를 같은 방식으로 풀지 않았다.
정수 퍼즐과 부동소수점 퍼즐을 완전히 다른 과제로 취급했다.

1. 정수 퍼즐에서 반복 패턴 정리
2. `bitXor`, `tmin`, `negate`처럼 짧은 함수로 워밍업
3. `isLessOrEqual`, `howManyBits` 같은 분기형 사고 문제 해결
4. 부동소수점 세 함수는 sign/exponent/fraction 분류표부터 작성
5. C 구현을 먼저 고정한 뒤 C++ 구현으로 동작을 맞춤

## 왜 이 순서를 택했는가

- 정수 퍼즐은 패턴 재사용이 많아서 먼저 묶는 편이 유리했다
- 부동소수점은 분류 실수가 치명적이라 별도 문맥으로 분리하는 편이 좋았다
- 공식 handout 검증은 C 기준이므로, C를 먼저 "진실 소스"로 잡는 것이 자연스러웠다
