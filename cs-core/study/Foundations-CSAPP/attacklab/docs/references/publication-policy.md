# Publication Policy

## Allowed In Public

- official problem contract without the official binaries
- the study-owned companion verifier implementations and their sample payloads
- verified payload files for the public `target1` self-study handout
- a self-written gadget-farm boundary without any target-specific cookie file
- workflow notes about stack layout, code injection, defenses, and ROP

## Not Allowed In Public

- supplied official target binaries
- target-specific cookie files copied from a course target
- raw exploit strings for a supplied private or course-issued `ctarget` or `rtarget`
- writeups that are only operational exploit recipes for third-party targets

## Local-Only Follow-Up

Private or course-issued Attack Lab assets can still be used for personal verification under
`problem/`, but they should remain outside committed Git history.
