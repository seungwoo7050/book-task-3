# 01. 접근 기록

## 실제로 택한 접근

phase를 "어떤 함수를 어떻게 뚫는가"보다
"어떤 바이트 구조를 만족해야 하는가"로 다시 정리했다.

- phase 1~3: code injection 계열
- phase 4~5: ROP와 상대 주소 계산 계열

그 위에 companion verifier를 얹어, 공개 저장소에서도 구조 검증이 가능하게 만들었다.

## 이 방식이 좋았던 이유

- 외부 target 바이너리를 재배포하지 않아도 된다
- 각 phase의 핵심 불변식을 문장으로 설명할 수 있다
- 공개 범위와 학습 가치를 동시에 잡을 수 있다

## 문서화에서 남긴 원칙

- 공개 문서는 공격 모델과 방어 차이를 설명한다
- raw exploit string은 최소한으로만 다룬다
- 비공개 target 정보는 문서에서 완전히 분리한다
