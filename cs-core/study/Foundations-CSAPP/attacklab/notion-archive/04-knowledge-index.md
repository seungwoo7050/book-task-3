# Attack Lab — 핵심 개념 인덱스

## 이 문서의 목적

Attack Lab을 통해 체화한 보안 개념과 공격 기법을 정리한 인덱스다.

---

## 버퍼 오버플로우 (Buffer Overflow)

**등장**: 모든 페이즈의 기반

`gets()` 함수가 입력 길이를 제한하지 않아서 스택 버퍼를 넘어 데이터를 쓸 수 있는 취약점. 리턴 주소를 덮어쓰면 함수 반환 시 원하는 주소로 점프할 수 있다.

**참고**: CS:APP 3/e 3.10.3절 — 버퍼 오버플로우

---

## 코드 인젝션 (Code Injection)

**등장**: Phase 2, Phase 3

스택이 실행 가능한 환경에서 버퍼에 기계어 코드를 주입하고, 리턴 주소를 버퍼 시작 주소로 덮어써서 주입된 코드를 실행하는 공격.

핵심 제약: 셸코드의 바이트 인코딩이 정확해야 하고, 버퍼 시작 주소를 정확히 알아야 한다.

---

## Return-Oriented Programming (ROP)

**등장**: Phase 4, Phase 5

스택이 실행 불가능한 환경에서, 기존 코드의 `ret`으로 끝나는 짧은 시퀀스(gadget)를 리턴 주소 체인으로 연결하는 공격.

각 gadget은 1~2개의 명령어를 실행한 뒤 `ret`으로 다음 gadget으로 넘어간다. 스택에 놓인 주소 배열이 곧 "프로그램"이 된다.

**참고**: CS:APP 3/e 3.10.5절 — ROP 공격 지원

---

## W^X (Write XOR Execute)

**등장**: Phase 4~5의 전환점

메모리 페이지가 쓰기 가능(W)이면 실행 불가(^X), 실행 가능이면 쓰기 불가라는 정책. 스택에 쓴 데이터를 코드로 실행할 수 없게 만든다.

Phase 1~3의 코드 인젝션이 Phase 4~5에서 불가능해지는 이유.

---

## ASLR (Address Space Layout Randomization)

**등장**: Phase 5

스택, 힙, 라이브러리 등의 메모리 영역 시작 주소를 실행마다 무작위로 변경하는 방어 기술. 절대 주소를 하드코딩할 수 없게 만든다.

Phase 5에서 런타임 `%rsp` 캡처 + 오프셋 계산이 필요한 이유.

---

## 스택 프레임 레이아웃

**등장**: 모든 페이즈

```
[상위 주소]
  - 호출자의 리턴 주소
  - 로컬 변수들
  - 버퍼 (40 bytes)
[하위 주소 ← %rsp]
```

버퍼의 끝에서 리턴 주소까지의 거리(이 경우 40바이트)가 오버플로우 페이로드의 패딩 크기를 결정한다.

---

## 리틀 엔디안 바이트 순서

**등장**: 모든 페이즈의 hex 페이로드 작성

x86-64는 리틀 엔디안이므로, 주소 `0x4017c0`은 메모리에 `c0 17 40 00 00 00 00 00` 순서로 저장된다. 모든 hex 페이로드에서 각 8바이트 주소를 리틀 엔디안으로 변환해야 한다.

---

## 페이즈별 페이로드 구조

| 페이즈 | 유형 | 페이로드 크기 | 핵심 구조 |
|---|---|---|---|
| 1 | 리턴 주소 덮어쓰기 | 48 bytes | 패딩(40) + touch1 주소(8) |
| 2 | 코드 인젝션 | 48 bytes | 셸코드(13) + 패딩(27) + 버퍼 주소(8) |
| 3 | 코드 인젝션 + 문자열 | 57 bytes | 셸코드(13) + 패딩(27) + 버퍼 주소(8) + 쿠키 문자열(9) |
| 4 | ROP 체인 | 72 bytes | 패딩(40) + gadget chain(32) |
| 5 | ROP + 상대 주소 | 105 bytes | 패딩(40) + gadget chain(56) + 쿠키 문자열(9) |

## 도구 참조

| 도구 | 역할 |
|---|---|
| `hex2raw` | hex 텍스트 → 바이너리 변환 |
| `ctarget` | 코드 인젝션 대상 (스택 실행 가능) |
| `rtarget` | ROP 대상 (W^X, ASLR) |
| `objdump -d` | 디스어셈블리 (gadget farm 분석) |
| `gdb` | 스택 레이아웃, 레지스터 확인 |
| Docker | 공식 타겟 실행 (linux/amd64) |
