# I-event-integration-lab 노트 안내

이 노트는 I 랩을 “저장과 전달은 같은 성공이 아니다”라는 질문으로 다시 읽게 만드는 보조 문서다.

## 추천 읽기 순서

1. `00-problem-framing.md`: 왜 outbox와 consumer가 필요한지 본다.
2. `01-approach-log.md`: 왜 동기 호출 대신 relay + stream 구조를 선택했는지 본다.
3. `05-development-timeline.md`: 실제 재현 명령과 검증 순서를 확인한다.
4. `02-debug-log.md`: copied MSA 구조를 event 랩으로 다듬는 과정에서 생긴 문제를 본다.
5. `03-retrospective.md`, `04-knowledge-index.md`: 다음 gateway 랩으로 이어지는 질문과 재사용 개념을 정리한다.
