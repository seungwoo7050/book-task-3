# 회고

## 이번 랩에서 얻은 것

로컬 인증을 "로그인 성공"이 아니라 "계정 수명주기 전체"로 설명할 수 있게 된 점이 가장 크다. 이메일 검증, 재설정, refresh rotation, CSRF를 한 흐름으로 묶어 두었기 때문에 이후 랩의 보안 주제를 붙이기 쉽다.

## 아직 약한 부분

- 실 메일 인프라와 전송 실패 복구는 다루지 않는다.
- 세션 감사 로그나 디바이스 단위 세션 관리도 아직 없다.

## 다시 보면 좋을 주제

- refresh token family invalidation
- 이메일 전송 실패 재시도
- 관리자용 계정 복구 흐름

## 포트폴리오 확장 아이디어

- 로그인 감사 로그와 디바이스 세션 화면을 추가한다.
- 이메일 검증 실패와 재발송 UX까지 정리해 개인 프로젝트로 확장한다.
