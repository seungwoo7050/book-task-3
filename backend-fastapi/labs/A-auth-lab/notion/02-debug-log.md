# 디버그 로그

## 실패 사례

초기 상태에서는 `make run`이나 Compose 부팅 직후에 인증 로직을 보기 전에 "table users does not exist" 같은 오류로 막히는 일이 있었다.

## 원인

학습 저장소에서는 README에 적힌 첫 명령이 바로 재현되어야 하는데, 수동 schema 준비 단계를 남겨 두면 독자는 인증 흐름보다 환경 트러블슈팅부터 하게 된다.

## 수정

- 앱 시작 시 필요한 스키마를 자동으로 준비하는 bootstrap 경로를 유지했다.
- 테스트 환경도 매번 깨끗한 스키마에서 시작하도록 정리했다.

## 검증 근거

- 문서 기준으로는 `make smoke`와 `docker compose up --build`가 수동 migration 단계를 요구하지 않는 흐름으로 설명된다.
- 이 문서 개편에서는 명령을 다시 실행하지 않았고, 마지막 기록된 실행 결과는 [../../../docs/verification-report.md](../../../docs/verification-report.md)에 있다.
이 실패 사례가 중요한 이유는, 학습 저장소에서는 인증 로직의 정교함보다 "첫 실행 경험이 매끄러운가"가 먼저 독자의 신뢰를 만든다는 점을 보여 주기 때문이다.
