# D-data-api-lab Notion Documents

각 노트는 특정 사용 사례에 대응한다. 아래 안내를 따라 읽기를 선택할 수 있다.

전체 흐름 안내:
1. `00-problem-framing.md` – 프로젝트 목표와 기대 결과 확인.
2. `01-approach-log.md` – 설계 방향과 그 이유.
3. `02-debug-log.md` – 개발 중 겪은 문제와 수정 팁.
4. `03-retrospective.md` – 회고 및 개선 포인트.
5. `04-knowledge-index.md` – 핵심 개념과 참고 문헌 정리.
6. `05-development-timeline.md` – 소스 외 정보(명령, 도구).

특정 목적별:
- 문제 정의가 필요하면 `00`을 본다.
- 설계 논의가 필요하면 `01`.
- 버그 사례가 필요하면 `02`.
- 면접/학습 메모는 `03`·`04`.
- 개발 환경과 단계는 `05`.
