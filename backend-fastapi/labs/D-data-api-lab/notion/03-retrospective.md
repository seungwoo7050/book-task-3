# 회고

## 이번 랩에서 얻은 것

CRUD 예제를 데이터 규칙의 집합으로 바라보는 감각이 생겼다. 특히 soft delete와 optimistic locking을 분리해서 설명할 수 있게 된 점이 크다.

## 아직 약한 부분

- page-based pagination은 실서비스형 대용량 목록에서 한계가 있다.
- 인증과 인가가 빠져 있어 실제 협업 제품 맥락은 일부러 단순화되어 있다.

## 다시 보면 좋을 주제

- cursor pagination
- 검색 인덱스와 정렬 전략
- 감사 로그와 변경 이력

## 포트폴리오 확장 아이디어

- 자신의 도메인 엔터티로 같은 문제를 다시 풀어 본다.
- 충돌 해결 UX나 diff 표시까지 확장하면 더 좋은 포트폴리오 소재가 된다.
