# K-distributed-ops-lab 노트 안내

이 노트는 K 랩을 “다중 서비스 구조도 운영 질문에 답할 수 있어야 한다”는 관점으로 읽게 만드는 보조 문서다.

## 추천 읽기 순서

1. `00-problem-framing.md`: 왜 운영성이 별도 랩으로 필요해졌는지 본다.
2. `01-approach-log.md`: 왜 최소 관측 기준만 남겼는지 본다.
3. `05-development-timeline.md`: 실제 검증과 문서 확인 순서를 따라간다.
4. `02-debug-log.md`: 운영성 랩에서 실제로 어떤 부팅/정리 문제가 생겼는지 본다.
5. `03-retrospective.md`, `04-knowledge-index.md`: capstone v2로 넘길 질문과 재사용 개념을 정리한다.
