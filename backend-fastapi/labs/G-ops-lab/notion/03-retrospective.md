# 회고

## 이번 랩에서 얻은 것

운영성은 "인프라 담당자만의 일"이 아니라 백엔드 설계의 일부라는 점이 분명해졌다. health/readiness 구분과 설정 캐시 격리 문제는 다른 랩에도 바로 재사용된다.

## 아직 약한 부분

- metrics와 logging은 최소 구현이라 깊이가 제한적이다.
- 실제 클라우드 운영 경험을 증명하지는 못한다.

## 다시 보면 좋을 주제

- Prometheus / Grafana 통합
- trace와 correlation id
- IaC와 운영 runbook

## 포트폴리오 확장 아이디어

- 운영 대시보드 스크린샷이나 예시 alert 정책까지 붙여 설명을 강화한다.
- 문서 수준 가정과 실제 검증 완료 범위를 표로 정리해 신뢰도를 높인다.
