# G-ops-lab Notion Documents

운영성에 초점을 맞춘 이 랩의 문서들을 어떻게 읽을지 안내한다.

추천 순서 (전체 맥락 파악):
1. `00-problem-framing.md` – 랩의 핵심 목표와 성공 기준 파악.
2. `01-approach-log.md` – 선택한 접근 방식과 버린 대안.
3. `02-debug-log.md` – health 체크, 라이프사이클 등에서 만난 문제.
4. `03-retrospective.md` – 회고와 남은 과제.
5. `04-knowledge-index.md` – 운영성 관련 용어와 개념 요약.
6. `05-development-timeline.md` – Docker/CI/health probe 등 실제 개발 로그.

목적별 사용:
- 운영 코드를 빠르게 훑으려면 `00`·`01`.
- healthcheck 설정이나 lru_cache 문제처럼 구체적인 버그는 `02`.
- 면접 요약 또는 개념 정리는 `04`.
- 개발 히스토리와 명령어는 `05`.
