# 회고

## 이번 랩에서 얻은 것

실시간 기능을 "연결 상태 관리"와 "메시지 전달"이라는 두 문제로 나눠 볼 수 있게 되었다. presence TTL과 fan-out을 별도 개념으로 설명할 수 있게 된 점이 특히 좋다.

## 아직 약한 부분

- 인메모리 구조라 수평 확장 설명은 문서 수준에 머문다.
- 토큰 검증도 학습용 최소 모델로 단순화되어 있다.

## 다시 보면 좋을 주제

- Redis pub/sub 기반 fan-out
- delivery ack와 replay
- room / channel 모델

## 포트폴리오 확장 아이디어

- 사용자별 unread 상태와 reconnect replay를 추가한다.
- 실시간 기능을 협업 도메인이나 알림 센터와 직접 연결해 본다.
