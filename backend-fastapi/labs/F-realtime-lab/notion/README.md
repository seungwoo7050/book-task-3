# F-realtime-lab Notion Documents

이 폴더에는 실시간 통신과 presence 추적에 관한 노트가 담겨있다.
각 문서의 용도와 읽기 순서를 아래에 정리했다.

읽기 순서 (전체 이해):
1. `00-problem-framing.md` – 랩 목표와 기대 상황.
2. `01-approach-log.md` – 설계 고민과 채택한 방향.
3. `02-debug-log.md` – 개발 중 마주친 이슈와 해결책.
4. `03-retrospective.md` – 완료 후 회고 및 개선점.
5. `04-knowledge-index.md` – 재사용 가능한 개념 및 용어집.
6. `05-development-timeline.md` – 개발 기록, 명령어, 환경 세팅.

목적에 따른 접근:
- 컨셉을 빠르게 이해하고 싶다면 `00`과 `01`.
- 디버깅 레퍼런스가 필요하면 `02`.
- 면접 대비 요약은 `03`과 `04`.
- 실제 설정과 테스트 프로세스는 `05`.
