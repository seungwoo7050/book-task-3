# H-service-boundary-lab 노트 안내

이 노트는 H 랩을 “서비스 분리의 첫 기준은 무엇인가”라는 질문으로 다시 읽게 만들기 위한 보조 문서다.

## 추천 읽기 순서

1. `00-problem-framing.md`: 왜 H 랩이 MSA 트랙의 첫 단계인지 본다.
2. `01-approach-log.md`: 왜 claim 계약과 DB ownership을 먼저 고정했는지 본다.
3. `05-development-timeline.md`: 실제 재현 명령과 검증 순서를 확인한다.
4. `02-debug-log.md`: copied 구조를 랩으로 바꾸는 과정에서 무엇이 깨졌는지 본다.
5. `03-retrospective.md`, `04-knowledge-index.md`: 다음 랩으로 이어질 질문과 재사용 개념을 정리한다.
