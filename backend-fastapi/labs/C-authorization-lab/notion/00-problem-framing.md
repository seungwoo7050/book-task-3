# 문제 프레이밍

## 학습 목표

인증과 별개로, 워크스페이스 기반 서비스에서 "누가 무엇을 할 수 있는가"를 독립적인 설계 문제로 다루는 것이 목표다.

## 왜 중요한가

- 인증이 끝났다고 해서 권한 문제가 사라지지 않는다.
- RBAC와 소유권 규칙을 코드로 분리해 두어야 capstone에서 인증 스택과 자연스럽게 결합할 수 있다.

## 선수 지식

- 사용자와 리소스 관계를 데이터 모델로 표현하는 기본기
- 역할 기반 접근 제어 개념
- FastAPI 서비스 계층 구조의 기본

## 성공 기준

- 워크스페이스 생성, 초대, 역할 변경, 문서 접근 규칙이 일관되게 설명 가능해야 한다.
- viewer / member / admin / owner의 하한선이 코드와 테스트에 반영되어야 한다.
- 인가 규칙이 인증 세부 구현과 분리되어야 한다.

## 제외 범위

- JWT나 cookie 기반 실제 인증
- 정책 엔진이나 조직 단위 멀티테넌시 전부
- 고급 선언형 인가 프레임워크 실험

이 랩의 가치는 인증 구현 없이도 인가 규칙만 따로 설명하고 테스트할 수 있다는 점에 있다.
