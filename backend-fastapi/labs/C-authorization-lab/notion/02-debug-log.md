# 디버그 로그

## 실패 사례

초기 구현에서는 viewer가 문서를 생성할 수 있는 버그와, 초대 이메일의 대소문자가 맞지 않아 수락이 실패하는 문제가 있었다.

## 원인

- 역할 하한선을 잘못 두면 정상 경로 테스트만으로는 권한 누수를 발견하기 어렵다.
- 초대와 사용자 생성 경로가 이메일 정규화를 같은 규칙으로 처리하지 않으면 불일치가 생긴다.

## 수정

- 문서 생성 권한의 최소 역할을 `member` 이상으로 고정했다.
- 초대와 사용자 저장 시 이메일을 같은 방식으로 소문자 정규화하도록 맞췄다.

## 검증 근거

- forbidden-path 테스트가 다시 403을 검증한다.
- 이 실패는 권한 랩에서 허용 경로보다 차단 경로 테스트가 더 중요하다는 점을 잘 보여 준다.

최종 검증 시점은 [../../../docs/verification-report.md](../../../docs/verification-report.md)를 따른다.
