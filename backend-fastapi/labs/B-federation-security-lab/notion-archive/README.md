# B-federation-security-lab Notion Documents

이 폴더는 기술 에세이 스타일 문서들이 모여있다. 각 문서는 특정 독자 질문에
답하도록 설계되어 있다.

권장 읽기 순서:
1. `00-problem-framing.md` – 프로젝트 목표와 제약을 전체적으로 파악.
2. `01-approach-log.md` – 설계 결정과 고려했던 대안을 이해.
3. `02-debug-log.md` – 구현 중 발생한 문제와 해결책을 확인.
4. `03-retrospective.md` – 끝난 뒤에 얻은 교훈과 개선할 점을 살펴본다.
5. `04-knowledge-index.md` – 재사용 가능한 개념/용어/참고 링크 정리.
6. `05-development-timeline.md` – 개발 및 빌드/테스트 과정의 세부사항.

목적별 안내:
- "무엇을 만들었나"가 궁금하면 `00`과 `01`을 읽는다.
- 디버깅 로그를 보고 싶다면 `02`.
- 학습 요약이나 면접 메모는 `03`과 `04`.
- 개발 환경과 커맨드 히스토리는 `05`가 답한다.
