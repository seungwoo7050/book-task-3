# backend-fastapi 문서 지도

이 폴더는 루트 공통 문서를 모아 둔 곳입니다. 개별 랩의 구현 세부보다 저장소 전체의 읽는 순서, 문서 규칙, 검증 기록을 먼저 설명합니다.

## 먼저 읽을 문서

1. [repo-standards.md](repo-standards.md): 이 저장소에서 `README`, `problem`, `fastapi`, `docs`, `notion`이 각각 어떤 역할을 맡는지 정의합니다.
2. [labs-curriculum.md](labs-curriculum.md): 필수 트랙과 심화 트랙의 학습 순서, 선수 지식, 랩 간 연결 이유를 정리합니다.
3. [verification-report.md](verification-report.md): 마지막 실제 검증 실행 시점과 범위, 확인된 사실과 아직 증명하지 않은 범위를 기록합니다.

## 이 폴더가 답하는 질문

- 이 레포를 어떤 순서로 읽어야 하는가
- 각 프로젝트 README가 무엇을 먼저 드러내야 하는가
- 문서에 적힌 검증이 실제 실행된 것인지, 아직 가정인지 어떻게 구분하는가
