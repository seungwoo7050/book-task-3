# Request Lifecycle

이 프로젝트는 요청 상태를 하나의 `loading` 불린으로 다루지 않는다. 대신 아래처럼 나눈다.

- `loading`
  - 새 요청이 진행 중인 상태
- `success`
  - 결과와 detail이 정상적으로 로드된 상태
- `empty`
  - 요청은 성공했지만 현재 query에 맞는 결과가 없는 상태
- `error`
  - 실패했지만 retry가 가능한 상태

이 분리는 제품 설명에도 중요하다. 사용자는 "왜 비어 있는가"와 "왜 실패했는가"를 다르게 받아들인다.

또한 list와 detail의 상태를 분리해서, 목록은 성공했지만 detail만 다시 로드되는 상황을 설명 가능하게 만들었다.
