# Notion Notes — Ops Triage Console

`01-ops-triage-console` 프로젝트의 노트들이 이 폴더에 있습니다. 각 파일의 목적은 다음과 같습니다.

1. **00-problem-framing.md** – 이 프로젝트를 선택한 이유와 달성하려는 목표.
2. **01-approach-log.md** – 구현 과정 및 설계 결정 로그.
3. **02-debug-log.md** – 버그 및 디버깅 기록.
4. **03-retrospective.md** – 회고와 배운 점.
5. **04-knowledge-index.md** – 용어, 패턴, 관련 기술 요약.
6. **05-development-timeline.md** – CLI 명령어, 도구 버전, 개발 단계 타임라인.

**활용 가이드**
- 전체 흐름을 파악하려면 00 → 05 순서대로 읽습니다.
- 성능, 낙관적 업데이트, 테스트 전략 등 특정 주제를 찾으려면 approach-log나 debug-log를 참조하세요.
- 면접이나 포트폴리오 설명 준비 시에는 retrospective와 development-timeline이 유용합니다.

이 문서들은 로컬에서만 사용되며, 언제든지 수정 가능합니다.
