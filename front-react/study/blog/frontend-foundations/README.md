# Frontend Foundations

이 트랙은 프레임워크가 도와주기 전에 브라우저 자체가 어떤 구조와 상태를 제공하는지부터 다시 밟아 나간다. 세 프로젝트는 모두 vanilla DOM으로 만들었지만, 관심사는 조금씩 다르다. 첫 번째는 읽히는 구조, 두 번째는 상태 경계, 세 번째는 비동기 요청의 수명주기다.

## 읽는 순서

1. [01 Semantic Layouts And A11y](01-semantic-layouts-and-a11y/00-series-map.md)
   - 예쁜 화면보다 먼저 semantic shell과 keyboard flow를 고정한 기록
2. [02 DOM State And Events](02-dom-state-and-events/00-series-map.md)
   - URL state, local persistence, ephemeral UI state를 어디서 갈라야 하는지 정리한 기록
3. [03 Networked UI Patterns](03-networked-ui-patterns/00-series-map.md)
   - loading, error, retry, abort, stale response를 하나의 화면에서 다룬 기록

앞에서 만든 감각이 뒤 프로젝트의 전제가 되므로, 가능하면 순서대로 읽는 편이 좋다.
