# msglab 회고

## 이 lab의 가치

`msglab`은 코드량보다 효과가 큰 프로젝트다. parser 책임을 먼저 정리해 두면 이후 서버 lab에서 디버깅 범위를 빠르게 줄일 수 있다.

## 아직 남는 한계

- 실제 socket read buffer와의 결합은 보여 주지 않는다.
- reply formatting 전부를 다루지는 않는다.

## 포트폴리오로 옮길 때의 조언

- “입력 정규화와 상태 변경을 분리했다”는 점을 분명히 적는다.
- transcript 테스트가 어떻게 회귀 방지 장치가 되는지 예시를 붙인다.
- 이후 `roomlab`에서 parser가 어떤 식으로 재사용되는지 연결해 준다.
