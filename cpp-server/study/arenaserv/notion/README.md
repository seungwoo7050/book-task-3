# arenaserv Notion 문서 안내

이 폴더에는 `arenaserv` capstone의 개발 기록이 정리되어 있습니다. 목적과 순서는 아래와 같습니다:

1. **00-problem-framing.md** – 게임 서버를 만들기 전 해결할 질문과 동기를 제시합니다.
2. **01-approach-log.md** – ticklab과 EventManager를 결합하는 방법 등 초기 설계를 서술합니다.
3. **02-debug-log.md** – 이벤트 flush, fd 매핑, overflow 테스트 등 버그 기록입니다.
4. **03-retrospective.md** – 프로젝트의 성취와 한계를 되돌아봅니다.
5. **04-knowledge-index.md** – 도출된 지식 항목을 색인 형식으로 정리했습니다.
6. **05-development-timeline.md** – 개발 과정 전체를 날짜와 명령어로 복원했습니다.

이 폴더를 처음 열었을 때, 위 순서대로 읽으면 전체적인 흐름을 따라가기 쉽습니다. 특정 주제만 보고 싶다면 `knowledge-index`나 `debug-log`를 먼저 확인하세요.
