# roomlab 회고

## 이 lab이 남기는 학습 효과

`roomlab`은 작지만 서버다운 문제를 처음 다루게 해 준다. 등록, 상태 전이, cleanup, broadcast가 한 프로젝트 안에서 만난다.

## 아직 약한 부분

- 고급 channel command는 아직 없다.
- 클라이언트 호환성과 capability negotiation은 다루지 않는다.

## 포트폴리오로 옮길 때의 조언

- 상태 전이 표 하나만 잘 만들어도 문서 설명력이 크게 올라간다.
- duplicate nick과 cleanup 사례를 꼭 같이 보여 준다.
- 이후 `ircserv`에서 추가된 command를 비교해 커리큘럼의 계단을 드러내면 좋다.
