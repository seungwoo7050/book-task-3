# roomlab Notion 문서 안내

`roomlab` 프로젝트와 연관된 생각, 결정, 버그, 회고가 이 디렉터리에 모여 있습니다. 읽기 좋은 순서는 다음과 같습니다:

1. **00-problem-framing.md** – 프로젝트 목표와 해결하려는 문제가 무엇인지 소개합니다.
2. **01-approach-log.md** – 구현을 어떻게 설계했는지, 선택의 이유를 설명합니다.
3. **02-debug-log.md** – 개발 중 마주친 버그와 해당 디버깅 과정을 살펴봅니다.
4. **03-retrospective.md** – 실습을 마치고 얻은 교훈과 개선 사항을 되돌아봅니다.
5. **04-knowledge-index.md** – 상태 머신과 IRC 관련 지식을 핵심적으로 정리했습니다.
6. **05-development-timeline.md** – 디렉터리 구조, 명령어, 빌드/테스트 순서를 시간 축으로 기록합니다.

문서를 통해 `eventlab`, `msglab` 등 하위 과제와의 관계도 볼 수 있으니, 전체 흐름을 이해하려면 순서대로 읽어보세요.
