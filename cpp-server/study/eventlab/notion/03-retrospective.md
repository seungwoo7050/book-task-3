# eventlab 회고

## 이 lab이 남기는 학습 효과

`eventlab`은 기능이 작아 보여도 이후 모든 서버 lab의 토대를 만든다. 이 단계를 건너뛰면 나중에 parser 버그와 event loop 버그가 섞여 보이기 쉽다.

## 아직 약한 부분

- 타이머 정확도 자체를 깊게 다루지는 않는다.
- 실제 프로토콜 협상이나 복잡한 backpressure 전략은 아직 없다.

## 다음에 다시 손본다면

- 상태 다이어그램 한 장을 추가해 연결 수명주기를 더 빠르게 설명하게 한다.
- smoke test 로그를 예시로 남겨 학습자가 검증 결과를 더 쉽게 상상하게 한다.

## 포트폴리오로 옮길 때의 조언

- “작지만 분명한 질문”을 먼저 해결한 프로젝트라는 점을 강조한다.
- 최종 제품보다 기초 책임 분리가 왜 중요했는지 설명한다.
- `roomlab`이나 `ircserv`와 연결해, 이 lab이 어떤 기반 역할을 했는지 적는다.
