# Problem

## canonical problem statement

기능 구현을 넘어서 config, health, logging, CI, cache, queue, rate limiting 같은 운영 준비 항목을 학습용 서비스에 붙이는 것이 canonical problem이다.

## 성공 기준

- health/live와 health/ready를 분리할 것
- config loader와 structured logging을 문서화할 것
- 테스트와 Docker/CI 예시를 함께 제공할 것

- 문제 설명
- NestJS 워크스페이스
