# 02-http-and-api-basics — Notion 문서 가이드

## 이 폴더의 목적

이 `notion/` 폴더는 프로젝트를 Notion으로 옮길 때 그대로 사용할 수 있는 문서 세트다.
처음 읽는 사람이 "프레임워크 없이 HTTP 서버를 직접 만든다"는 것이 왜 필요했고 어떤 감각을 남기는지 이해할 수 있도록 작성되어 있다.

## 문서 목록과 읽는 순서

| 순서 | 문서 | 설명 |
|------|------|------|
| 1 | [essay.md](essay.md) | 프로젝트 에세이 — 프레임워크 없이 HTTP를 다뤄본 이유와 경험을 서사적으로 설명한다 |
| 2 | [timeline.md](timeline.md) | 개발 타임라인 — 환경 설정부터 curl 검증까지 전체 개발 과정을 순서대로 기록한다 |

## 누가 언제 읽으면 좋은가

- **프레임워크 뒤에서 무슨 일이 일어나는지 궁금한 사람**: `essay.md`를 읽으면 라우팅, JSON 파싱, 에러 응답의 원형을 이해할 수 있다
- **코드를 재현하며 따라가고 싶은 사람**: `timeline.md`를 따라가면 된다
- **왜 Express나 NestJS 전에 이걸 하는지 알고 싶은 사람**: `essay.md` 서두를 읽으면 된다
