# Problem

## canonical problem statement

CRUD 위에 validation, error handling, logging, response shaping을 공통 파이프라인으로 올리는 것이 canonical problem이다.

## 성공 기준

- 성공 응답 형식과 실패 응답 형식을 일관되게 유지할 것
- Express와 NestJS 각각에서 파이프라인 지점을 명확히 나눌 것
- unit/e2e 테스트로 규약을 다시 확인할 것

## 제공 자료

- Express/NestJS starter code
- 테스트 힌트
- 개념 문서
- 학습 로그
