# Streaming CLI Checklist

로그 파일을 처리하는 CLI는 "작동한다"보다 "실패를 설명한다"가 더 중요하다.

- 입력 경로는 현재 작업 디렉터리 기준으로 해석되므로 README 예제를 그대로 적어 둔다.
- 파일 형식이 한 줄당 하나의 JSON 객체인지 먼저 확정한다.
- 메모리 사용을 줄이려면 `createReadStream`과 `readline`으로 줄 단위 처리한다.
- 환경 변수는 옵션이 많아질수록 숨은 입력이 되므로 README에 기본값과 허용값을 적는다.

이 체크리스트는 이후 ETL 스크립트, 마이그레이션 도구, 운영 로그 분석 도구에도 그대로 재사용할 수 있다.
