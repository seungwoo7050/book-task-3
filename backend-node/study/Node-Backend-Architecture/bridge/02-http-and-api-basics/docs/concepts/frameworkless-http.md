# Frameworkless HTTP Notes

프레임워크 없이 HTTP 서버를 만들면 나중에 Express나 NestJS가 대신해 주는 일을 직접 보게 된다.

- 라우팅은 결국 메서드와 URL을 비교해 분기하는 일이다.
- JSON body 파싱은 자동이 아니라 스트림을 모아 직접 해석해야 한다.
- 응답 코드는 컨트롤러가 아니라 "요청을 어디서 거부했는가"에 따라 달라진다.
- 테스트는 서버 전체를 띄우되 저장소는 메모리 구현으로 두는 것이 가장 단순하다.

이 감각이 있어야 이후 middleware, pipe, guard가 무엇을 추상화하는지 정확히 이해할 수 있다.
