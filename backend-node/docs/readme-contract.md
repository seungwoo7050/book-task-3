# README Contract

이 문서는 이 저장소에서 공개 README가 답해야 할 질문을 고정한다. 목적은 “코드가 많다”가 아니라 “문제, 답, 검증이 1분 안에 보인다”는 상태를 유지하는 것이다.

## 루트 README

- 이 레포가 왜 존재하는가
- 어떤 문제군을 풀고 있는가
- 대표 결과물이 무엇인가
- 전체 카탈로그와 검증 행렬이 어디에 있는가
- 처음 읽는 사람이 어떤 순서로 내려가야 하는가

## 트랙 README

- 이 트랙이 전체 커리큘럼에서 어떤 역할을 하는가
- 프로젝트가 어떤 그룹으로 묶이는가
- 각 프로젝트의 한 줄 문제와 답 형태가 무엇인가
- 대표 검증 명령이 무엇인가

## 프로젝트 README

- 한 줄 문제는 무엇인가
- 성공 기준이 무엇인가
- 내가 만든 답은 무엇인가
- 제공 자료가 어디에 있는가
- 실행/검증 명령이 무엇인가
- 왜 다음 단계로 이어지는가

## problem/README.md

- canonical problem statement가 무엇인가
- 성공 기준은 무엇인가
- 제공된 starter code, fixture, reference가 무엇인가
- 어떤 구현 레인에서 풀게 되는가

## 구현 레인 README

- 이 레인이 담당하는 해결 전략이 무엇인가
- 실제 실행과 검증 명령이 무엇인가
- 현재 디렉터리 구조를 어디서 봐야 하는가
- 다른 레인과 비교할 때 무엇이 다른가

## docs/README.md

- 개념 문서가 무엇을 설명하는가
- 어떤 순서로 읽으면 좋은가
- 구현 세부 대신 오래 유지할 설명은 어디 있는가

## notion/README.md

- 장문 에세이, 타임라인, 디버깅 로그가 무엇을 담는가
- 어떤 독자가 어느 파일부터 읽어야 하는가
- 구현 세부가 바뀌어도 유지할 서사는 무엇인가
