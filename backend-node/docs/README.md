# Repository Docs

이 디렉터리는 저장소 전체에 적용되는 커리큘럼, README 계약, 검증 행렬, 경로 이관 기록을 담는다.

## 읽기 순서

1. [readme-contract.md](readme-contract.md)
2. [project-catalog.md](project-catalog.md)
3. [verification-matrix.md](verification-matrix.md)
4. [curriculum-map.md](curriculum-map.md)
5. [skill-matrix.md](skill-matrix.md)
6. [migration-map.md](migration-map.md)
7. [legacy-audit.md](legacy-audit.md)

## 문서 역할

- `readme-contract.md`: 루트, 트랙, 프로젝트, 구현 레인 README가 각각 답해야 할 질문
- `project-catalog.md`: 전체 프로젝트를 `문제 | 답 | 검증 | 상태` 축으로 정리한 카탈로그
- `verification-matrix.md`: 레인별 canonical verify command와 최신 상태
- `curriculum-map.md`: 왜 이 순서로 프로젝트를 배치했는지 설명하는 지도
- `skill-matrix.md`: 프로젝트별 기대 역량과 면접 질문 축
- `migration-map.md`: 그룹 재배치 전후 경로 대응표
- `legacy-audit.md`: 초기 원본 자산을 어떻게 현재 학습 경로로 재구성했는지 남기는 역사 문서
