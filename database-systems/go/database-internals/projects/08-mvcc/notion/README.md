# 학습 노트 안내

이 폴더는 프로젝트 README에서 `문제 / 내 해법 / 검증`을 확인한 뒤, 설계 판단과 디버그 포인트를 더 추적하고 싶을 때 여는 보조 입구입니다.

## 언제 읽는가

- 구현이 왜 이런 분해를 택했는지 판단 근거가 궁금할 때 읽습니다.
- 테스트가 막는 실패 지점과 다음 프로젝트로 넘길 질문을 정리하고 싶을 때 읽습니다.
- README보다 긴 학습 메모가 필요하지만 `notion-archive/`까지 내려갈 정도는 아닐 때 읽습니다.

## 읽는 순서

1. `00-problem-framing.md`: 이번 단계의 범위, 성공 기준, 커리큘럼 위치를 먼저 확인합니다.
2. `01-approach-log.md`: 구현이 택한 분해 방식과 핵심 선택을 확인합니다.
3. `02-debug-log.md`: 다시 깨질 수 있는 지점과 회귀 확인 포인트를 확인합니다.
4. `03-retrospective.md`: 이번 단계에서 남긴 단순화와 다음 확장 질문을 확인합니다.
5. `04-knowledge-index.md`: 용어, 핵심 파일, 개념 문서, 검증 앵커를 빠르게 다시 찾습니다.

## 함께 보면 좋은 표면

- 현재 기준의 문제 정의와 검증 진입점은 `../README.md`와 `../problem/README.md`에 있습니다.
- 개념 설명은 `../docs/`에 있고, 이전 장문 기록은 `../notion-archive/`에 보존합니다.
