# Documentation Style Guide

기준일은 2026-03-11입니다. 이 레포의 공개 문서는 학습 아카이브답게 친절해야 하지만, 방문자가 첫 화면에서 길을 잃지 않도록 `front-door README` 규칙을 강하게 유지합니다.

## 기본 원칙

- 루트 README는 레포가 푸는 문제군과 시작 경로를 바로 보여 줍니다.
- 트랙 README는 프로젝트 순서를 고정하고, 각 프로젝트의 `문제 / 내 해법 / 검증 / 다음 단계`를 표 형태로 제공합니다.
- 프로젝트 README는 첫 화면에 반드시 `문제`, `내 해법`, `검증`, `코드 지도`, `읽는 순서`, `한계와 확장`을 포함합니다.
- `problem/README.md`는 요구사항과 제약, 제공 자료, provenance를 설명하는 문서입니다. 해법 요약은 프로젝트 README로 올립니다.
- 긴 설명과 개념 정리는 `docs/`나 `notion/`으로 보내고, README는 인덱스 성격을 유지합니다.

## 언어 규칙

- 설명형 문서는 한국어를 기본으로 씁니다.
- 코드 식별자, 파일 경로, 명령, 프로토콜명, 라이브러리명은 영어 그대로 둡니다.
- 영어를 그대로 두는 편이 더 정확한 technical token은 번역하지 않습니다.

## 구조 규칙

- 실제 프로젝트는 각 트랙의 `projects/<nn-slug>/` 아래에 둡니다.
- 프로젝트 내부에서는 `problem/`, `docs/`, `notion/`, `notion-archive/` 표면을 유지합니다.
- 루트 `docs/`는 `catalog/`, `curriculum/`, `policies/`로 나눠 탐색 비용을 낮춥니다.
