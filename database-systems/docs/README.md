# Docs Index

이 디렉터리는 이 레포를 어떻게 읽어야 하는지 설명하는 공용 문서 모음입니다. 처음 온 학습자는 여기서 커리큘럼 설계 이유와 문서 계약을 먼저 확인하면 나머지 프로젝트를 훨씬 덜 헤매고 읽을 수 있습니다.

## 먼저 읽으면 좋은 문서

1. [documentation-style-guide.md](documentation-style-guide.md): 이 레포의 문서가 어떤 톤과 구조를 따라야 하는지 정리한 기준입니다.
2. [curriculum-audit.md](curriculum-audit.md): 왜 Python-first / Go-second 구조를 택했는지 설명합니다.
3. [curriculum-map.md](curriculum-map.md): 과거 과제군이 현재 프로젝트로 어떻게 재구성됐는지 보여 줍니다.
4. [language-crosswalk.md](language-crosswalk.md): Python 입문 트랙과 Go 정본 트랙을 어떻게 대응해 읽으면 좋은지 정리합니다.
5. [migration-conventions.md](migration-conventions.md): 프로젝트 문서 표면과 provenance, verification, `notion/` 정책을 명시합니다.

## 이 디렉터리에서 답하려는 질문

- 이 레포는 무엇을 배우게 하는가
- 왜 어떤 프로젝트는 나뉘고, 어떤 프로젝트는 합쳐졌는가
- Python과 Go는 어떤 관계인가
- 공개 문서와 학습 노트는 어떤 역할 차이가 있는가
- 포트폴리오용 레포로 옮길 때 어떤 정보를 남겨야 하는가
