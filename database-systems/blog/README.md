# Blog Index

`database-systems/`의 README가 "무엇을 만들었는가"를 빠르게 보여 주는 표면이라면, 이 `blog/`는 "어떻게 이해가 바뀌었는가"를 남기는 긴 로그다.

## 이 계층에서 고정한 원칙

- 단위는 `독립 프로젝트`다. 트랙 전체 회고문으로 쓰지 않는다.
- 문체는 결과 요약이 아니라 탐색 로그다. 틀린 가설과 폐기한 시도를 숨기지 않는다.
- 근거는 `src/`, `tests/`, `README/problem/docs`, 재검증 CLI만 사용한다.
- 코드는 핵심 조각만 본문 inline으로 남긴다.
- CLI는 실제 실행 순서를 유지해 기록한다.

3. 각 프로젝트의 `00-series-map.md`로 들어가 chronology를 순서대로 읽는다.
