# Fullstack 자소서 Module

저는 좋은 제품은 화면과 API가 따로 존재하는 것이 아니라, 하나의 사용자 흐름 안에서 맞물려야 완성된다고 생각합니다. 그래서 프론트엔드에서 시작했지만 자연스럽게 인증, 데이터, 비동기 처리, 검증 체계까지 함께 보게 됐습니다.

`mini-vrew`에서는 사용자에게 보이는 경험과 브라우저 처리 파이프라인을 함께 설계했고, `Client Onboarding Portal`에서는 validation과 retry를 제품 흐름에 녹였습니다. FastAPI 기반 `workspace-backend` 계열에서는 같은 도메인을 인증과 서비스 경계, 알림 흐름까지 포함해 정리했습니다.

제가 풀스택 역할에 적합하다고 생각하는 이유는 두 영역을 조금씩 한다는 의미가 아니라, 사용자 흐름을 앞뒤로 연결해 설명할 수 있기 때문입니다. 어느 한쪽만 보는 대신, 어떤 경계에서 실패가 생기고 어떻게 복구할지를 먼저 생각하는 편입니다.

협업에서도 기능 구현만이 아니라 검증 기준과 문서를 함께 정리하는 사람이 되고 싶습니다. 그래야 팀 안에서 맥락 비용이 줄고, 기능이 더 안정적으로 이어질 수 있다고 생각합니다.

입사 후에는 제품의 핵심 흐름과 현재 서비스 구조를 빠르게 이해하고, 맡은 기능을 화면과 API 양쪽에서 안정적으로 완성하는 데 기여하고 싶습니다.
