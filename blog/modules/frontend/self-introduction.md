# Frontend 자소서 Module

저는 프론트엔드 개발을 좋아하는 이유가 사용자가 가장 먼저 체감하는 변화를 직접 만들 수 있기 때문이라고 생각합니다. 다만 좋은 화면은 보기 좋은 것만으로 완성되지 않고, 상태 전이와 실패 처리, 검증 루프까지 함께 설계되어야 한다고 믿습니다.

`mini-vrew`에서는 브라우저 안에서 무거운 처리와 사용자 경험을 함께 다뤘고, `Ops Triage Console`과 `Client Onboarding Portal`에서는 운영 화면과 고객-facing 흐름을 각각 다른 기준으로 설계했습니다. 이 과정에서 기능을 한 번에 쌓기보다 사용자 흐름을 기준으로 쪼개고 검증하는 방식을 익혔습니다.

제가 프론트엔드 역할에 잘 맞는 이유는 화면만 보지 않고 브라우저와 런타임까지 함께 보기 때문입니다. 브라우저 API, 데이터 흐름, validation, retry, 테스트 루프를 함께 보며 구현해 왔습니다.

협업에서도 설명 가능한 결과물을 남기는 태도를 중요하게 생각합니다. README와 검증 명령, 문서를 정리해 다른 사람이 프로젝트를 빠르게 따라올 수 있게 만드는 편입니다.

입사 후에는 제품과 사용자 흐름을 빠르게 이해하고, 맡은 화면을 안정적으로 완성하는 동시에 팀 안의 프론트엔드 품질 기준을 높이는 데 기여하고 싶습니다.
