# Spring Backend 자소서 Module

저는 Spring 백엔드를 배울 때도 기능을 얼마나 많이 붙였는지보다, 같은 도메인을 어디까지 설계와 검증의 언어로 설명할 수 있는지가 더 중요하다고 생각했습니다. 그래서 `backend-spring`은 랩과 캡스톤의 연결 구조를 분명하게 남기는 방향으로 정리했습니다.

대표 결과물인 `commerce-backend-v2`에서는 persisted auth, JPA + Flyway, Redis cart/throttling, outbox + Kafka notification을 modular monolith에 연결했습니다. 같은 커머스 도메인을 baseline보다 더 깊게 구현해, 왜 v2가 필요한지를 문서와 검증 경로로 설명할 수 있게 만들었습니다.

제가 Spring 백엔드 역할에 적합하다고 생각하는 이유는 기술 키워드보다 경계와 검증을 먼저 보기 때문입니다. 인증, 인가, 이벤트, 캐시를 모두 넣더라도 실제 문제와 검증 기준이 분명하지 않으면 설득력이 떨어진다고 생각합니다.

협업에서도 이런 기준은 중요합니다. 코드와 문서, 테스트가 서로 다른 이야기를 하지 않도록 정리하고, 다른 사람이 프로젝트를 빠르게 따라올 수 있게 만드는 데 기여하고 싶습니다.

입사 후에는 도메인과 현재 아키텍처를 빠르게 이해하고, 맡은 서비스의 안정성과 설명 가능성을 함께 높이는 백엔드 엔지니어로 기여하고 싶습니다.
