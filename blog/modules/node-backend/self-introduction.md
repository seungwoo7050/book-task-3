# Node Backend 자소서 Module

저는 Node.js 백엔드를 학습할 때도 언어나 프레임워크 자체보다, 어떤 서비스 표면을 얼마나 설명 가능하게 만들었는지를 더 중요하게 봤습니다. 그래서 `backend-node`도 단순 실습 모음이 아니라, 문제와 답과 검증이 바로 읽히는 구조로 정리했습니다.

대표 결과물인 `10 shippable-backend-service`에서는 NestJS, Postgres, Redis, Swagger, Docker Compose를 한 제출용 서비스 표면으로 묶었습니다. JWT auth, RBAC, migration, cache, throttling을 실제 실행 경로와 함께 정리하며, 구현과 문서가 분리되지 않게 만드는 데 집중했습니다.

제가 Node 백엔드 역할에 적합하다고 생각하는 이유는 서비스 구조와 실행 경로를 함께 보는 습관 때문입니다. 단순히 코드가 돌아간다는 것보다, 다른 사람이 README만 보고 다시 띄울 수 있는 상태를 중요하게 생각합니다.

협업에서도 이런 태도는 그대로 이어집니다. 문제를 먼저 정리하고, 구현 결과를 문서와 검증 명령으로 남겨 팀이 같은 기준 위에서 일할 수 있게 돕고 싶습니다.

입사 후에는 현재 서비스의 요구사항과 운영 기준을 빠르게 파악하고, 맡은 기능을 안정적으로 구현하면서 검증 가능성과 문서화를 함께 높이는 데 기여하고 싶습니다.
