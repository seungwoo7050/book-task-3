# Infobank 경력기술서 Module

## 한 줄 소개

과제 요구를 공식 제출 답과 확장 답, 검증, 발표 자료까지 닫힌 결과물로 정리하는 경험을 갖고 있습니다.

## 핵심 역량

- release gate / compatibility gate
- compare artifact / evaluation proof
- runbook / 발표 자료 / 제출 산출물 정리

## 대표 경험

### 1. MCP 추천 최적화 capstone

- release candidate CRUD, compatibility gate, release gate, artifact export까지 정리

### 2. 챗봇 상담 품질 관리 capstone

- baseline/candidate 비교, guardrail, evidence verification, dashboard proof 구성

## 검증 습관

- 공식 답과 확장 답을 분리하고, 어떤 버전을 제출 기준으로 삼는지 문서 앞단에서 명시합니다.

## 성장 방향

과제형 결과물을 빠르게 제품화 관점으로 재구성하는 감각을 더 큰 실무 문제에도 연결하고 싶습니다.
