# Common Core 경력기술서 Module

## 한 줄 소개

42서울과 현재 워크스페이스의 CS·네트워크·데이터 프로젝트를 통해, 기능 구현 이전에 런타임과 경계를 먼저 이해하려는 개발 기반을 다졌습니다.

## 핵심 역량

- 시스템 기초: 메모리, 프로세스, 네트워크, 디버깅 중심 학습
- 팀 프로젝트 경험: 개인 파트보다 팀 단위 문제 해결과 검증 흐름을 중시
- 설명 가능한 결과물: 문서, 검증 명령, 재현 경로를 함께 정리하는 습관

## 대표 경험

### 1. 42서울 정규과정 수료

- 시스템 프로그래밍, 네트워크, 팀과제를 중심으로 개발 기반을 다졌습니다.
- 상세 사례는 `ft_transcendence` 하나만 사용하며, Django 백엔드 전담, 42 OAuth, JWT, TOTP 기반 2FA, OpenAPI 문서화 경험으로 정리합니다.
- `minishell`, `irc`, `raytracing`는 시스템 기초와 팀 협업 경험을 보여 주는 보조 사례로만 사용합니다.

### 2. CS / Network / Database 학습 축 구축

- [cs-core](../../../cs-core/README.md), [network-atda](../../../network-atda/README.md), [database-systems](../../../database-systems/README.md)를 학습 순서에 맞게 순차적으로 완료했습니다.
- 각 프로젝트를 독립적으로 끝내는 데서 그치지 않고, 이후 가지 프로젝트와 연결되는 기반 지식으로 정리했습니다.

## 검증 습관

- 검증 명령과 실행 진입점을 README 앞단에 둡니다.
- 프로젝트가 많아져도 `무슨 문제를 풀었고 어떻게 다시 검증하는지`가 흐려지지 않도록 문서 구조를 고정합니다.

## 성장 방향

공통 코어는 특정 직무를 위한 장식이 아니라, 어떤 직무로 가더라도 문제를 깊게 파고들 수 있는 기반입니다. 이후 제출본에서는 이 기반 위에 직무별 대표 결과물을 덧붙여 포지션 적합성을 분명히 드러냅니다.
