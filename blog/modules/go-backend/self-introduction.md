# Go Backend 자소서 Module

저는 백엔드에서 특히 API가 어떤 권한 경계와 데이터 모델을 가지는지 설명하는 일을 좋아합니다. Go 학습 트랙은 그 기준을 가장 제품형에 가깝게 다듬은 구간이었습니다.

`18 Workspace SaaS API`에서는 JWT auth, RBAC, invitation, issue workflow, notification을 한 제품형 API로 묶었고, `15 Event Pipeline`과 `17 Game Store Capstone`에서는 transaction과 outbox 같은 비동기 경계를 반복적으로 다뤘습니다. 이 경험을 통해 단순 CRUD보다 데이터 흐름과 실패 복구를 먼저 보는 습관을 길렀습니다.

제가 Go 백엔드 역할에 적합하다고 생각하는 이유는 언어를 사용했다는 사실보다, 구조와 검증을 함께 정리해 왔기 때문입니다. `go test`, `make e2e`, `make smoke` 같은 재현 경로를 꾸준히 남겼고, OpenAPI와 실제 시나리오를 함께 맞추려 노력했습니다.

협업에서도 구현만큼 중요한 것은 다른 사람이 다시 실행하고 이해할 수 있게 만드는 일이라고 생각합니다. README와 문서, 검증 명령을 함께 정리하는 습관으로 팀의 맥락 비용을 줄이고 싶습니다.

입사 후에는 먼저 현재 서비스의 핵심 도메인과 권한 경계를 빠르게 이해하고, 맡은 API를 안정적으로 구현하면서 검증 기준을 더 분명하게 만드는 데 기여하고 싶습니다.
