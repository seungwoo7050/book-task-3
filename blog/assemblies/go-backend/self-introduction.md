# Go Backend 자소서

저는 백엔드에서 특히 API가 어떤 권한 경계와 데이터 모델을 가지는지 설명하는 일을 좋아합니다. 42서울과 공통 코어 학습으로 시스템 기반을 쌓은 뒤, Go 트랙에서는 그 기준을 제품형 API로 가장 분명하게 다듬었습니다.

대표 결과물인 `18 Workspace SaaS API`에서는 JWT auth, RBAC, invitation, issue workflow, notification을 한 제품형 API로 묶었습니다. API server와 worker를 분리하고, Postgres/Redis와 smoke/e2e를 함께 정리하면서 단순 CRUD보다 구조와 검증을 더 앞에 두는 습관을 만들었습니다.

FastAPI 기반 `workspace-backend` 계열은 같은 협업형 도메인을 다른 경계로 다시 바라보게 해 준 공통 백엔드 기반이었습니다. 덕분에 특정 프레임워크의 문법보다, 인증과 비동기 처리, 서비스 분해를 어떤 기준으로 설명할지에 더 집중하게 됐습니다.

협업에서도 저는 구현만큼 중요한 것이 다른 사람이 다시 실행하고 이해할 수 있게 만드는 일이라고 생각합니다. README와 검증 명령, 문서를 함께 정리해 팀의 맥락 비용을 줄이고 싶습니다.

입사 후에는 먼저 현재 서비스의 핵심 도메인과 권한 경계를 빠르게 이해하고, 맡은 API를 안정적으로 구현하면서 검증 기준을 더 분명하게 만드는 데 기여하고 싶습니다.
