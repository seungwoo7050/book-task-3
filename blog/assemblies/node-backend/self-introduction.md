# Node Backend 자소서

저는 Node.js 백엔드를 학습할 때도 언어나 프레임워크 자체보다, 어떤 서비스 표면을 얼마나 설명 가능하게 만들었는지를 더 중요하게 봤습니다. 42서울과 공통 코어에서 쌓은 기반 위에, `backend-node`는 그 기준을 제출용 서비스 표면으로 옮겨 놓은 트랙이었습니다.

대표 결과물인 `10 shippable-backend-service`에서는 NestJS, Postgres, Redis, Swagger, Docker Compose를 한 제출용 서비스 표면으로 묶었습니다. JWT auth, RBAC, migration, cache, throttling을 실제 실행 경로와 함께 정리하며, 구현과 문서가 분리되지 않게 만드는 데 집중했습니다.

FastAPI 기반 `backend-common` 경험은 Node 제출본에도 중요한 공통 기반이 됐습니다. 특정 프레임워크 문법보다 인증과 API 경계, async flow를 어떤 기준으로 설명할지를 먼저 생각하게 해 줬기 때문입니다.

협업에서도 저는 문제 정의와 검증 기준을 먼저 정리하고, 다른 사람이 다시 실행하고 이해할 수 있게 만드는 사람이 되고 싶습니다. README와 문서, 검증 명령을 함께 정리하는 습관으로 팀의 맥락 비용을 줄이고자 합니다.

입사 후에는 현재 서비스의 요구사항과 운영 기준을 빠르게 파악하고, 맡은 기능을 안정적으로 구현하면서 검증 가능성과 문서화를 함께 높이는 데 기여하고 싶습니다.
