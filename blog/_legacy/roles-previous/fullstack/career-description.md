# Fullstack 경력기술서

> 풀스택 포지션 제출용으로 정리한 프로젝트 기반 경력기술서 초안입니다.

## 한 줄 소개

프론트엔드에서 출발했지만, 사용자 흐름을 완성하기 위해 API, 인증, 데이터 모델, 비동기 처리까지 함께 설계하는 풀스택 개발자입니다.

## 핵심 역량 요약

- 사용자 흐름 설계: React/Next.js 기반 제품형 UI와 실제 서비스 플로우 설계 경험
- 서비스 경계 이해: JWT auth, RBAC, invitation, async notification, retry, cache 등 API 설계 감각
- 재현 가능한 구현: typecheck, E2E, `go test`, `pytest`, smoke/demo CLI까지 포함한 검증 습관
- 문서화 능력: README, docs, notion, blog를 분리해 결과물을 설명 가능한 형태로 정리

## 대표 경험 1. 현재 워크스페이스 전체 프로젝트 순차 완료

- 형태: 장기 자기주도 학습 및 구현 아카이브
- 핵심 내용:
  - 프론트엔드, 백엔드, 모바일, 보안, 네트워크, CS 기초 프로젝트를 학습 순서에 맞게 순차적으로 구현하고 검증했습니다.
  - 특히 `front-react`, `backend-go`, `backend-fastapi`, `infobank`를 통해 제품 UI와 API 경계를 함께 설명 가능한 구조로 정리했습니다.
- 풀스택 관점의 의미:
  - 화면 구현과 서비스 구조를 분리해 보지 않고, 하나의 제품 문제로 연결해 보는 습관을 만들었습니다.

## 대표 경험 2. mini-vrew

- 형태: 프론트엔드 단독 개발 및 배포
- 링크:
  - GitHub: <https://github.com/seungwoo7050/mini-vrew>
  - Deploy: <https://mini-vrew.vercel.app>
- 핵심 내용:
  - 브라우저 기반 비디오 편집 웹앱을 설계하고 구현했습니다.
  - 업로드, 편집, 자막, 필터, 내보내기까지 이어지는 사용자 흐름을 하나의 제품 경험으로 연결했습니다.

  - 사용자에게 보이는 화면과 처리 파이프라인을 함께 설계하는 습관을 보여 줍니다.

## 기술 스택

### 핵심 스택

- React
- Next.js
- TypeScript
- Go
- FastAPI
- PostgreSQL
- Redis
- Docker

### 제품 흐름 관련 경험

- JWT auth / refresh
- RBAC / invitation
- async jobs / notification
- cache / retry / optimistic update
- Playwright / Vitest / go test / smoke

### 보조 스택

- Node.js / NestJS
- Spring Boot
- React Native

## 검증과 재현 습관

- 프론트엔드 프로젝트는 typecheck, unit test, E2E를 함께 묶습니다.
- 백엔드 프로젝트는 `go test`, `make e2e`, `make smoke`, `pytest`, demo CLI로 재현 경로를 남깁니다.
- 구현 결과를 단순 코드 묶음이 아니라 "다른 사람이 읽고 돌려볼 수 있는 프로젝트"로 정리하는 것을 중요하게 생각합니다.

## 성장 방향

- 실무에서는 사용자 요구와 서비스 구조가 충돌하는 지점을 더 많이 경험하며, 제품과 기술을 함께 조율하는 개발자로 성장하고 싶습니다.
- 장기적으로는 프론트와 백엔드를 모두 구현할 수 있는 사람을 넘어, 제품 흐름 전체의 설계 품질을 높이는 풀스택 개발자가 되고 싶습니다.
