# AI 블로그 작성 프롬프트 템플릿

이 문서는 AI에게 회사/개인 프로젝트의 탐색형 학습 로그를 작성하도록 지시할 때 사용할 수 있는 프롬프트 모음입니다.
아래 템플릿을 복사하여 AI 모델(예: GPT‑4, 내부 LLM 등)에 입력하고, 필요한 자료를 함께 제공하세요.

---

## 기본 템플릿
```
너는 이 저장소의 소스코드를 살펴보는 개발자다. 프로젝트를 처음 접한 상태에서 하나씩 기능을 구현하고 버그를 해결해 가는 과정을 **날짜/세션 타임라인 블로그**로 기록하라.

입력 자료:
- 최근 커밋 메시지 3~5개
- 변경된 파일 목록과 간단한 diff 설명
- 실패한 테스트 또는 로그 출력 스니펫
- CLI 히스토리 요약 (실행한 명령어 목록)

각 타임라인 엔트리는 다음 필드를 가져야 한다: 목표, 진행, 이슈, 판단, 조치, 검증, 다음.
- "처음엔 ~라고 생각했고", "이 시점에서는 ~를 몰랐다"와 같은 시점 표현을 사용하라.
- 가능한 경우 핵심 코드 스니펫을 포함하라.
- CLI가 없으면 "CLI 기록 없음"이라고 명시하라.
- 결과만 요약하지 말고, 실패와 가설 흐름을 드러내라.

출력 형식: Markdown.
```

## 세부 전략 프롬프트
- **커밋 중심**: "아래 커밋 로그를 읽고, 각 커밋이 어떤 가설/수정/테스트를 나타내는지 설명하면서 타임라인으로 정리해라."
- **테스트 중심**: "테스트 출력과 관련 코드 diff를 보고 어떤 버그를 찾았고 어떻게 고쳤는지 타임라인으로 적어라."
- **CLI 중심**: "명령어 히스토리(예: `make test`, `npm run dev`)를 보고 진행 상황과 검증 신호를 연대기 순으로 정리하라."

## 사용 방법
1. 리포지터리 루트에서 필요한 정보(커밋, diff, 로그, CLI)를 추출해 메모에 붙여넣는다.
2. 위 템플릿을 복사해 AI에게 입력한다.
3. 생성된 초안에서 chronology/가설 흐름/코드 스니펫을 사람 눈으로 검토 후 수정한다.
4. 수정 완료한 글을 `blog/` 디렉터리에 새로운 파일로 저장하고, PR로 리뷰를 받는다.

## 팁
- **샘플**: 잘 쓴 타임라인 문서를 같이 제공하면 스타일을 모방하게 할 수 있다.
- **분할**: 큰 프로젝트는 ‘모듈별’ 또는 ‘기간별’로 프롬프트를 나누어 사용한다.
- **반복**: AI 출력이 불충분하면 질문을 덧붙여 재요청한다.
- **기록**: 프롬프트 템플릿을 버전 관리해두고 필요 시 개선한다.

이 파일은 다른 팀원도 사용할 수 있도록 공개된 학습 자료로 활용하세요.
