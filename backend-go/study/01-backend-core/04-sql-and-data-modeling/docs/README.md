# Docs

## Overview

이 과제는 SQL 스키마 설계와 관계 모델링을 Go 코드와 함께 익히는 브리지 단계다.
정규화, FK, unique constraint, join, transaction을 작은 게임 상점 모델로 묶었다.

## Why This Project

API 과제로 바로 넘어가기 전에 SQL 모델링 감각이 없으면 이후 `database/sql`,
transaction, optimistic locking 과제를 표면적으로만 다루게 된다. 이 과제는 그
공백을 메우는 역할을 한다.
