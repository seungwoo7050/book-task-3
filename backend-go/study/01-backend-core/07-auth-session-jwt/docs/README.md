# Docs

## Overview

이 과제는 세션 기반 로그인과 JWT 기반 인증을 나란히 보여 준다.
입문자가 authentication과 authorization을 분리해 생각하도록 만드는 데 목적이 있다.

## Why This Project

실제 백엔드 실무에서는 CRUD보다 먼저 auth가 붙는 경우가 많다. 그래서 이 과제는
`401`과 `403`의 차이, password hashing, session/JWT의 책임 범위를 빠르게
고정하는 역할을 한다.
