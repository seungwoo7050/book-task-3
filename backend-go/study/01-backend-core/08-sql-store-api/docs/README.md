# Docs

## Overview

이 과제는 `database/sql` 기반 CRUD, migration up/down, repository 계층,
optimistic update, transaction rollback을 한 번에 묶는 핵심 브리지 과제다.

## Why This Project

실제 백엔드 개발자는 결국 영속 저장소를 다뤄야 한다. 그래서 이 과제는 in-memory
API 다음 단계로, 가장 단순한 `database/sql` 흐름을 명확히 보여 주는 데 초점을 뒀다.
