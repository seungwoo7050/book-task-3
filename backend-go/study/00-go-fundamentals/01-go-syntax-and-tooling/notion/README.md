# 01 Go Syntax And Tooling — Notion 문서 안내

이 폴더는 프로젝트 01을 수행하면서 겪은 과정, 판단, 시행착오를 기록한 에세이 모음이다.
소스코드만으로는 보이지 않는 맥락—왜 그 구조를 택했고, 어디서 막혔고, 무엇을 배웠는지—을 담고 있다.

## 읽기 순서

| 순서 | 파일 | 언제 읽으면 좋은가 |
|------|------|---------------------|
| 1 | [00-problem-framing.md](00-problem-framing.md) | 프로젝트를 처음 접할 때. 문제를 왜 이렇게 정의했는지 알고 싶을 때. |
| 2 | [01-approach-log.md](01-approach-log.md) | 구현 방향을 잡는 과정을 따라가고 싶을 때. |
| 3 | [02-debug-log.md](02-debug-log.md) | 비슷한 실수를 피하고 싶을 때. 어디서 막혔는지 궁금할 때. |
| 4 | [03-retrospective.md](03-retrospective.md) | 프로젝트를 끝낸 뒤 무엇을 얻었는지 정리하고 싶을 때. |
| 5 | [04-knowledge-index.md](04-knowledge-index.md) | 특정 개념이나 표준 라이브러리 사용법을 빠르게 찾고 싶을 때. |
| 6 | [05-timeline.md](05-timeline.md) | 전체 개발 과정을 시간순으로 재현하고 싶을 때. |

## 이 문서들의 성격

- **블로그형 에세이**: "처음 보는 사람이 흐름을 따라오게 만드는" 문체로 쓰였다.
- **소스코드 보완**: 코드에서 드러나지 않는 CLI 명령, 도구 선택 이유, 삽질 과정을 포함한다.
- **재현 가능성**: 이 문서와 소스코드를 함께 보면 프로젝트를 처음부터 다시 만들 수 있다.
