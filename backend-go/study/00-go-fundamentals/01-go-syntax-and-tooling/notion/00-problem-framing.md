# 문제 정의 — 왜 문자열 정규화와 빈도 계산인가

## 출발점

Go를 처음 배울 때 가장 빠르게 손에 익혀야 하는 것들이 있다. 변수 선언, 함수 정의, 제어 흐름, 그리고 `go run`과 `go test`를 터미널에서 돌려 보는 감각이다. 이걸 한 번에 연습하려면 적당히 작으면서도 여러 문법 요소를 건드리는 문제가 필요했다.

문자열을 받아서 단어로 쪼개고, 빈도를 세고, 가장 많이 나온 단어를 찾는 것. 이 정도면 `for`, `if`, `map`, `slice`, 함수 반환값을 전부 한 번씩은 쓰게 된다. 동시에 문제 자체가 너무 깊지 않아서, "언어를 익히는 데 집중"할 수 있다.

## 왜 이 크기인가

처음에는 파일을 읽어서 통계를 내는 것까지 넣을까 고민했다. 하지만 파일 I/O가 들어가면 에러 처리와 `os.Open` 같은 것들이 함께 들어오고, 그러면 입문자의 시선이 문법이 아니라 I/O 쪽으로 분산된다. 그래서 의도적으로 stdin/flag 처리를 빼고, 하드코딩된 문자열 하나에서 시작하되 `os.Args`로 간단히 확장할 수 있는 구조로만 열어 뒀다.

## 핵심 제약

- **외부 패키지 없음**: `go mod init`만 하면 바로 돌아가야 한다. 표준 라이브러리(`strings`, `sort`, `unicode`)만 사용한다.
- **테스트 가능해야 함**: `go test ./...` 한 줄로 검증이 끝나야 한다.
- **CLI로도 확인 가능해야 함**: `go run ./cmd/toolingdemo`로 눈에 보이는 출력이 나와야 한다.

## 문제를 이렇게 잘라낸 이유

"문자열 처리"라는 큰 주제에서 정규화 → 빈도 계산 → 요약 이 세 단계만 남겼다. 정규식도 쓸 수 있었지만, `strings.FieldsFunc`로도 충분한 문제였기 때문에 더 단순한 쪽을 택했다. 입문 과제에서 정규식을 꺼내면, 정규식 자체를 설명하느라 Go 문법 연습이라는 본래 목적이 흐려진다.

결과적으로 이 프로젝트는 "Go 문법과 도구 사용에만 집중하는 가장 작은 예제"로 설계됐다.
