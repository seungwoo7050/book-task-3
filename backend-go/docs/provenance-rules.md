# Provenance Rules

## 출처 라벨

- `Legacy source`: legacy에서 가져온 문제나 구현 뿌리
- `Study-authored`: study에서 새로 작성한 문제, 문서, 구현
- `Public note`: `notion/`에 추적되는 접근 로그, 디버그 기록, 회고

## 주장 규칙

- 재현 가능한 명령이 있으면 그대로 적는다.
- 아직 검증하지 못한 내용은 추정으로 적지 않고 `제외 범위` 또는 `현재 한계`에 둔다.
- 존재하지 않는 파일이나 로컬 절대 경로를 README에서 참조하지 않는다.
- 외부 원문 기반 문제는 한국어 canonical 문서로 다시 쓰고, legacy 또는 원문 출처는 provenance 섹션에 남긴다.
