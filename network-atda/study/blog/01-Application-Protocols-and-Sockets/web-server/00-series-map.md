# Web Server 시리즈 맵

이 lab의 핵심 질문은 HTTP를 얼마나 많이 구현했느냐가 아니다. 요청 한 줄을 읽고, 파일 하나를 열어, 적절한 상태 코드와 헤더를 붙이고, 연결을 닫는 최소 서버 생애주기를 어떻게 손으로 구현하는가가 더 중요하다.

## 이 lab를 읽는 질문

- 요청 라인에서 최소한 무엇만 파싱해도 파일 서빙이 가능한가
- 200과 404는 어떤 분기에서 갈리는가
- multi-threaded accept loop는 어디까지가 현재 구현의 범위인가

## 이번에 고정한 사실

- 기본 페이지는 빈 path에 대해 `hello.html`
- 파일 확장자로 `Content-Type` 추정
- 파일이 없으면 HTML 404 body와 함께 `404 Not Found`
- 각 연결은 daemon thread 하나에서 처리
