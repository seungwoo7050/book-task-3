# 03 회고

## 이번 프로젝트에서 확실히 얻은 것
- TCP 서버가 실제로는 요청 파싱, 파일 시스템, 응답 직렬화가 한 덩어리로 묶인다는 점을 체감했다.
- 브라우저가 왜 상태 코드와 `Content-Type`에 민감한지 작은 서버로 직접 확인했다.

## 아직 약한 부분
- 보안 경계는 학습용 수준이다. 경로 검증과 자원 제한은 더 보강해야 한다.
- 스레드당 연결 모델은 작은 예제에는 충분하지만 높은 동시성에는 맞지 않는다.

## 포트폴리오로 키우려면
- 정적 파일 루트 제한과 경로 정규화를 넣어 안전한 개발용 서버로 확장한다.
- 요청 로그와 간단한 벤치마크 결과를 추가해 성능/관측성까지 보여준다.
- `asyncio` 버전을 병렬 구현으로 추가해 같은 문제를 다른 동시성 모델로 비교한다.
