# 02 디버그 로그

이 파일은 과거의 날짜 순서를 복원하려는 문서가 아니다. 백업 노트와 현재 테스트를 함께 보고, 지금도 재현하거나 설명할 수 있는 문제만 남긴다.

## 사례 1. 한 번의 timeout이 전체 프로그램을 중단시킴
- 증상: 몇 번째 요청에서든 `socket.timeout`이 발생하면 나머지 반복이 실행되지 않았다.
- 원인: 예외를 루프 바깥에서 처리해 손실 한 건이 전체 세션 실패로 번졌다.
- 수정: 각 반복 안에서 timeout을 잡아 손실만 기록하고 다음 시도로 진행하게 바꿨다.
- 확인: `python/tests/test_udp_pinger.py`의 손실 테스트와 제공 서버 반복 실행으로 확인했다.

## 사례 2. 평균 RTT가 손실 개수에 따라 이상하게 커짐
- 증상: 응답이 없는 패킷까지 평균에 포함하면 통계가 실제 RTT보다 과장됐다.
- 원인: 손실을 숫자 `0` 또는 timeout 값으로 섞어 계산하면 성공한 RTT의 의미가 사라진다.
- 수정: 성공 샘플만 별도 리스트에 저장하고 손실은 카운트만 올렸다.
- 확인: 응답이 전혀 없는 경우 RTT 요약을 생략하는지와 손실 수가 따로 집계되는지를 출력으로 확인했다.

## 사례 3. 응답 내용이 원문과 달라 파싱이 어색함
- 증상: 서버 응답이 대문자로 바뀌어 오는데 요청 문자열과 동일하다고 가정하면 비교가 꼬였다.
- 원인: 제공 서버는 요청 내용을 그대로 돌려주는 대신 대문자 변환을 적용한다.
- 수정: 응답 본문은 표시용으로만 사용하고 RTT 계산은 송신 시각과 수신 시각에만 의존하도록 정리했다.
- 확인: `test_reply_is_uppercased`와 실제 제공 서버 응답을 함께 확인했다.
