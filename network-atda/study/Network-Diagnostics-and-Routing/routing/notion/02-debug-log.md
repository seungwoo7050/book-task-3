# 02 디버그 로그

이 파일은 과거의 날짜 순서를 복원하려는 문서가 아니다. 백업 노트와 현재 테스트를 함께 보고, 지금도 재현하거나 설명할 수 있는 문제만 남긴다.

## 사례 1. 한 노드가 받은 DV를 수정하면서 다른 노드 상태까지 함께 변함
- 증상: 업데이트 순서에 따라 결과가 흔들리거나 설명되지 않는 값 변화가 나타났다.
- 원인: 같은 프로세스 안에서 딕셔너리 참조를 공유하면 실제 네트워크 패킷처럼 복사본이 만들어지지 않는다.
- 수정: `get_dv()`와 수신 저장 시점에 `deep copy`를 사용했다.
- 확인: 반복 실행에서 결과가 안정적인지와 테스트 통과 여부로 확인했다.

## 사례 2. 같은 iteration 안에서 먼저 갱신된 값이 뒤 노드 계산에 섞여 순서 의존성이 생김
- 증상: 노드 순회를 바꾸면 iteration 수나 중간 출력이 달라졌다.
- 원인: 동기식 시뮬레이션인데도 메시지 수집과 적용을 한 단계에 섞어 버렸다.
- 수정: 모든 노드의 현재 DV를 먼저 모은 뒤, 그 스냅샷만 사용해 한 회차를 갱신하도록 2-phase 구조를 유지했다.
- 확인: `test_convergence`와 수동 출력 비교로 검증했다.

## 사례 3. 자기 자신으로 가는 비용이나 next-hop이 잘못 채워짐
- 증상: 초기 상태나 수렴 결과에서 self-route가 불필요하게 갱신됐다.
- 원인: 자기 자신을 일반 목적지처럼 갱신 대상으로 섞으면 self cost `0`이라는 불변 조건이 흐려진다.
- 수정: 초기화 단계에서 self cost를 고정하고 self link를 별도 예외로 다뤘다.
- 확인: `test_no_self_links`와 초기 DV 테스트로 확인했다.
