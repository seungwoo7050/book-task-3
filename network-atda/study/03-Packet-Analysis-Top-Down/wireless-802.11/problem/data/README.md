# Trace Files

## Required Files

| File | Description |
|------|-------------|
| `wireless-trace.pcap` | Pre-captured 802.11 wireless traffic in monitor mode, containing beacon frames, probe requests/responses, authentication, association, data frames, and ACK frames |

> Note: The repository includes a reproducible **synthetic** `wireless-trace.pcap`
> generated by `Wireshark-Labs/scripts/generate_synthetic_traces.py`.
> You may replace it with textbook/real monitor-mode captures when available.

## Self-Capture Instructions

To capture your own 802.11 wireless traffic, you need a wireless adapter that supports **monitor mode**:

### macOS

```bash
# Create a capture using the built-in Wi-Fi diagnostics
# Or use the airport utility:
sudo /System/Library/PrivateFrameworks/Apple80211.framework/Versions/Current/Resources/airport en0 sniff <channel>
```

### Linux

```bash
# Enable monitor mode
sudo airmon-ng start wlan0

# Capture with tshark
sudo tshark -i wlan0mon -w wireless-trace.pcap -a duration:60

# Or use Wireshark GUI (run as root or with appropriate permissions)
sudo wireshark
```

### Important Notes

- Monitor mode is required to see full 802.11 headers (management and control frames)
- Normal (managed mode) captures only show data frames with fake Ethernet headers
- The capture must include radiotap or PPI headers for proper dissection
- Capture near an active AP to ensure beacon frames and association traffic are visible
- A 60-second capture typically provides sufficient material for analysis

Place captured files in this directory before running analysis commands.
