# ARP 보안 위협과 LAN 스위칭 심화

## 개요

ARP는 인증 메커니즘 없이 설계되었으므로 다양한 보안 위협에 노출된다.
또한 Ethernet 스위칭 환경에서의 MAC 주소 학습, VLAN, STP 등
링크 계층의 핵심 개념들이 ARP 동작과 밀접하게 관련된다.
이 문서는 Ethernet/ARP 실습에서 관찰한 내용을 보안과 스위칭 관점에서 확장한다.

---

## 1. ARP 보안 위협

### ARP 스푸핑 (ARP Spoofing / ARP Poisoning)

공격자가 위조된 ARP 응답을 전송하여 다른 호스트의 ARP 캐시를 오염시킨다.

```
정상 상황:
  Gateway IP: 192.168.1.1 → Gateway MAC: AA:AA:AA:AA:AA:AA

공격 후:
  Gateway IP: 192.168.1.1 → 공격자 MAC: CC:CC:CC:CC:CC:CC
  → 피해자의 모든 외부 트래픽이 공격자를 경유
```

### 중간자 공격 (MITM)

ARP 스푸핑을 양방향으로 수행하면 통신 양쪽 모두를 속일 수 있다:

```
피해자 A의 ARP 캐시: B의 IP → 공격자 MAC
피해자 B의 ARP 캐시: A의 IP → 공격자 MAC

A → 공격자 → B (공격자가 트래픽 열람/변조 가능)
B → 공격자 → A
```

### ARP 스톰

대규모 브로드캐스트 도메인에서 다수의 ARP 요청이 동시에 발생하면
네트워크 대역폭을 소모하고 스위치 CPU 부하를 증가시킨다.

---

## 2. ARP 보안 대책

### Dynamic ARP Inspection (DAI)

스위치가 ARP 패킷의 유효성을 DHCP Snooping 바인딩 테이블과 대조하여 검증한다:

| 검증 항목 | 내용 |
| :--- | :--- |
| 출발지 MAC | ARP 헤더의 Sender MAC과 Ethernet 헤더의 Source MAC 일치 여부 |
| IP-MAC 바인딩 | DHCP Snooping 테이블의 정당한 바인딩과 일치 여부 |
| 대상 MAC | ARP 응답의 Target MAC 유효성 |

### Static ARP 엔트리

중요한 호스트(게이트웨이 등)의 ARP 엔트리를 정적으로 설정하여
캐시 오염을 방지한다:

```bash
arp -s 192.168.1.1 AA:AA:AA:AA:AA:AA
```

### 802.1X 포트 인증

스위치 포트에 연결되는 장치의 인증을 요구하여
비인가 장치의 네트워크 접근 자체를 차단한다.

## 3. Ethernet 스위칭 심화

### MAC 주소 학습 (Self-Learning)

스위치는 수신 프레임의 **출발지 MAC 주소**를 MAC 주소 테이블에 기록한다:

```
프레임 수신 (포트 1, 출발지 MAC: AA:BB:CC:DD:EE:FF)
→ MAC 테이블: AA:BB:CC:DD:EE:FF → 포트 1 (에이징 타이머 시작)
```

- **유니캐스트**: 목적지 MAC이 테이블에 있으면 해당 포트로만 전달
- **브로드캐스트**: 모든 포트로 전달 (ARP 요청의 경우)
- **알 수 없는 유니캐스트**: 모든 포트로 플러딩 (flooding)

### VLAN (Virtual LAN)

물리적 스위치를 논리적으로 분리하여 브로드캐스트 도메인을 제한한다:

```
VLAN 10: 포트 1-12 (영업부)
VLAN 20: 포트 13-24 (개발부)
→ VLAN 10의 ARP 브로드캐스트는 VLAN 20에 전달되지 않음
```

**ARP와의 관계**:
- 같은 VLAN 내에서만 ARP 요청이 전파됨
- VLAN 간 통신은 라우터(L3 스위치)를 경유해야 함
- 브로드캐스트 스톰의 영향 범위를 VLAN 단위로 격리

### STP (Spanning Tree Protocol)

스위치 간 루프를 방지하기 위해 일부 포트를 차단 상태로 전환한다:

```
스위치 A ──── 스위치 B
  |              |
  └──── 스위치 C ──┘   ← 루프 발생 가능

STP 적용 후:
  스위치 C의 한 포트 차단 → 루프 방지
```

루프가 발생하면 Ethernet 프레임이 무한 순환하며
브로드캐스트 스톰을 유발하므로 STP는 필수적이다.

---

## 4. Gratuitous ARP

호스트가 자신의 IP에 대한 ARP 요청을 브로드캐스트하는 특수한 방식이다:

```
송신자 IP = 대상 IP = 자기 자신의 IP
→ "내 IP는 이것이고 내 MAC은 이것이다" 공지
```

### 용도

| 목적 | 설명 |
| :--- | :--- |
| IP 충돌 감지 | 응답이 오면 다른 호스트가 같은 IP를 사용 중 |
| ARP 캐시 갱신 | 네트워크의 모든 호스트가 새 MAC 주소를 학습 |
| 장애 복구 | 이중화 서버가 Failover 시 새 MAC을 공지 |

---

## 5. 실습과의 연결

Ethernet/ARP Wireshark 실습에서 관찰한 내용을 보안/스위칭 관점으로 확장:

1. **ARP 브로드캐스트 관찰**: ff:ff:ff:ff:ff:ff 대상 프레임 → 대규모 네트워크에서 스톰 위험
2. **ARP 응답의 무인증**: 실습에서 확인한 단순 요청/응답 → 스푸핑 공격의 근본 원인
3. **EtherType 0x0806**: ARP 프레임 식별 → DAI에서의 필터링 기준
4. **ARP 캐시 타임아웃**: 실습에서 관찰한 캐시 동작 → Static ARP로 보안 강화
5. **MAC 주소의 역할**: Ethernet 프레임 전달의 기초 → 스위치 MAC 학습과 VLAN 분리
