# 02 디버그 로그

이 파일은 과거의 날짜 순서를 복원하려는 문서가 아니다. 백업 노트와 현재 테스트를 함께 보고, 지금도 재현하거나 설명할 수 있는 문제만 남긴다.

## 사례 1. 원본 서버가 요청을 이해하지 못함
- 증상: 프록시에는 정상 요청처럼 보였는데 원본 서버는 `400` 또는 빈 응답을 돌려줬다.
- 원인: 프록시에 들어오는 절대 URL을 원본 서버에 그대로 전달하면 origin-form을 기대하는 서버와 맞지 않는다.
- 수정: 호스트/포트를 분리한 뒤 원본 요청 라인은 상대 경로만 남기도록 바꿨다.
- 확인: `TestURLParsing`과 `problem/script/test_proxy.sh`의 실제 포워딩 검증으로 확인했다.

## 사례 2. URL에 따라 캐시 저장이 실패하거나 경로가 깨짐
- 증상: 쿼리 문자열이나 깊은 경로가 있는 URL에서 캐시 파일 생성이 불안정했다.
- 원인: URL 원문에는 파일 시스템 경로 구분자와 예약 문자가 들어갈 수 있다.
- 수정: URL 전체를 MD5 해시로 변환해 고정 길이 파일명을 만들었다.
- 확인: `TestCachePath`와 origin 서버 종료 후 캐시 히트 시나리오로 확인했다.

## 사례 3. 원본 서버 연결 오류와 timeout이 같은 실패로 보임
- 증상: 접속 거부와 응답 지연이 모두 같은 메시지로만 나타났다.
- 원인: 예외를 한 종류로 뭉개면 디버깅과 사용자 설명이 어려워진다.
- 수정: 연결 실패는 `502`, 응답 대기는 `504`로 구분해 반환했다.
- 확인: 테스트 스크립트와 수동 재현에서 각각 다른 코드가 내려오는지 확인했다.
