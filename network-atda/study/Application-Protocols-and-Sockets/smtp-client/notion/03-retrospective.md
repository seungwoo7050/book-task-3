# 03 회고

## 이번 프로젝트에서 확실히 얻은 것
- 텍스트 프로토콜도 결국 엄격한 상태 전이 규칙 위에서만 안정적으로 동작한다는 점을 배웠다.
- `헤더`와 `전달 경로`가 다른 계층이라는 사실이 메일에서 특히 중요하다는 점을 이해했다.

## 아직 약한 부분
- 멀티라인 응답, 인증, TLS, 대용량 본문 같은 실제 운영 이슈는 다루지 않았다.
- 에러 복구는 fail-fast 중심이어서 사용자 친화적인 재시도 전략은 아직 없다.

## 포트폴리오로 키우려면
- mock 서버 로그와 함께 SMTP 세션을 시각화한 다이어그램을 추가한다.
- `STARTTLS`와 인증 단계를 분리한 확장 버전을 만들어 기본 SMTP와 비교한다.
