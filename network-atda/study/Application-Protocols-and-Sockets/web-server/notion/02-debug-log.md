# 02 디버그 로그

이 파일은 과거의 날짜 순서를 복원하려는 문서가 아니다. 백업 노트와 현재 테스트를 함께 보고, 지금도 재현하거나 설명할 수 있는 문제만 남긴다.

## 사례 1. 응답을 보내고도 클라이언트가 종료되지 않음
- 증상: `curl`이나 브라우저가 본문을 받은 뒤에도 소켓이 열린 채로 남아 요청이 매달렸다.
- 원인: 메시지 끝을 구분할 방법을 충분히 주지 않은 상태에서 연결을 계속 유지하면 단순 클라이언트가 종료 시점을 모른다.
- 수정: `Connection: close`를 명시하고 요청 하나를 처리한 뒤 소켓을 닫는 흐름으로 단순화했다.
- 확인: `python/tests/test_web_server.py`의 순차 요청 테스트와 `problem/script/test_server.sh`로 확인했다.

## 사례 2. 서버를 빠르게 재시작하면 포트 바인딩이 실패함
- 증상: 직전 실행을 종료한 직후 다시 띄우면 `Address already in use`가 발생했다.
- 원인: `TIME_WAIT` 상태가 남아 있는 포트에 바로 재바인딩하려고 했다.
- 수정: 서버 소켓에 `SO_REUSEADDR`를 설정해 개발용 재시작 흐름을 안정화했다.
- 확인: 여러 번 연속으로 `make -C study/Application-Protocols-and-Sockets/web-server/problem test`를 돌려 재현이 사라졌는지 확인했다.

## 사례 3. HTML 파일을 내려도 브라우저가 텍스트로 취급함
- 증상: `hello.html`이 정상 전송되어도 렌더링 방식이 기대와 달랐다.
- 원인: `Content-Type`을 확장자별로 나누지 않으면 기본값이 모호해진다.
- 수정: `.html`, `.txt` 등 최소 확장자 맵을 두고 헤더에 반영했다.
- 확인: `test_response_contains_html`와 수동 `curl -i` 확인으로 검증했다.
