# Trace Files

## Required Files

| File | Description |
|------|-------------|
| `tls-trace.pcap` | Pre-captured HTTPS session with full TLS handshake, certificate exchange, and encrypted application data |

> Note: The repository includes a reproducible **synthetic** `tls-trace.pcap`
> generated by `Wireshark-Labs/scripts/generate_synthetic_traces.py`.
> You may replace it with a textbook or self-captured trace for deeper analysis.

## Optional Files

| File | Description |
|------|-------------|
| `sslkeylog.txt` | Pre-master secret log file for TLS decryption in Wireshark (for Q20) |

## Self-Capture Instructions

### Capture HTTPS Traffic

```bash
# Start Wireshark capture on your primary network interface, then visit any HTTPS website
wireshark -k -i en0 -f "tcp port 443"
```

### Enable TLS Decryption (Optional)

To decrypt TLS traffic, export the session keys using `SSLKEYLOGFILE`:

#### Chrome / Firefox

```bash
# Set environment variable before launching the browser
export SSLKEYLOGFILE=$PWD/data/sslkeylog.txt

# Launch browser
open -a "Google Chrome"
# or
open -a "Firefox"
```

#### Configure Wireshark Decryption

1. Edit → Preferences → Protocols → TLS
2. Set "(Pre)-Master-Secret log filename" to the path of `sslkeylog.txt`
3. Wireshark will automatically decrypt sessions whose keys are in the log

### Capture Tips

- Capture on port 443 to focus on HTTPS traffic
- Visit a simple HTTPS website (e.g., https://example.com) for clean analysis
- A single page load typically generates sufficient handshake and data traffic
- Close other browser tabs to reduce noise from concurrent TLS sessions
- Capture both the initial handshake and some data transfer (5-10 seconds is usually sufficient)

Place captured files in this directory before running analysis commands.
