# DNS Lab — Problem Specification

## 안내

이 문서는 제공 과제 사양을 source-close하게 보존하기 위해 원문 중심으로 유지한다.
공개용 문제 요약과 학습 맥락은 상위 `README.md`를 먼저 참고한다.


## Objective

Analyze pre-captured DNS packet traces using Wireshark to understand how the Domain Name System resolves domain names into IP addresses. You will examine DNS query/response structure, record types, and the `nslookup` / `dig` tools.

## Trace Files

The following trace files are provided in `data/`:

| File | Scenario | Description |
| :--- | :--- | :--- |
| `dns-nslookup.pcapng` | nslookup queries | DNS queries generated by `nslookup` for various domains |
| `dns-web-browsing.pcapng` | Web browsing | DNS queries triggered by opening web pages in a browser |

## Instructions

Open each trace file in Wireshark and apply the `dns` display filter. Then answer the following questions.

---

## Part 1: nslookup

**Trace file**: `dns-nslookup.pcapng`

Use `nslookup` to generate DNS queries. The trace captures the resulting DNS traffic.

1. What transport-layer protocol is used by DNS? What port number does the DNS server listen on?
2. To what IP address is the DNS query message sent? Is this the IP address of your default local DNS server? How do you determine the IP of your local DNS server?
3. Examine the DNS query message. What is the **type** of the DNS query? Does the query contain any "answers"?
4. Examine the DNS response message. How many **answers** are provided? What does each answer contain?
5. What is the canonical name (`CNAME`) for the queried host, if any? What are the IP addresses returned?
6. Examine the response for an `nslookup` of a mail server (MX record query). What mail server names and preference values are returned?

---

## Part 2: nslookup — Authoritative and Non-Authoritative

7. To what IP address is the DNS query message sent? Is this the IP address of your default local DNS server?
8. Examine the DNS query message. What type of DNS query is it? Does the query message contain any answers?
9. Examine the DNS response message. Provide the authoritative nameservers. What is the difference between an **authoritative** and **non-authoritative** answer?
10. Does the response contain any IP addresses of the authoritative nameservers? If so, list them.

---

## Part 3: DNS and Web Browsing

**Trace file**: `dns-web-browsing.pcapng`

11. Locate the first DNS query message when browsing to `www.ietf.org`. What is the destination IP address of this query? Is this the address of your local DNS resolver?
12. Examine the DNS query. What type is it? Does it contain any answers?
13. Examine the DNS response. How many answers are present? List the information in each answer (type, value, TTL).
14. Consider the subsequent TCP SYN packet that opens the connection to the web server. Does the destination IP address correspond to one of the IP addresses in the DNS response?
15. Examine the DNS query after some time has passed. Has the TTL of the cached record changed? What does this tell you about DNS caching?
16. How many different DNS queries and responses were generated by a single page load? What types of records were queried (A, AAAA, CNAME, etc.)?
