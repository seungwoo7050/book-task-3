# 01 접근 기록

이 문서는 정확한 작업 일지를 복원하려는 문서가 아니라, 현재 구현을 만든 핵심 선택을 다시 읽기 좋게 정리한 기록이다.

## 핵심 설계 선택
### 1. identifier는 PID 하위 16비트를 사용했다
- 이유: 프로세스 단위로 자신이 만든 Echo Reply를 식별하기 쉽다.
### 2. timeout 대기는 `select.select()`로 처리했다
- 이유: 소켓 읽기와 timeout을 단순하게 분리하면서 플랫폼 차이를 줄일 수 있다.
### 3. 응답 파싱 시 IP 헤더 길이를 동적으로 계산했다
- 이유: 옵션이 붙은 IP 헤더에서도 ICMP 위치를 정확히 찾기 위해서다.
### 4. 테스트를 deterministic/unit와 live/raw-socket으로 분리했다
- 이유: 권한 요구와 외부 네트워크 가변성 때문에 같은 층위의 테스트로 묶으면 관리가 어렵다.

## 보류하거나 버린 선택
- 운영체제 `ping` 명령을 래핑하는 방식: ICMP 패킷 구성과 checksum 학습 포인트가 사라진다.
- IP 헤더를 항상 20바이트라고 가정: 옵션이 있는 패킷을 만나면 바로 잘못된 파싱이 된다.

## 구현 순서 메모
- checksum 함수와 패킷 빌드부터 먼저 단위 테스트로 고정했다.
- 그다음 raw socket 송수신 루프와 timeout, 통계를 붙였다.
- 마지막에 live/manual 검증 경로를 문서화했다.
