# 02 디버그 로그

이 파일은 과거의 날짜 순서를 복원하려는 문서가 아니다. 백업 노트와 현재 테스트를 함께 보고, 지금도 재현하거나 설명할 수 있는 문제만 남긴다.

## 사례 1. checksum이 맞지 않아 응답이 오지 않음
- 증상: 패킷을 보냈지만 대상 호스트가 Echo Reply를 주지 않았다.
- 원인: ICMP checksum은 헤더와 payload 전체를 대상으로 계산해야 하며 바이트 정렬도 맞아야 한다.
- 수정: RFC 1071 방식으로 checksum 함수를 다시 맞추고 단위 테스트를 추가했다.
- 확인: `TestInternetChecksum`과 실제 ping 실행으로 확인했다.

## 사례 2. IP 헤더 길이를 20바이트로 고정해 응답 파싱이 어긋남
- 증상: 일부 응답에서 ICMP 타입과 시퀀스 번호가 이상하게 읽혔다.
- 원인: IPv4 헤더에는 옵션이 들어갈 수 있어 실제 길이는 `IHL`을 봐야 한다.
- 수정: 수신 버퍼에서 `IHL * 4`를 계산해 ICMP 시작 위치를 찾도록 수정했다.
- 확인: `TestPacketBuilding`과 응답 파싱 테스트, 수동 패킷 캡처로 확인했다.

## 사례 3. 전부 손실된 경우 RTT 요약 출력이 깨짐
- 증상: 성공 응답이 하나도 없는데도 평균 RTT를 계산하려다 예외가 나거나 의미 없는 숫자가 찍혔다.
- 원인: 성공 샘플이 비어 있는 경우를 별도로 처리하지 않았다.
- 수정: 응답이 한 건도 없으면 RTT 요약은 생략하고 손실률만 출력하게 정리했다.
- 확인: `test_ping_handles_total_packet_loss_without_rtt_summary`로 검증했다.
