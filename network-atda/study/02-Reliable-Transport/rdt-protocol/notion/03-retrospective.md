# 03 회고

## 이번 프로젝트에서 확실히 얻은 것
- 신뢰 전송은 checksum, ACK, 타이머, 순서 제어가 한꺼번에 맞아야 성립한다는 점을 이해했다.
- GBN이 왜 구현은 단순하지만 불필요한 재전송 비용을 치르는지도 실감했다.

## 아직 약한 부분
- 시뮬레이터 기반이라 실제 소켓 지연이나 운영체제 버퍼와의 상호작용은 보지 못한다.
- 성능 비교는 정성적 설명 위주이고 계량적 벤치마크는 부족하다.

## 포트폴리오로 키우려면
- 동일 메시지 집합에 대해 `rdt3`, `GBN`, `SR`의 재전송 횟수와 완료 시간을 표로 비교한다.
- 상태 전이 다이어그램과 로그 시각화를 추가해 프로토콜 설명 자료로 확장한다.
