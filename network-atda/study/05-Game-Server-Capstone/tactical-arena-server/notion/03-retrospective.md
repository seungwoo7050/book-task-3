# 03 회고

## 이번 프로젝트에서 확실히 얻은 것
- 작은 네트워크 과제에서 따로 배운 개념들이 실제 시스템에서는 동시에 등장한다는 사실을 체감했다.
- 프로토콜 설계, 동시성 제어, 상태 머신, persistence, 테스트 계층화가 한 프로젝트 안에서 연결되는 감각을 얻었다.

## 아직 약한 부분
- 보안과 운영 관점에서는 여전히 학습용 서버다. 인증, anti-cheat, 메트릭, 운영 배포가 없다.
- 실시간 보정, rollback netcode, 관전자 모드 같은 고급 기능은 아직 없다.

## 포트폴리오로 키우려면
- 프로토콜 다이어그램, 부하 테스트 그래프, reconnect 시연 영상을 함께 묶어 공개용 아키텍처 사례로 정리한다.
- 메트릭 수집과 관리자 대시보드를 붙여 운영 가능한 개발용 서버로 한 단계 확장한다.
