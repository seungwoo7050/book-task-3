# backend-spring 문서 지도

이 폴더는 저장소 공통 문서를 모아 둔 곳입니다. 개별 랩의 구현 세부보다, 이 레포를 어떤 순서로 읽고 어떤 기준으로 해석해야 하는지를 먼저 설명합니다.

## 먼저 읽을 문서

1. [repo-standards.md](repo-standards.md): `README`, `problem`, `spring`, `docs`, `notion`이 각각 어떤 역할을 맡는지 정의합니다.
2. [curriculum.md](curriculum.md): 전체 학습 순서와 각 랩이 무엇을 가르치는지 정리합니다.
3. [verification-report.md](verification-report.md): 마지막 실제 검증 실행 시점과 범위, 아직 증명하지 않은 부분을 기록합니다.
4. [publication-status.md](publication-status.md): 이 레포를 GitHub나 포트폴리오에서 어떻게 포지셔닝해야 하는지 정리합니다.

## 이 폴더가 답하는 질문

- 이 레포를 어떤 순서로 읽어야 하는가
- 프로젝트 README는 무엇을 먼저 드러내야 하는가
- 검증이 실제 실행된 사실인지, 아직 가정인지 어떻게 구분하는가
- `commerce-backend`와 `commerce-backend-v2`를 어떤 위계로 읽어야 하는가
