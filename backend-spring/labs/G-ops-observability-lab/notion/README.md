# G-ops-observability-lab 학습 노트 가이드

이 폴더는 운영성을 독립 학습 주제로 분리한 이유와 그 과정에서의 판단을 기록한 공개 학습 노트 세트다.

## 먼저 읽을 문서

1. [00-problem-framing.md](00-problem-framing.md): 왜 운영을 별도 랩으로 뒀는지
2. [05-development-timeline.md](05-development-timeline.md): Prometheus, health, logback 설정을 어떤 순서로 쌓았는지
3. [01-approach-log.md](01-approach-log.md): full observability stack 대신 기본기를 먼저 택한 이유

## 목적별 읽기

- 구현 depth와 lab 이름 사이의 괴리를 보려면 [02-debug-log.md](02-debug-log.md)
- health/readiness, structured logging, trace ID, Prometheus 개념을 복습하려면 [04-knowledge-index.md](04-knowledge-index.md)
- 다음 운영 학습 주제를 보려면 [03-retrospective.md](03-retrospective.md)
