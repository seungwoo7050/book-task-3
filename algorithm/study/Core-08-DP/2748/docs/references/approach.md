# 접근 정리 — BOJ 2748 (Fibonacci Number 2 (피보나치 수 2))

## 문제 요약

BOJ 2748는 입력 형식은 간결하지만, 실제 정답성은 세 가지 축에서 결정된다.
첫째, 입력을 문제 계약대로 안정적으로 해석해야 한다.
둘째, `반복 DP로 피보나치 수를 하향식 없이 누적 계산` 로직이 상태 불변식을 유지해야 한다.
셋째, 최종 출력 포맷이 채점기 기대와 정확히 일치해야 한다.
이번 문서의 목적은 단순 풀이 설명이 아니라, BOJ 2748의 의사결정 근거를 재사용 가능한 형태로 정리하는 것이다.

## 핵심 아이디어

1. 상태 정의: BOJ 2748에서 매 단계 이후 유지되어야 할 상태를 명시한다.
2. 전이 규칙: BOJ 2748 입력 단위를 처리할 때 `반복 DP로 피보나치 수를 하향식 없이 누적 계산`의 상태 갱신 순서를 고정한다.
3. 종료 조건: 반복/재귀를 언제 멈추고 무엇을 출력할지 분리한다.
4. 검증 루프: BOJ 2748 공식 테스트와 수동 케이스를 분리 기록해 전이 규칙이 실제 출력에서 유지되는지 확인한다.

이 문제에서 채택한 전략은 `반복 DP로 피보나치 수를 하향식 없이 누적 계산` 이다.
이 전략을 고른 이유는 BOJ 2748 제약에서 `반복 DP로 피보나치 수를 하향식 없이 누적 계산`가 구현 난이도와 검증 가능성의 균형을 가장 안정적으로 맞추기 때문이다.
특히 BOJ 2748처럼 입력 변형이 잦은 문제에서는, 상태를 단순하게 유지하는 설계가 디버깅 비용을 줄인다.

## 정당성(간단 증명)

### 상태 불변식

BOJ 2748의 각 단계가 끝났을 때, 현재 상태는 다음 단계를 수행하기에 충분한 정보를 잃지 않아야 한다.
즉, BOJ 2748에서는 앞 단계에서 계산한 핵심 값이 다음 전이에서 같은 의미로 재사용되어야 한다.
이 불변식이 BOJ 2748에서 무너지면 후속 계산 전체가 오염되므로, 구현 검토의 첫 체크포인트로 고정했다.

### 전이 타당성

전이 함수는 BOJ 2748 맥락에서 "현재 상태 + 새 입력"을 "다음 상태"로 바꾸는 계약(contract)으로 정의했다.
BOJ 2748 구현에서는 모든 분기에서 동일한 규칙을 적용하도록 구성해, 특정 입력에서만 다른 경로를 타는 비결정성을 제거했다.
수동 케이스에서 `작은 N에서 초기값(F0,F1)과 점화식 적용 순서를 검증한다.`를 집중 검증한 이유도 이 전이 타당성을 확인하기 위해서다.

### 종료 및 출력 정합성

종료 단계는 계산 완료 시점과 출력 직렬화 시점을 분리해 다룬다.
BOJ 2748는 정답 값만 맞아도 출력 포맷이 틀리면 오답이므로, 출력 규칙을 별도 검증 대상으로 취급했다.
공식 테스트 로그 `Results: 1/1 passed, 0 failed`는 BOJ 2748 종료 정합성이 유지되었음을 보여주는 1차 근거다.

## 복잡도

- 시간 복잡도: `O(N)`
- 공간 복잡도: `O(1)`
- 변수 정의: `N=피보나치 인덱스`

복잡도는 숫자 표기가 목적이 아니라, BOJ 2748의 입력 상한에서 해당 전략이 안전한지 설명하는 근거다.
BOJ 2748 문서에서는 복잡도 표기와 실행 로그를 나란히 배치해 이론 추정과 관측 결과가 어긋나지 않는지 확인한다.
