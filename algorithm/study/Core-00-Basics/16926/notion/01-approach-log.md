# 접근 과정 — 층 분해, 1차원 변환, 모듈러 회전

## 핵심 발상: 레이어를 1차원 리스트로 펼치기

2차원 배열에서 직사각형 테두리를 직접 회전시키는 건 인덱스 관리가 복잡하다.
하지만 테두리의 원소들을 시계 방향으로 순회하면서 1차원 리스트에 옮겨 담으면, 회전은 리스트 shift가 된다.

구체적으로:
1. 바깥에서 안쪽으로 각 레이어(k=0, 1, ...)를 순회한다
2. 레이어의 원소를 순서대로 `ring` 리스트에 추출한다 (위→오른쪽→아래→왼쪽)
3. 반시계 $R$번 회전 = 리스트를 왼쪽으로 $R$칸 shift = `ring[R:] + ring[:R]`
4. shift된 리스트를 같은 순서로 배열에 다시 써넣는다

이 방법의 장점은 **회전 로직이 리스트 슬라이싱 한 줄**로 끝난다는 것이다.
복잡한 2차원 인덱싱은 추출/삽입 단계에만 집중하면 된다.

## R이 10⁹일 때: 모듈러의 힘

레이어 하나의 원소 수를 $L$이라 하면, $L$번 회전하면 원래대로 돌아온다.
따라서 실제 회전 횟수는 `R % L`이다.

이 모듈러 처리를 빠뜨리면, $R=10^9$에서 즉시 시간 초과가 난다.
각 레이어마다 $L$이 다르다는 점에도 주의해야 한다.

예를 들어 4×4 배열이면:
- 바깥 레이어: $L = 4+4+4+4-4 = 12$개
- 안쪽 레이어: $L = 2+2+2+2-4 = 4$개

## 인덱싱의 네 방향

레이어 $k$에서 원소를 추출하는 순서:
```
1. 윗변:   arr[k][k .. M-1-k]        → 왼→오
2. 오른변: arr[k+1 .. N-1-k][M-1-k]  → 위→아래
3. 아랫변: arr[N-1-k][M-2-k .. k]    → 오→왼 (역순)
4. 왼변:   arr[N-2-k .. k+1][k]      → 아래→위 (역순)
```

코너를 중복 포함하지 않도록 범위 경계를 주의해야 한다.
이 부분이 실수가 가장 잘 나는 곳이다.

## 복잡도 정리

| 항목 | 값 |
|------|-----|
| 시간 | $O(N \times M)$ — 각 원소를 정확히 한 번 추출/삽입 |
| 공간 | $O(N \times M)$ — ring 리스트 크기의 합이 전체 원소 수 |

$N, M \le 300$이므로 최대 90,000개 원소. Python에서도 무리 없다.
