# 문제 프레이밍

> 프로젝트: 팰린드롬인지 확인하기
> 아래 내용은 `notion-archive/00-problem-framing.md`에 남아 있던 기존 학습 기록을 현재 공개 노트 기준으로 다듬은 버전이다.

## 문제를 처음 읽었을 때

BOJ 10988은 단어 하나를 입력받아서 회문(palindrome)인지 판별하는 문제다.
처음 읽었을 때 드는 생각은 "이건 너무 쉬운 거 아닌가?"였다.

그런데 가만히 생각해 보면, "쉽다"는 감각 자체가 이미 문자열을 뒤집는 연산이 머릿속에 있기 때문이다.
프로그래밍을 처음 배우는 사람에게 "문자열을 뒤집어서 원본과 비교하라"는 발상이 자연스럽지 않을 수 있다.

이 문제의 진짜 가치는 **"비교 기반 판별"이라는 패턴을 처음 체화하는 데 있다.**

## 문제의 핵심 구조

- **입력**: 소문자 영어 단어 하나 ($1 \le |word| \le 100$)
- **출력**: 회문이면 `1`, 아니면 `0`
- **제약**: 별도의 자료 구조 없이 문자열 연산만으로 풀 수 있음

여기서 눈에 띄는 점은, 출력이 `Yes`/`No`가 아니라 `1`/`0`이라는 것이다.
사소해 보이지만 채점기는 정확한 문자열 매칭을 하기 때문에, 이런 출력 형식을 놓치면 바로 오답이 된다.

## 왜 이 문제를 골랐는가

Core-00-Basics 트랙의 첫 번째 문제이기 때문이다.
알고리즘 학습 커리큘럼을 설계할 때, Bronze 난이도에서 "입력 → 처리 → 출력" 파이프라인을 가장 깔끔하게 연습할 수 있는 문제를 찾았고, 10988이 그 역할에 적합했다.

CLRS 기준으로는 Ch 1~3(알고리즘 소개, 점근적 분석)에 해당한다.
문제 자체는 단순하지만, "왜 이 풀이가 O(n)인가?"를 설명하려면 Ch 3의 점근 표기법을 알아야 한다.

## 지금 이 프로젝트에서 먼저 고정할 것

- `팰린드롬인지 확인하기`에서 실제로 확인하려는 학습 목표는 `작은 입력을 안정적으로 읽고, 조건 분기를 코드와 문서로 함께 정리하는 감각`이다.
- 문제를 읽을 때는 "정답을 맞힌다"보다 어떤 상태와 규칙을 끝까지 유지해야 하는지를 먼저 적어 두는 편이 좋다.
- 비슷한 유형을 다시 만났을 때 바로 적용할 수 있는 작은 판단 기준을 남겨 두는 것이 핵심이었다.
- 학습자 입장에서는 `05-development-timeline.md`를 같이 열어 두면 실제 재현 순서와 문제 해석이 어떻게 맞물리는지 더 잘 보인다.
