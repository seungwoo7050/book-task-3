# 회고 — DP를 처음 "체감"한 문제

## 이 문제가 남긴 것

### DP의 핵심을 한 문장으로 이해한 순간

"이전 결과를 배열에 저장해 두고, 새 원소를 처리할 때 그걸 다시 꺼내 쓴다."
이 한 문장이 DP의 본질이라는 걸, 코드를 직접 짜보고 나서야 체감했다.

교과서에서 "최적 부분 구조"니 "중복 부분 문제"니 하는 말을 읽었을 때는 감이 안 왔다.
하지만 `dp[i] = max(dp[i], dp[j] + 1)`이라는 한 줄을 직접 타이핑하고 나서,
"아, `dp[j]`가 이미 최적 답이라서 여기에 1을 더하면 되는 거구나" 싶었다.

### Python/C++ 동시 구현의 가치

이 문제에서 처음으로 같은 알고리즘을 두 언어로 구현해 봤다.
Python은 `max(dp)`로 한 줄이면 되지만, C++에서는 `*max_element(dp.begin(), dp.end())`를 써야 한다.
알고리즘은 같아도 언어 관용구가 다르다는 걸 여기서 느꼈다.

교차 검증을 통해 두 구현이 같은 답을 내는 것을 확인했고, 이 패턴은 이후 프로젝트에서 계속 유지했다.

### O(N²)이 "나쁜 것"이 아니라는 감각

$N \le 1\,000$이면 $O(N^2)$이 충분하다.
무조건 최적 복잡도를 추구하는 것보다, **제약 조건에 맞는 가장 간단한 방법**을 택하는 게 더 현명하다는 걸 배웠다.
$O(N \log N)$ 방식은 Core-08-DP에서 다시 만날 것이다.
