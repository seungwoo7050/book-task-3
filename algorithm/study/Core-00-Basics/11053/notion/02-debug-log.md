## 주의했던 포인트

이 문제 자체가 DP 입문에 가깝기 때문에, 알고리즘 로직보다는 "DP를 처음 짤 때 흔히 하는 실수"를 점검하는 데 시간을 썼다.

### 초기값 실수

처음에 `dp = [0] * n`으로 초기화하면, 원소가 하나뿐인 경우 답이 `0`이 나온다.
모든 위치는 최소한 자기 자신 하나로 길이 1의 LIS를 이루므로, `dp = [1] * n`이 맞다.

### 엄격 증가 vs 비엄격 증가

`a[j] < a[i]`와 `a[j] <= a[i]`는 이 문제에서 결정적으로 다른 결과를 낸다.
`[3, 3, 3, 3, 3]`에 대해:
- `<` 사용: LIS = 1 (정답)
- `<=` 사용: LIS = 5 (오답)

문제가 "strictly increasing"을 명시하고 있으므로 `<`만 사용해야 한다.

### 답 집계

처음에 `print(dp[-1])`로 써서 "마지막 위치에서 끝나는 LIS"만 출력할 뻔했다.
하지만 LIS가 수열의 중간 어딘가에서 끝날 수도 있다.
예: `[1, 5, 2, 3]`에서 LIS는 `[1, 2, 3]`이고 `dp = [1, 2, 2, 3]`. 답은 `dp[-1] = 3`이 맞지만,
`[1, 5, 2, 3, 1]`이면 `dp = [1, 2, 2, 3, 1]`이고 `dp[-1] = 1`인데 정답은 3이다.
`max(dp)`를 쓰면 이런 케이스를 놓치지 않는다.
