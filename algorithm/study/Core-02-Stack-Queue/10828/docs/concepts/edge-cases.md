# 경계 사례 점검 — BOJ 10828 Stack

## 1. 빈 스택에서 pop

```
1
pop
```
→ `-1` 출력. Underflow 시 에러가 아닌 `-1`을 반환해야 한다.

## 2. 빈 스택에서 top

```
1
top
```
→ `-1` 출력. `pop`과 동일한 underflow 처리.

## 3. push 직후 pop

```
2
push 5
pop
```
→ `5` 출력. 가장 기본적인 LIFO 동작 확인.

## 4. 연속 pop으로 스택 비우기

```
5
push 1
push 2
pop
pop
pop
```
→ `2`, `1`, `-1`. 마지막 pop은 빈 스택.

## 5. size와 empty 조합

```
3
size
empty
push 1
```
→ `0`, `1`. 출력 명령만 있는 경우도 정상 처리.

## 6. N = 1 (최소 입력)

```
1
empty
```
→ `1`. 명령이 하나뿐인 경우.

## 7. N = 10,000 (최대 입력)

모든 명령이 `push`일 때 출력 없음. 출력 버퍼에 아무것도 쌓이지 않아야 한다.

## 핵심 주의점

- `pop`과 `top` 모두 빈 스택 검사 필수
- 출력이 없는 명령(`push`)과 있는 명령의 구분
- 대량 I/O 시 `sys.stdin.readline` + 출력 버퍼링 필수
