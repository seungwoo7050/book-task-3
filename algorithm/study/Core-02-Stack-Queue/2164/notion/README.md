# notion/ 안내

## 이 폴더의 역할

이 폴더는 README와 `docs/`보다 긴 과정 기록을 담는 공개 학습 노트 모음이다. 프로젝트를 처음 이해하는 엔트리포인트는 아니며, 선택 이유와 디버깅 이력을 길게 남길 때만 내려온다.

## 노트 인덱스

- `00-problem-framing.md`: 문제를 내 말로 다시 정리
- `01-approach-log.md`: 채택 전략과 대안 비교
- `02-debug-log.md`: 실패 사례와 수정 기록
- `03-retrospective.md`: 이번 문제에서 얻은 교훈
- `04-knowledge-index.md`: 다시 꺼내 쓸 개념과 참고 자료
- `05-development-timeline.md`: 구현과 검증을 다시 밟는 재현 문서

## 읽는 순서

1. 상위 프로젝트 README에서 문제, 답, 검증 명령을 먼저 확인한다.
2. 재현 흐름이 필요하면 `05-development-timeline.md`부터 연다.
3. 선택 이유와 실패 기록이 궁금하면 `01`, `02`를 읽는다.
4. 마지막에 `03`, `04`로 학습 내용을 정리한다.

## 경계

- 현재 공개판은 `notion/`에 유지하고, 이전 버전 메모는 `../notion-archive/`에 둔다.
- README와 `docs/`에 이미 나온 길찾기 설명을 여기서 다시 반복하지 않는다.
