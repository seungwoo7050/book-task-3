# 경계 사례 점검 — BOJ 5430 AC

## 1. 빈 배열에 D 적용

```
입력: p="D", n=0, arr="[]"
출력: error
```
즉시 error. deque가 비어있을 때 D 실행.

## 2. R만 반복

```
입력: p="RRR", n=3, arr="[1,2,3]"
출력: [3,2,1]
```
홀수 번 R → 한 번 뒤집은 것과 동일.

## 3. 원소가 하나일 때 R

```
입력: p="R", n=1, arr="[42]"
출력: [42]
```
뒤집어도 동일.

## 4. D로 모든 원소 삭제 후 빈 배열 출력

```
입력: p="DD", n=2, arr="[1,2]"
출력: []
```
error가 아닌 빈 배열 `[]` 출력.

## 5. D로 모든 원소 삭제 후 D 추가

```
입력: p="DDD", n=2, arr="[1,2]"
출력: error
```
세 번째 D에서 빈 배열 → error.

## 6. R과 D 교차

```
입력: p="RDD", n=4, arr="[1,2,3,4]"
출력: [2,1]
```
R 후 D: 뒤에서 pop (4 제거), 또 D: 뒤에서 pop (3 제거)
결과: [1,2] → is_reversed이므로 출력 시 [2,1].

## 7. 큰 입력 (n = 100,000, |p| = 100,000)

Lazy reversal 없이 순진하게 뒤집으면 TLE.
$O(|p| + n)$이어야 통과.

## 8. 빈 배열 파싱

```
n=0, arr="[]"
```
`"[]"[1:-1]` → `""`, `"".split(',')` → `['']`
→ 빈 문자열이 들어가므로 `n==0` 조건으로 빈 deque 직접 생성 필요.

## 핵심 주의점

- `n=0`일 때 파싱 우회 필수
- D로 비울 수 있으므로 매 D마다 empty 체크
- 최종 출력 형식: `[x1,x2,...,xn]` (공백 없음)
- error 발생 시 즉시 break하고 남은 명령 무시
