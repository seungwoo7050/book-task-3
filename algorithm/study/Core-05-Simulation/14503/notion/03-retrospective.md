# BOJ 14503 — 회고

## 배운 것

시뮬레이션 문제에서 가장 어려운 것은 "알고리즘"이 아니라 "규칙의 정확한 해석"이다. 이 문제에서 "왼쪽 회전 후 전진"과 "방향 유지한 채 후진"의 차이를 놓치면 완전히 다른 결과가 나온다.

## 방향 관리의 패턴

`(d + 3) % 4`로 왼쪽 회전, `(d + 2) % 4`로 후진 방향. 이 패턴은 격자 위 방향 관리가 필요한 모든 시뮬레이션에서 재사용된다. 방향 벡터 `dr, dc`와 함께 하나의 "방향 관리 키트"를 구성한다.

## cleaned 배열 분리의 중요성

grid를 직접 변경하는 대신 별도의 `cleaned` 배열을 사용한 것이 정확성에 기여했다. 후진 시 "벽인지 빈칸인지" 판별은 원본 grid를 기준으로 해야 하고, "청소했는지" 판별은 별도 배열이 필요하다.

## 만약 다시 풀다면

C++에서 `pair<int,int>`와 enum 방향으로 더 구조화된 구현을 시도할 것이다.
