# 회고

> 프로젝트: 톱니바퀴
> 아래 내용은 `notion-archive/03-retrospective.md`에 남아 있던 기존 학습 기록을 현재 공개 노트 기준으로 다듬은 버전이다.

## 배운 것

시뮬레이션 문제의 핵심 교훈: **상태 변경의 순서가 결과를 결정한다**. "먼저 판단하고, 나중에 적용"하는 패턴은 톱니바퀴뿐 아니라, 게임 시뮬레이션이나 물리 엔진에서도 통용되는 원칙이다.

## deque 활용의 자신감

deque의 `appendleft`/`pop`과 `append`/`popleft` 조합으로 원형 배열의 회전을 깔끔하게 구현할 수 있었다. Core-02에서 deque를 큐로만 사용했다면, 이 문제에서는 원형 자료구조로의 활용을 확인했다.

## 전파 패턴의 일반화

이 문제의 "한 방향으로 전파, 중간에 조건 불충족 시 멈춤" 패턴은 도미노, 충격파, 네트워크 전파 등 다양한 시뮬레이션에서 나타난다. `for` 루프 + 조건 `break`의 조합이 이 패턴의 기본 형태.

## 이번 프로젝트가 남긴 기준

- `톱니바퀴`를 통해 `복잡한 설명을 작은 상태 전이 규칙으로 나누어 구현하는 연습`를 코드와 문장으로 같이 설명하는 감각을 다시 확인했다.
- 짧은 정답 코드보다, 왜 이 선택이 안전했는지 적어 둔 문장이 나중에 더 오래 남는다.
- 비슷한 유형을 다시 만났을 때 바로 적용할 수 있는 작은 판단 기준을 남겨 두는 것이 핵심이었다.
