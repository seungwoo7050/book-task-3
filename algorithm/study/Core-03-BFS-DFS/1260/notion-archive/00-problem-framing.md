# BOJ 1260 — DFS와 BFS: 그래프 탐색의 두 얼굴

## 첫인상

그래프 탐색 문제를 처음 접할 때, DFS와 BFS는 "둘 다 그래프를 탐색한다"는 점에서 비슷해 보인다. 하지만 1260번은 같은 그래프에 대해 DFS와 BFS를 **동시에** 구현하고, 방문 순서를 비교 출력하라고 요구한다. 이 문제가 좋은 출발점인 이유는, 두 탐색의 차이를 **같은 입력 위에서** 눈으로 확인할 수 있기 때문이다.

## 문제 구조 파악

- 정점 $N$개, 간선 $M$개의 **무방향 그래프**
- 시작 정점 $V$에서 DFS 방문 순서 출력
- 같은 그래프에서 BFS 방문 순서 출력
- 방문할 수 있는 정점이 여러 개면 **번호가 작은 것** 먼저

"번호가 작은 것 먼저"라는 조건이 핵심이다. 인접 리스트를 정렬하지 않으면 답이 달라진다.

## 왜 이 문제를 선택했는가

Core-03 트랙의 첫 문제로, DFS와 BFS의 기본 구현 패턴을 확립하기 위해 선택했다. CLRS Ch 22.2-22.3의 의사코드를 실제 코드로 옮기는 연습이 목적이다. 이후 24479(DFS 방문 순서 기록)와 7576(다중 소스 BFS)으로 확장하는 출발점이다.

## 난이도 평가

Silver 등급. 알고리즘 자체는 교과서 수준이지만, 두 탐색을 한 프로그램에서 깔끔하게 분리 구현하는 것과, 인접 리스트 정렬 조건을 놓치지 않는 세심함이 요구된다.
