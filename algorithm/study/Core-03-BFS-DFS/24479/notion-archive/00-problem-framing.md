# BOJ 24479 — 알고리즘 수업: DFS 방문 순서

## 첫인상

1260번에서 DFS 결과를 "방문 순서대로 출력"했다면, 이 문제는 한 단계 더 나아간다. 각 정점이 **몇 번째로** 방문되었는지를 기록해야 한다. 단순한 차이처럼 보이지만, 구현에서는 "카운터를 어떻게 관리할 것인가"라는 새로운 설계 결정이 필요하다.

## 문제 구조 파악

- $N$개 정점, $M$개 간선의 무방향 그래프
- 시작 정점 $R$에서 DFS 수행
- 이웃 중 **번호가 작은 정점** 먼저 방문
- 각 정점의 방문 순서 출력 (방문하지 않았으면 0)

핵심은 "방문 순서"를 단순 리스트가 아닌 **배열 인덱싱**으로 기록하는 것이다.

## 왜 이 문제를 선택했는가

1260의 DFS를 경험한 후, 방문 순서를 '기록'하는 변형을 훈련하기 위해 선택했다. 백준의 "알고리즘 수업" 시리즈는 교과서 의사코드와 실제 구현의 갭을 메우는 데 적합하다.

## 난이도 평가

Bronze 등급이지만, $N$이 최대 100,000까지 가능해서 재귀 한도 관리가 필요하다. 알고리즘 자체보다 Python 런타임 제약(재귀 한도, I/O 속도)을 다루는 실전 경험이 핵심이다.
