# BOJ 24479 — 회고: DFS의 변주

## 배운 것

같은 DFS라도 "무엇을 기록하느냐"에 따라 구현이 달라진다. 1260에서는 방문 순서 리스트, 이 문제에서는 정점별 방문 번호. CLRS의 DFS-VISIT에서 `u.d`(발견 시각)를 기록하는 것과 동일한 패턴이다.

## Python 재귀의 한계

$N = 100,000$에서 재귀 DFS를 쓰는 것은 Python에서는 사실 위험한 선택이다. 재귀 한도를 올려도 스택 오버플로우가 발생할 수 있다. 이 문제는 Bronze라서 통과하지만, 더 큰 입력이 필요한 문제에서는 **명시적 스택** 방식으로 전환해야 한다.

## 카운터 관리의 교훈

`order = [0]`이라는 리스트 래핑 패턴은 Python 클로저의 특성을 활용한 관용적 해법이다. Python 3에서는 `nonlocal` 키워드도 가능하지만, 리스트 래핑이 더 범용적이다.

## 다음 단계로의 연결

이 문제의 "방문 순서 기록" 패턴은 위상 정렬(DFS의 finishing time)이나 SCC(Tarjan 알고리즘의 dfn/low)에서 재등장한다. Core-0D-MST-Topo의 2252(위상 정렬)와 직접 연결된다.
