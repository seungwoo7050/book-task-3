# BOJ 7576 — 접근 과정: 격자 위의 다중 소스 BFS

## 그래프 모델링

격자를 그래프로 변환하는 첫 번째 과제. 각 칸이 정점, 상하좌우가 간선이다. 인접 리스트를 명시적으로 만들 필요 없이, 방향 벡터 `dx, dy`로 이웃을 계산한다.

```python
dx = [0, 0, 1, -1]
dy = [1, -1, 0, 0]
```

## 다중 소스 BFS 전략

단일 소스 BFS와의 유일한 차이: **초기 큐에 소스가 하나가 아니라 여러 개**라는 것. 입력을 읽으면서 `grid[i][j] == 1`인 모든 칸을 `(i, j, 0)` 형태로 큐에 넣는다.

```python
for i in range(N):
    row = list(map(int, input().split()))
    grid.append(row)
    for j in range(M):
        if row[j] == 1:
            q.append((i, j, 0))
```

이는 CLRS의 BFS에서 가상 슈퍼 소스 $s^*$를 추가한 것과 동일하다. $s^*$에서 모든 초기 익은 토마토로 가중치 0 간선을 연결한 셈.

## 방문 처리의 간소화

별도의 `visited` 배열 없이, **grid 자체를 방문 배열로 활용**했다. `grid[nx][ny] == 0`인 경우에만 진행하고, 방문 시 `grid[nx][ny] = 1`로 변경한다. 이렇게 하면 메모리를 절약하고 이중 체크를 피할 수 있다.

## 최대 일수 추적

BFS 레벨이 곧 날짜다. 큐에서 꺼낼 때의 `day`가 해당 칸이 익는 데 걸린 날짜이고, BFS가 끝났을 때 최대 `day`가 정답이다.

## 불가능 판별

BFS 종료 후 grid를 순회하여 `0`이 남아 있으면 도달 불가능한 안 익은 토마토가 있다는 뜻이므로 `-1`을 출력한다.

## 대안으로 고려한 것

- **날짜별 큐 분리**: `day`를 튜플에 포함하지 않고, 레벨별로 큐를 교체하는 방식. 메모리는 덜 쓰지만 코드가 복잡해짐
- **visited 별도 배열**: grid 변경 없이 별도 배열 사용. 정당하지만 grid가 이미 변경 가능하므로 불필요
