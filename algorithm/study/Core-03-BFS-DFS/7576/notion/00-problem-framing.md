# 문제 프레이밍

> 프로젝트: 토마토
> 아래 내용은 `notion-archive/00-problem-framing.md`에 남아 있던 기존 학습 기록을 현재 공개 노트 기준으로 다듬은 버전이다.

## 첫인상

"토마토가 익는다"는 일상적인 비유 뒤에, 이 문제는 **다중 소스 BFS**라는 핵심 기법을 요구한다. 1260에서 배운 단일 시작점 BFS를 "여러 시작점에서 동시에 출발"하는 형태로 확장하는 것이 핵심이다.

## 문제 구조 파악

- $N \times M$ 격자 ($2 \leq M, N \leq 1000$)
- `1`: 익은 토마토, `0`: 안 익은 토마토, `-1`: 빈 칸
- 매일 익은 토마토의 상하좌우에 있는 안 익은 토마토가 익는다
- 모든 토마토가 익는 최소 일수 출력 (불가능하면 -1)

핵심 관찰: "동시에 여러 곳에서 퍼진다" → 모든 초기 익은 토마토를 **동시에** 큐에 넣고 BFS를 시작하면 된다.

## 왜 이 문제를 선택했는가

Core-03의 Gold 문제로, BFS의 실전 응용을 체험하기 위해 선택했다. 1260의 기본 BFS에서 출발해, 격자 그래프 + 다중 소스라는 두 가지 확장을 동시에 다룬다. CLRS Ch 22.2의 BFS에 "가상 슈퍼 소스"를 추가하는 아이디어와 정확히 대응한다.

## 난이도 평가

Gold 등급. 알고리즘 아이디어 자체(다중 소스 BFS)는 간단하지만, 격자 탐색의 디테일(범위 검사, 방문 처리, 불가능 판별)과 성능 최적화(C++ 구현 포함)가 요구된다.

## 지금 이 프로젝트에서 먼저 고정할 것

- `토마토`에서 실제로 확인하려는 학습 목표는 `그래프 표현을 고르고 방문 순서를 안정적으로 제어하는 연습`이다.
- 문제를 읽을 때는 "정답을 맞힌다"보다 어떤 상태와 규칙을 끝까지 유지해야 하는지를 먼저 적어 두는 편이 좋다.
- 비슷한 유형을 다시 만났을 때 바로 적용할 수 있는 작은 판단 기준을 남겨 두는 것이 핵심이었다.
- 학습자 입장에서는 `05-development-timeline.md`를 같이 열어 두면 실제 재현 순서와 문제 해석이 어떻게 맞물리는지 더 잘 보인다.
