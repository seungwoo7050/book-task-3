# BOJ 7576 — 디버깅 기록

## 함정 1: M과 N의 순서

**증상**: 첫 제출에서 인덱스 에러 발생

**원인**: 입력이 `M N` 순서(열 먼저, 행 나중)인데, 직관적으로 `N M`(행 먼저)로 읽어서 격자 크기가 뒤바뀜.

**해결**: 문제를 다시 읽고 `M, N = map(int, input().split())`로 수정. grid는 `N`행 `M`열, 범위 검사도 `0 <= nx < N`과 `0 <= ny < M`.

## 함정 2: 방문 표시 시점

**증상**: 같은 칸이 여러 번 큐에 들어가 메모리 초과

**원인**: "큐에서 꺼낼 때" 방문 표시를 하면, 여러 이웃이 같은 칸을 큐에 넣을 수 있다.

**해결**: "큐에 넣을 때" 즉시 `grid[nx][ny] = 1`로 익힘 처리. BFS의 정석 패턴이지만 실수하기 쉽다.

## 함정 3: 이미 모두 익은 경우

**증상**: 초기부터 안 익은 토마토가 없는 입력에서 0이 아닌 값 출력

**원인**: BFS가 한 번도 안 돌면 `ans`가 초기값 그대로. 하지만 정답은 0이어야 한다.

**해결**: `ans = 0`으로 초기화. BFS가 한 번도 확장하지 않아도 0이 그대로 출력된다.
