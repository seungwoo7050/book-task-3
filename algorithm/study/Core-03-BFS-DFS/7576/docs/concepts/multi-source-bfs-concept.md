# 다중 소스 BFS(Multi-Source BFS) 개념 정리

## 정의

**다중 소스 BFS**는 여러 시작점에서 동시에 BFS를 수행하는 기법이다.
단일 소스 BFS를 확장하여, 모든 소스를 큐에 초기 삽입한다.

## CLRS 연결 (Ch 22.2)

CLRS Ch 22.2의 BFS는 단일 소스 $s$에서 시작:
```
BFS(G, s)
  s.d = 0
  Q = {s}
  while Q ≠ ∅
    u = DEQUEUE(Q)
    for each v ∈ G.Adj[u]
      if v.color == WHITE
        v.d = u.d + 1
        ENQUEUE(Q, v)
```

다중 소스 확장: 가상 슈퍼 소스 $s^*$를 추가하여
$s^* \to s_1, s^* \to s_2, \ldots$를 가중치 0으로 연결한 것과 동일.

## 토마토 문제에서의 적용

### 모델링
- 격자의 각 칸 = 그래프의 정점
- 상하좌우 인접 = 간선
- 초기 익은 토마토(1) = 소스
- 안 익은 토마토(0) = 목적지
- 빈 칸(-1) = 장애물

### 동작
1. 모든 초기 익은 토마토를 큐에 삽입 (day = 0)
2. BFS 웨이브가 매일 한 칸씩 확장
3. 최대 도달 거리 = 최소 일수
4. BFS 후 `0`이 남으면 → `-1`

## 그리드 BFS 패턴

```python
dx = [0, 0, 1, -1]
dy = [1, -1, 0, 0]

while q:
    x, y, day = q.popleft()
    for k in range(4):
        nx, ny = x + dx[k], y + dy[k]
        if 0 <= nx < N and 0 <= ny < M and grid[nx][ny] == 0:
            grid[nx][ny] = 1  # visited 역할
            q.append((nx, ny, day + 1))
```

### 핵심 최적화
- `visited` 배열 대신 격자 값 직접 수정 → 메모리 절약
- `day` 값을 큐 원소에 포함 → 별도 거리 배열 불필요
