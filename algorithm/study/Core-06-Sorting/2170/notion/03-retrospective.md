# BOJ 2170 — 회고

## 배운 것

"정렬 후 스위프"는 정렬을 단순한 출력 정리가 아닌 **알고리즘의 핵심 단계**로 활용하는 패턴이다. 정렬이 없으면 모든 쌍을 비교해야 해서 $O(N^2)$이지만, 정렬 후 한 번의 선형 스위프로 $O(N \log N)$에 해결된다.

## 구간 합치기 패턴의 범용성

이 패턴은 다양한 문제에서 나타난다:
- 스케줄링 (회의실 배정 — Core-09 BOJ 1931)
- 기하 (직사각형 넓이 합)
- 이벤트 처리 (서버 로그 분석)

"겹치는 것을 한 번만 세라"는 요구사항이 나오면 구간 합치기를 먼저 떠올리면 좋다.

## Python의 한계

$N = 10^6$에서 Python은 빡빡하다. C++ 구현을 유지한 이유. Gold 등급 이상에서 대량 입력 + 정렬 조합은 C++이 안전하다.
