# BOJ 9663 — 회고: 가지치기의 위력

## 배운 것

N-Queen의 탐색 트리는 깊이가 $N$, 각 노드에서 $N$개의 분기가 있으므로 최대 $N^N$개의 노드가 있다. 하지만 열/대각선 검사로 가지치기하면 실제 방문 노드 수는 극적으로 줄어든다. $N = 8$에서 $8^8 = 16,777,216$ 대신 수만 개 수준.

이것이 백트래킹의 핵심 가치다: **전체를 탐색하지 않고도 전체 답을 찾는다**.

## 3개 배열로 충돌 검사의 우아함

`col`, `diag1`, `diag2` 세 boolean 배열로 $O(1)$ 충돌 검사를 구현한 것이 이 풀이의 핵심. 15649의 `used` 배열을 3차원으로 확장한 것이다. 이 패턴은 다른 제약 조건 만족 문제(CSP)에도 적용 가능하다.

## Python vs C++ 성능 차이

$N = 15$에서 Python은 시간 초과, C++은 1초 미만. 재귀 호출이 많은 백트래킹에서 Python의 함수 호출 오버헤드가 치명적이다. Gold 이상 백트래킹 문제는 C++ 구현을 병행하는 것이 현실적.
