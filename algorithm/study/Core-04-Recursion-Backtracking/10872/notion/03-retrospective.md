# BOJ 10872 — 회고: 재귀의 본질

## 배운 것

팩토리얼은 재귀의 "hello world"다. 하지만 이 단순한 문제에서 재귀의 두 필수 요소를 확인했다:

1. **기저 조건**: 재귀를 멈추는 조건. 없으면 무한 루프.
2. **문제 축소**: 매 호출마다 $n$이 1씩 줄어 기저에 반드시 도달.

## 백트래킹으로의 연결

팩토리얼의 재귀 구조는 단방향이다 — 내려갔다 올라오면 끝. 하지만 15649(순열)에서는 "선택하고 → 재귀하고 → 되돌리는" 패턴이 추가된다. 이 되돌림(backtrack)이 팩토리얼에는 없는 새로운 차원이다.

## 만약 다시 풀다면

반복문 구현과 재귀 구현의 성능을 비교해 볼 것이다. $N \leq 12$에서는 차이가 없지만, $N$이 커지면 재귀의 스택 오버헤드가 드러난다.
