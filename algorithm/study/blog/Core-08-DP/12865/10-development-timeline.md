# BOJ 12865 — 개발 타임라인 (전반)

## Phase 1
### Session 1
- 목표: 0/1 배낭 문제를 구현한다. N개의 물건 각각을 넣거나 빼서 무게 한도 K 이내에서 최대 가치를 구한다.
- 진행: 처음엔 2차원 DP 표를 그렸다. `dp[i][w]` = "i번째 물건까지 고려, 총 무게 w일 때 최대 가치". 그런데 N과 K가 각각 최대 100, 100000이라 2차원 배열이 가능하지만 메모리를 아끼고 싶었다.
- 이슈: 1차원으로 압축하면 같은 물건을 두 번 쓰는 버그가 생길 수 있다는 걸 들은 적이 있다. 어떤 순서로 순회해야 하지?
- 판단: 역순 순회(`for j in range(K, w-1, -1)`)를 하면 `dp[j-w]`가 아직 이번 물건을 포함하지 않은 값이라 0/1 성질이 보존된다. 이게 이 문제의 핵심 포인트다.

### Session 2
- 목표: 1차원 DP를 구현하고 역순 순회가 정말 안전한지 확인한다.

이 시점의 핵심 코드:

```python
dp = [0] * (K + 1)

for _ in range(N):
    w, v = map(int, input().split())
    for j in range(K, w - 1, -1):
        dp[j] = max(dp[j], dp[j - w] + v)
```

만약 `range(w, K+1)` 정순으로 돌면 어떻게 되나? 같은 물건이 `dp[w]`를 갱신하고, 그 값이 `dp[2w]` 갱신에 또 쓰여서 같은 물건을 여러 번 넣는 셈이 된다. 역순이면 큰 쪽 먼저 갱신하니까 작은 쪽은 아직 "이전 라운드" 값이라 안전하다.
