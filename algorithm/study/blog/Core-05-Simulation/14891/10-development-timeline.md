# BOJ 14891 — 개발 타임라인 (전반)

## Phase 1
### Session 1
- 목표: 4개의 톱니바퀴가 맞물려 있을 때, 하나를 돌리면 인접한 톱니바퀴도 연쇄적으로 회전하는 시뮬레이션을 구현한다.
- 진행: 처음엔 "돌리는 즉시 옆 기어도 돌린다"고 생각했는데, 그러면 돌아간 후의 맞물림 상태로 다음 기어를 판정하게 되어 틀린다.
- 이슈: 핵심은 "모든 기어의 회전 방향을 먼저 결정한 뒤, 동시에 회전을 적용"하는 것이다. 판정과 실행을 분리해야 한다.
- 판단: dirs 배열에 4개 기어의 회전 방향을 먼저 채우고, 마지막에 한 번에 적용한다.

### Session 2
- 목표: 전파 로직과 회전을 구현한다.

이 시점의 핵심 코드:

```python
dirs = [0] * 4
dirs[num] = d

for i in range(num - 1, -1, -1):
    if gears[i][2] != gears[i + 1][6]:
        dirs[i] = -dirs[i + 1]
    else:
        break

for i in range(num + 1, 4):
    if gears[i][6] != gears[i - 1][2]:
        dirs[i] = -dirs[i - 1]
    else:
        break
```

톱니바퀴의 인덱스 2는 오른쪽 맞물림 부분, 인덱스 6은 왼쪽 맞물림 부분이다. 처음엔 이 인덱스를 헷갈려서 2와 6을 바꿨다가 틀렸다. 문제에서 0번 인덱스가 12시 방향이고 시계 방향으로 번호를 매기니까, 3시 방향이 2, 9시 방향이 6이다.

- 이슈: 전파가 중간에 "같은 극"이면 멈춰야 한다. break를 빼먹으면 멀리 있는 기어까지 잘못 전파된다.
