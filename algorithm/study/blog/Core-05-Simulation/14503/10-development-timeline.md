# BOJ 14503 — 개발 타임라인 (전반)

## Phase 1
### Session 1
- 목표: 로봇 청소기의 동작 규칙을 코드로 옮긴다.
- 진행: 문제 설명이 길어서 처음엔 규칙을 정리하는 데 시간이 걸렸다. 핵심은 세 단계다. (1) 현재 칸 청소, (2) 왼쪽으로 회전하며 4방향 안 청소한 곳 탐색, (3) 네 방향 다 막히면 뒤로 이동하거나 종료.
- 이슈: "왼쪽으로 회전"이 방향 인코딩에서 `(d + 3) % 4`라는 걸 바로 떠올리기가 쉽지 않았다. 0=북, 1=동, 2=남, 3=서 순서에서 왼쪽은 `d - 1`인데, 음수를 피하려면 `(d + 3) % 4`로 쓴다.
- 판단: 방향 배열 `dr, dc`를 먼저 고정하고 나서 회전 로직을 따로 검증했다.

### Session 2
- 목표: 구현을 완성하고 예제를 확인한다.

이 시점의 핵심 코드:

```python
for _ in range(4):
    d = (d + 3) % 4  # 왼쪽으로 회전
    nr, nc = r + dr[d], c + dc[d]
    if 0 <= nr < N and 0 <= nc < M and grid[nr][nc] == 0 and not cleaned[nr][nc]:
        r, c = nr, nc
        found = True
        break
```

이 for 루프가 "최대 4번 왼쪽으로 돌면서 갈 곳을 찾는" 탐색이다. 처음엔 `if found: break`를 for 밖에 뒀다가 로직이 꼬였다. for 안에서 찾으면 바로 break하고, 못 찾으면 뒤로 이동 로직으로 넘어가야 한다.
