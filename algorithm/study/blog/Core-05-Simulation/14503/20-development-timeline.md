# BOJ 14503 — 개발 타임라인 (후반)

## Phase 2
### Session 3
- 목표: "뒤로 이동" 로직의 정확성을 확인한다.
- 진행: 뒤로 이동은 현재 방향의 반대 방향이니까 `(d + 2) % 4`. 벽(grid == 1)이면 종료, 아니면 이동만 하고 방향은 유지한다. 이미 청소한 곳이라도 벽이 아니면 이동할 수 있다.
- 이슈: 처음엔 "뒤로 이동할 때도 청소 여부를 체크해야 하나" 헷갈렸다. 문제를 다시 읽어보니 뒤로 이동할 때는 청소 여부와 무관하게, 벽인지 아닌지만 확인한다.

### Session 5
- 정리:
  - 시뮬레이션 문제는 코드보다 규칙 정리가 먼저다. 규칙을 잘못 읽으면 코드 구조 전체가 달라진다.
  - `(d + 3) % 4`와 `(d + 2) % 4`라는 방향 연산이 이 문제의 핵심 빌딩 블록이다. 매번 if-else 분기로 방향을 처리하려 했으면 훨씬 지저분해졌을 것이다.
  - `cleaned` 배열과 `grid`를 분리한 것도 중요했다. grid는 벽 정보이고 cleaned는 방문 정보라서, 이 둘을 합치면 "뒤로 이동" 판단에서 꼬인다.
