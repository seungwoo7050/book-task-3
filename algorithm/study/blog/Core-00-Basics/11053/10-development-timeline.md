# BOJ 11053 — 개발 타임라인 (전반)

## Phase 1
### Session 1
- 목표: 문제를 읽고 "가장 긴 증가하는 부분 수열"이 정확히 뭘 요구하는지 정리한다.
- 진행: 처음엔 부분 수열이 연속 구간이라고 생각했다. 그런데 예제 `{10, 20, 10, 30, 20, 50}`에서 답이 4인 걸 보고, 연속이 아니라 원래 순서만 유지하면서 골라 뽑는 subsequence라는 걸 알았다.
- 이슈: subsequence라면 가능한 조합이 지수적이라서 브루트포스는 N=1000에서 절대 안 된다. 뭔가 중복 계산을 줄이는 구조가 필요하다.
- 판단: 각 위치에서 "여기서 끝나는 LIS 길이"를 저장하면, 이전 위치들 중 자기보다 작은 것만 찾아서 최대를 뽑으면 될 것 같다. 아직은 점화식이라기보다 직관이었다.

### Session 2
- 목표: 직관을 코드로 옮긴다.
- 진행: `dp[i]`를 "i번째 원소에서 끝나는 LIS 길이"로 정의했다. 초기값은 모두 1 — 자기 자신만으로도 길이 1짜리 수열이다. 매 i에 대해 0부터 i-1까지 훑으면서 `a[j] < a[i]`인 경우만 `dp[i] = max(dp[i], dp[j] + 1)`로 갱신한다.
- 이슈: 처음엔 `dp[n-1]`이 답이라고 생각했는데, LIS가 마지막 원소에서 끝나라는 보장이 없다. 예제가 50으로 끝나서 우연히 맞았을 뿐이다.
- 판단: 답은 `max(dp)` 전체에서 뽑아야 한다.

이 시점의 핵심 코드:

```python
dp = [1] * n

for i in range(1, n):
    for j in range(i):
        if a[j] < a[i]:
            dp[i] = max(dp[i], dp[j] + 1)

print(max(dp))
```

처음엔 이게 맞는 건지 확신이 없어서 손으로 예제를 따라가 봤다. `{10, 20, 10, 30, 20, 50}` → dp가 `[1, 2, 1, 3, 2, 4]`로 채워지는 걸 확인하고 나서야 구조가 맞다는 확신이 생겼다.

### Session 3
- 목표: 엣지 케이스에서 깨지는지 점검한다.
- 진행: `<` 대신 `<=`를 쓰면 어떻게 되는지 실험해 봤다. "비감소 수열"이 아니라 "엄격 증가 수열"이니까 `<=`를 쓰면 같은 값이 이어질 때 길이가 늘어나서 틀린다.
- 이슈: dp 초기값을 0으로 바꿔봤더니 원소 하나짜리 LIS 길이가 0이 나온다. 1이어야 한다.
- 판단: 두 가지 실수 포인트를 고정했다 — (1) 비교는 반드시 `<`, (2) 초기값은 반드시 1.
