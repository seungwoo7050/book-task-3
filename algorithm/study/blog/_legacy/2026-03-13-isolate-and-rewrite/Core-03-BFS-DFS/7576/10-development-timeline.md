# BOJ 7576 — 개발 타임라인 (전반)

## Phase 1
### Session 1
- 목표: 토마토가 익는 과정을 그래프 탐색으로 모델링한다.
- 진행: 처음엔 "모든 토마토가 익을 때까지의 최소 일수"를 구해야 하니까 각 익은 토마토에서 BFS를 따로 돌려야 하나 생각했다. 그런데 BFS를 여러 번 돌리면 방문 처리가 꼬인다.
- 이슈: 핵심 깨달음은 "모든 익은 토마토를 처음부터 큐에 넣고 BFS를 한 번만 돌리면 된다"는 것이었다. 이게 multi-source BFS다.
- 판단: 입력을 읽으면서 값이 1인 칸을 전부 `(행, 열, day=0)`으로 큐에 넣는다. BFS가 레벨 단위로 퍼지면서 day가 자동으로 증가한다.

### Session 2
- 목표: multi-source BFS를 구현한다.

이 시점의 핵심 코드:

```python
for i in range(N):
    row = list(map(int, input().split()))
    grid.append(row)
    for j in range(M):
        if row[j] == 1:
            q.append((i, j, 0))
```

이 코드가 중요한 이유는, 입력을 읽는 동시에 초기 시드를 큐에 채우는 구조가 multi-source BFS의 출발점이기 때문이다. 처음엔 입력을 다 읽고 나서 다시 돌면서 시드를 모았는데, 한 번에 하는 게 더 깔끔하다.

- 진행: BFS 본체는 일반 BFS와 같다. 4방향 탐색, 범위 체크, 방문 여부 대신 `grid[nx][ny] == 0`(안 익은 토마토)인지 확인.
- 이슈: 마지막에 안 익은 토마토(0)가 남아 있는지 전체 스캔을 해야 한다. 벽(-1)으로 둘러싸인 곳은 영원히 익지 않으니 -1을 출력해야 한다.
