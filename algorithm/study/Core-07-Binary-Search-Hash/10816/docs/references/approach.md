# Approach — BOJ 10816 (Number Card 2 (숫자 카드 2))

## 문제 요약

BOJ 10816는 입력 형식은 간결하지만, 실제 정답성은 세 가지 축에서 결정된다.
첫째, 입력을 문제 계약대로 안정적으로 해석해야 한다.
둘째, `빈도 해시맵(counter)으로 카드 개수를 누적하고 질의별 출력` 로직이 상태 불변식을 유지해야 한다.
셋째, 최종 출력 포맷이 채점기 기대와 정확히 일치해야 한다.
이번 문서의 목적은 단순 풀이 설명이 아니라, BOJ 10816의 의사결정 근거를 재사용 가능한 형태로 정리하는 것이다.

## 핵심 아이디어

1. 상태 정의: BOJ 10816에서 매 단계 이후 유지되어야 할 상태를 명시한다.
2. 전이 규칙: BOJ 10816 입력 단위를 처리할 때 `빈도 해시맵(counter)으로 카드 개수를 누적하고 질의별 출력`의 상태 갱신 순서를 고정한다.
3. 종료 조건: 반복/재귀를 언제 멈추고 무엇을 출력할지 분리한다.
4. 검증 루프: BOJ 10816 공식 테스트와 수동 케이스를 분리 기록해 전이 규칙이 실제 출력에서 유지되는지 확인한다.

이 문제에서 채택한 전략은 `빈도 해시맵(counter)으로 카드 개수를 누적하고 질의별 출력` 이다.
이 전략을 고른 이유는 BOJ 10816 제약에서 `빈도 해시맵(counter)으로 카드 개수를 누적하고 질의별 출력`가 구현 난이도와 검증 가능성의 균형을 가장 안정적으로 맞추기 때문이다.
특히 BOJ 10816처럼 입력 변형이 잦은 문제에서는, 상태를 단순하게 유지하는 설계가 디버깅 비용을 줄인다.

## 정당성(간단 증명)

### 상태 불변식

BOJ 10816의 각 단계가 끝났을 때, 현재 상태는 다음 단계를 수행하기에 충분한 정보를 잃지 않아야 한다.
즉, BOJ 10816에서는 앞 단계에서 계산한 핵심 값이 다음 전이에서 같은 의미로 재사용되어야 한다.
이 불변식이 BOJ 10816에서 무너지면 후속 계산 전체가 오염되므로, 구현 검토의 첫 체크포인트로 고정했다.

### 전이 타당성

전이 함수는 BOJ 10816 맥락에서 "현재 상태 + 새 입력"을 "다음 상태"로 바꾸는 계약(contract)으로 정의했다.
BOJ 10816 구현에서는 모든 분기에서 동일한 규칙을 적용하도록 구성해, 특정 입력에서만 다른 경로를 타는 비결정성을 제거했다.
수동 케이스에서 `양수/음수/미존재 값이 섞인 질의에서 빈도 조회를 검증한다.`를 집중 검증한 이유도 이 전이 타당성을 확인하기 위해서다.

### 종료 및 출력 정합성

종료 단계는 계산 완료 시점과 출력 직렬화 시점을 분리해 다룬다.
BOJ 10816는 정답 값만 맞아도 출력 포맷이 틀리면 오답이므로, 출력 규칙을 별도 검증 대상으로 취급했다.
공식 테스트 로그 `Results: 1/1 passed, 0 failed`는 BOJ 10816 종료 정합성이 유지되었음을 보여주는 1차 근거다.

## 복잡도

- 시간 복잡도: `O(N+M)`
- 공간 복잡도: `O(N)`
- 변수 정의: `N=카드 수, M=질의 수`

복잡도는 숫자 표기가 목적이 아니라, BOJ 10816의 입력 상한에서 해당 전략이 안전한지 설명하는 근거다.
BOJ 10816 문서에서는 복잡도 표기와 실행 로그를 나란히 배치해 이론 추정과 관측 결과가 어긋나지 않는지 확인한다.

## 대안 비교

| 대안 | 장점 | 미채택 이유 |
|:--- |:--- |:--- |
| 정렬 + lower/upper bound | 해시 없이도 해결 가능하다. | 구현 길이가 늘고 경계 처리가 복잡해진다. |
| 각 질의 선형 카운트 | 직관적이다. | 시간 복잡도 O(N*M)로 비효율적이다. |
| 현재 전략: 빈도 해시맵(counter)으로 카드 개수를 누적하고 질의별 출력 | 코드-문서-검증 로그를 일관되게 연결하기 쉽다. | 해당 없음 (채택) |

BOJ 10816에서는 "이론상 최적"보다 "실수 확률이 낮고 재검증 가능한 방법"을 우선 채택했다.
이 판단은 BOJ 10816 학습 기록을 다시 사용할 때 설명 가능성과 유지보수성을 동시에 확보하기 위한 선택이다.

## 실수 포인트

- 기본값 0 처리 누락(KeyError)
- 입출력 버퍼링 누락
- 질의 순서대로 출력하지 않는 문제
- 테스트는 통과했더라도, BOJ 10816 코드 변경 시에는 동일 체크리스트를 다시 실행해야 한다.
