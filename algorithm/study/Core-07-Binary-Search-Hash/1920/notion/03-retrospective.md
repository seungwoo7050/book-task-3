# BOJ 1920 — 회고

## 배운 것

"존재 여부 판별"은 해시 셋이 최적이다. $O(1)$ 평균 탐색은 이진 탐색의 $O(\log N)$보다 상수 배 빠르고, 정렬 전처리도 불필요하다. 단, 해시 충돌이 많은 악의적 입력에서는 $O(N)$까지 느려질 수 있으므로 이진 탐색의 최악 보장이 더 안전한 경우도 있다.

## 해시 vs 이진 탐색 트레이드오프

| 항목 | 해시 셋 | 이진 탐색 |
|------|--------|----------|
| 평균 쿼리 | $O(1)$ | $O(\log N)$ |
| 최악 쿼리 | $O(N)$ | $O(\log N)$ |
| 전처리 | $O(N)$ | $O(N \log N)$ |
| 메모리 | 더 큼 | 원본 배열 재사용 |
