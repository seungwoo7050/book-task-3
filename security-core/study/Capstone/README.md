# Capstone

`Capstone/`은 foundations 프로젝트에서 분리해 익힌 보안 vocabulary를 하나의 통합 review workflow로 다시 조합하는 구간입니다.
각 capstone은 앞선 프로젝트 코드를 그대로 import하지 않고, 같은 판단 기준을 다른 입출력 계약 위에서 다시 설명합니다.

## 프로젝트 지도

| 순서 | 프로젝트 | 상태 | 핵심 질문 |
| --- | --- | --- | --- |
| 01 | [collab-saas-security-review](collab-saas-security-review/README.md) | `verified` | primitive, auth, backend defense, dependency triage를 하나의 remediation workflow로 어떻게 묶을까 |

## 권장 읽기 순서

1. [../Foundations-Security/README.md](../Foundations-Security/README.md)에서 앞선 control vocabulary를 복습합니다.
2. [collab-saas-security-review/README.md](collab-saas-security-review/README.md)에서 capstone 범위를 읽습니다.
3. `problem/README.md`, `docs/README.md`, `python/README.md`, `notion/README.md` 순서로 통합 의도를 따라갑니다.
