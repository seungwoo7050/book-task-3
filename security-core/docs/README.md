# Docs Index

이 디렉터리는 `security-core`의 공개 표면 계약과 읽기 순서를 관리하는 공용 문서 묶음입니다. 프로젝트별 세부 설명은 각 프로젝트 안에 두고, 여기서는 레포 전체가 같은 질문에 답하도록 기준점을 고정합니다.

## 먼저 읽을 문서

1. [project-catalog.md](project-catalog.md): 프로젝트 전체를 `문제 / 내 답 / 검증 / 상태`로 한 번에 보는 단일 인덱스입니다.
2. [readme-contract.md](readme-contract.md): 루트 README와 프로젝트 README가 어떤 섹션 순서를 따라야 하는지 정의합니다.
3. [roadmap.md](roadmap.md): 번호형 시퀀스와 capstone 위치를 설명합니다.
4. [documentation-policy.md](documentation-policy.md): 문서 언어, 검증 명령, provenance 표기 원칙을 정리합니다.

## 이 디렉터리에서 답하는 질문

- 왜 이 레포를 번호형 프로젝트 시퀀스로 읽어야 하는가
- 각 프로젝트가 어떤 문제를 풀고 어떤 해답을 구현했는가
- README, `problem/`, `docs/`, `python/`, `notion/`이 각각 무엇을 설명해야 하는가
- 보안 주장을 어떤 명령과 fixture로 다시 검증할 수 있는가
