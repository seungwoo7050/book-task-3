# 전역 문서 인덱스

이 디렉터리는 레포 전체 구조, 경로 이동, 문서 규칙을 설명하는 공용 문서 묶음이다. 각 과제의 상세 내용은 `projects/` 아래로 내리고, 여기서는 공통 기준만 유지한다.

## 먼저 볼 문서

- [`catalog/README.md`](./catalog/README.md): 전역 인덱스와 경로 이전 문서 목록
- [`catalog/path-migration-map.md`](./catalog/path-migration-map.md): pre-migration 경로와 현재 경로 대응표
- [`curriculum/project-selection-rationale.md`](./curriculum/project-selection-rationale.md): 왜 두 과제를 이 구조로 재설계했는지
- [`curriculum/curriculum-map.md`](./curriculum/curriculum-map.md): stage와 capstone 전체 맵
- [`curriculum/reference-spine.md`](./curriculum/reference-spine.md): 레퍼런스 뼈대
- [`policies/readme-contract.md`](./policies/readme-contract.md): 루트/프로젝트/stage/capstone README 규칙
- [`policies/code-comment-language.md`](./policies/code-comment-language.md): 설명 주석과 docstring 언어 규칙

## 디렉터리 역할

- `catalog/`: 경로 이동, 프로젝트 인덱스, 탐색 문서
- `curriculum/`: 왜 이 과제를 선택했고 어떤 순서로 읽게 하는지
- `policies/`: 문서 표면과 코드 설명 언어 규칙

## 기억할 점

- 루트와 프로젝트 README는 `문제 / 공식 답 / 확장 답 / 검증`을 먼저 보여준다.
- `notion/`은 현재 재현용 노트, `notion-archive/`는 역사 보존용 노트다.
- 경로 예시는 추측이 아니라 실제 파일 기준으로 적는다.
