# 07 운영자 대시보드와 실험 콘솔 회고

## 이번 stage로 좋아진 점

- 추천 시스템이 단일 API가 아니라 운영 도구까지 포함한다는 점을 보여 준다.
- 학생이 자기 포트폴리오에서 운영자 UI를 어떻게 설명할지 참고할 수 있다.
- 최종 capstone의 화면 중심 시연 경로가 명확해진다.

## 아직 약한 부분

- 별도 stage 구현이 없으므로 실제 동작은 capstone 버전으로 내려가 확인해야 한다.
- 이 단계는 UI 컴포넌트 목록보다, 운영자가 무엇을 보고 어떤 결정을 내리는지에 초점을 둔다.

## 학생이 여기서 바로 가져갈 것

- 운영 콘솔을 "화면 모음"이 아니라 실험, compare, release 판단을 연결하는 의사결정 도구로 설명하는 방식
- 발표 문서와 실제 대시보드 화면을 같은 정보 구조 위에 올리는 방식

## 05-development-timeline.md와 같이 읽을 포인트

- operator dashboard와 experiment console이 각각 어떤 운영 질문에 답하는지 먼저 정리한다.
- capstone 발표 자료를 읽을 때도 이 stage의 정보 구조가 어떻게 화면 시연으로 이어지는지 함께 본다.

## 나중에 다시 볼 것

- 운영자 대시보드 IA 설계
- 실험 콘솔과 release console을 같은 제품 서사로 묶는 방식
