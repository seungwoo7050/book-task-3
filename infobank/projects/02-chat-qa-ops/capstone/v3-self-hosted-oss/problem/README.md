# v3 self-hosted OSS 문제 정의

## 이번 버전의 목표

`v3-self-hosted-oss`의 목표는 `v2`를 single-team self-hosted QA Ops 도구로 끌어올리는 것이다. 데모가 아니라 설치 가능한 운영형 아카이브로 한 단계 더 확장하는 버전이다.

## 필수 요구

- single admin auth
- transcript JSONL import
- Markdown ZIP KB import
- 비동기 evaluation job
- run-scoped dashboard / session review
- Docker Compose quickstart

## 범위 밖

- multi-tenant
- RBAC / SSO
- live chatbot serving
- billing / hosted SaaS
