# capstone 제출 정리 문제 정의

## 이 stage가 맡는 문제

운영형 MCP 추천 시스템 데모를 버전 스냅샷으로 마감하고, 필요하면 self-hosted OSS 확장까지 이어지는 구조로 설명하는 것이다.

## 현재 기준 성공 조건

- `v0`, `v1`, `v2`가 서로 다른 역할의 runnable snapshot으로 읽힌다.
- `v2`가 최종 제출 버전이라는 점이 분명하다.
- `v3`는 제품화 확장 버전이라는 점이 분명하다.
