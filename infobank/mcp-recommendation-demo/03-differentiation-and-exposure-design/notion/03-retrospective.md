# 03 차별화 포인트와 노출 설계 회고

## 이번 stage로 좋아진 점

- 추천 결과를 한국어 문장으로 납득 가능하게 설명할 수 있다.
- 차별화 포인트가 catalog 데이터와 UI 설명에 함께 반영된다.
- 학생이 자기 서비스 소개 문구와 recommendation copy를 함께 설계할 힌트를 얻는다.

## 아직 약한 부분

- 별도 stage 구현이 없으므로 실제 동작은 capstone 버전으로 내려가 확인해야 한다.
- 이 단계는 ranking 수치 자체보다, 추천 이유를 어떻게 표현할지를 다룬다.

## 학생이 여기서 바로 가져갈 것

- 추천 이유 문구와 UI exposure copy를 한 덩어리의 사용자 설명 계약으로 다루는 방식
- 한국어 사용자 설명력을 추천 품질의 일부로 취급하는 방식

## 05-development-timeline.md와 같이 읽을 포인트

- reason template와 UI 노출 파일을 먼저 나란히 읽으면 이 stage의 역할이 분명해진다.
- capstone 대시보드에서 보이는 한국어 근거를 읽을 때도 이 stage의 설계 의도를 기준으로 본다.

## 나중에 다시 볼 것

- 추천 로직과 설명 문구를 함께 설계하는 방식
- 한국어 사용자에게 보이는 exposure copy 정리법
