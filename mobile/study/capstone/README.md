# capstone

`capstone`은 같은 incident domain을 두 번 다루는 마무리 단계다.
첫 프로젝트는 system/contract 증명, 두 번째 프로젝트는 hiring-facing 제품 완성작을 목표로 한다.

## 포함 프로젝트

1. [01-incident-ops-mobile](01-incident-ops-mobile/README.md)
2. [02-incident-ops-mobile-client](02-incident-ops-mobile-client/README.md)

## 왜 두 프로젝트로 나누는가

- `01-incident-ops-mobile`은 DTO, backend, demo flow를 canonical source로 고정한다.
- `02-incident-ops-mobile-client`는 같은 도메인을 더 높은 UX 완성도와 오프라인 복구 경험으로 다시 구현한다.

## 되돌아보기

이 stage까지 오면 foundations의 화면/상호작용, architecture의 boundary 이해,
product-systems의 local-first 흐름이 하나의 제품 도메인 안에서 합쳐진다.
